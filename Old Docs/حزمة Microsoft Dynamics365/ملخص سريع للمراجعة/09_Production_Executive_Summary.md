# الملخص التنفيذي لإدارة الإنتاج في Microsoft Dynamics 365

## 1. الهدف من هذا الملخص

هذا الملف يلخص أهم الأفكار المستخرجة من دراسة إدارة الإنتاج في Microsoft Dynamics 365 Supply Chain Management. الهدف منه مساعدة مستشار ERP أو مسؤول نظام إنتاج على مراجعة المفاهيم الأساسية بسرعة قبل اجتماع عميل، أو قبل تحليل نظام إنتاج، أو قبل مقارنة نظام ERP محلي مثل ناتج مع نموذج إنتاج عالمي.

الملخص لا يستبدل الدراسة التفصيلية، لكنه يضع أمامك الصورة التنفيذية: كيف يتحول الإنتاج من بيانات أساس إلى أمر إنتاج، ثم إلى صرف مواد، واستلام منتج، وتكلفة، وWIP، وانحرافات، وتقارير، وإغلاق مالي.

---

## 2. الفكرة الكبرى

الإنتاج في ERP ليس شاشة لإدخال أمر إنتاج فقط. هو منظومة مترابطة تبدأ من تعريف المنتج والمواد، ثم BOM وRoute، ثم أمر إنتاج يمر بمراحل تشغيلية ومالية واضحة، وينتهي بتكلفة فعلية وتقارير رقابية.

الفكرة الأساسية في Dynamics 365 أن الإنتاج مرتبط مباشرةً بـ:

- بيانات المنتج والمواد.
- BOM أو Formula.
- Route وOperations وResources.
- المخزون والمستودعات.
- WIP.
- تكلفة المنتج.
- دفتر الأستاذ العام.
- الانحرافات والتقارير.

نجاح نظام الإنتاج لا يعتمد على تشغيل الشاشة، بل على جودة البيانات، ودقة BOM وRoute، ووضوح سياسة الصرف، وفهم WIP، واعتماد المالية لسياسة التكلفة والإغلاق.

## 3. دورة الإنتاج المختصرة

التسلسل المختصر لدورة الإنتاج في Dynamics 365:

1. Master Data
2. BOM / Formula
3. Route / Operations / Resources
4. Production Order Created
5. Estimated
6. Scheduled
7. Released
8. Started
9. Material Issue
10. Route / Job Reporting
11. Report as Finished
12. Quality / Scrap
13. Ended / Closed
14. Variance Analysis
15. Reporting

| المرحلة | ماذا يحدث؟ | الأثر الرئيسي |
|---|---|---|

| BOM / Formula | تعريف مكونات المنتج أو الوصفة | تحدد المواد المطلوبة والتكلفة المتوقعة |
| Route / Operations / Resources | تعريف مراحل التصنيع والموارد والأوقات | تؤثر على الجدولة وتكلفة التشغيل |
| Created | إنشاء أمر الإنتاج | لا يوجد أثر مخزني أو مالي فعلي بعد |
| Estimated | حساب التكلفة المقدرة وحركات On order | أول أثر مخزني تقديري |
| Scheduled | جدولة العمليات والموارد | حجز طاقة إنتاجية وتحديد مواعيد |
| Released | إطلاق الأمر للتنفيذ | يصبح الأمر متاحًا للإنتاج والمستودع |
| Started | بدء التنفيذ | يمكن تسجيل الصرف والوقت |
| Material Issue | صرف المواد للإنتاج غالبًا عبر Picking List Journal | نقص مخزون المواد وزيادة WIP |
| Route / Job Reporting | تسجيل وقت العمليات والموارد | يؤثر على تكلفة التشغيل والانحرافات |
| Report as Finished | استلام المنتج النهائي | زيادة مخزون المنتج التام وتخفيض WIP تقديريًا |
| Quality / Scrap | فحص المنتج وتسجيل الهالك أو الكمية المعيبة | يؤثر على توفر المخزون والتكلفة |
| Ended / Closed | إغلاق أمر الإنتاج | تسجيل التكلفة الفعلية والانحرافات |
| Variance Analysis | تحليل الفروقات | كشف أخطاء BOM وRoute والسعر والكمية |
| Reporting | مراجعة الإنتاج والتكلفة وWIP | دعم القرار الإداري والمالي |

## 5. BOM في دقيقة واحدة

- BOM تمثل المواد والمكونات المطلوبة لإنتاج المنتج.
- هي أساس الصرف؛ لأن الكمية المتوقعة للمواد تأتي منها.
- هي أساس التكلفة؛ لأن تكلفة المواد تُحسب منها.
- BOM Version تحدد أي نسخة من التركيبة تستخدم حسب الموقع أو الفترة أو الكمية.
- BOM غير المعتمدة لا تصلح للتطبيق، وBOM المفعّلة هي الافتراضية للاستخدام.
- BOM الخاطئة تؤدي إلى صرف خاطئ، نقص مخزون، تكلفة غير صحيحة، وانحرافات متكررة.
- Scrap داخل BOM يرفع كمية المواد المطلوبة بناءً على الهالك الثابت أو المتغير.
- Phantom يساعد على تفجير مكونات نصف مصنعة داخل الأمر الأب.
- Pegged Supply ينشئ توريدًا أو إنتاجًا مرتبطًا بحالة محددة.
- Vendor Line يستخدم عند وجود تصنيع أو خدمة خارجية مرتبطة بالإنتاج.

---

## 6. Route في دقيقة واحدة

Route يشرح كيف يُصنع المنتج، وليس ماذا يدخل فيه. إذا كانت BOM تجيب عن سؤال “ما المواد؟”، فإن Route يجيب عن “ما المراحل والموارد والأوقات؟”.

- Route يتكون من Operations وموارد وأزمنة تشغيل.
- Operations قد تكون تسلسلية أو متوازية.
- Resources قد تكون آلات، عمال، أدوات، مواقع، أو موردين خارجيين.
- Setup Time يؤثر على تكلفة الأمر، لأنه وقت تجهيز غالبًا لا يتكرر لكل وحدة.
- Run Time يؤثر على تكلفة الوحدة والجدولة.
- Cost Categories تحدد تكلفة العمليات والموارد.
- Route غير دقيقة تؤدي إلى جدولة غير واقعية وتكلفة تشغيل خاطئة وانحرافات عند الإغلاق.

---

## 7. Production Order في دقيقة واحدة

أمر الإنتاج هو مركز دورة الإنتاج في Dynamics 365. لا يمثل مجرد طلب تصنيع، بل كيان يربط BOM وRoute والمخزون والتكلفة والمالية.

يمر الأمر بحالات واضحة:

Created → Estimated → Scheduled → Released → Started → Reported as Finished → Ended

النقاط الحرجة:

- في Created لا توجد حركات فعلية.
- في Estimated تظهر حركات مخزون تقديرية وتكلفة مقدرة.
- في Released يصبح الأمر جاهزًا للتنفيذ.
- في Started يبدأ تسجيل الصرف والوقت.
- في Report as Finished يدخل المنتج النهائي للمخزون.
- في Ended تُحسم التكلفة الفعلية والانحرافات.
- ترك أوامر الإنتاج مفتوحة يؤدي إلى WIP متضخم وتقارير مالية مشوشة.
- تعديل BOM أو Route داخل الأمر يجب أن يكون مضبوطًا ومبررًا.

---

## 8. Material Issue في دقيقة واحدة

صرف المواد يتم عادة عبر Picking List Journal، وهو مرتبط مباشرةً بـ BOM.

- الكمية المتوقعة للصرف تأتي من BOM وكمية الأمر.
- الصرف اليدوي يعطي رقابة أعلى لكنه يتطلب انضباطًا.
- الصرف التلقائي أو Flushing مناسب للعمليات المتكررة، لكنه خطر إذا كانت BOM غير دقيقة.
- صرف كمية أكثر أو أقل من BOM يظهر لاحقًا في الانحرافات.
- صرف مادة بديلة قد يولد Production Substitution Variance.
- الصرف يخفض مخزون المواد ويزيد WIP إذا كان WIP مفعلًا.
- مرتجعات المواد من الإنتاج يجب تسجيلها لتصحيح المخزون وWIP.

---

## 9. Report as Finished في دقيقة واحدة

Report as Finished هو مرحلة الإبلاغ عن المنتج كمنجز واستلامه في المخزون.

- يزيد مخزون المنتج النهائي.
- يخفض WIP تقديريًا عند ترحيل المنتج.
- يدعم الإنتاج الجزئي؛ يمكن إنتاج جزء من الكمية وترك الباقي مفتوحًا.
- يتعامل مع Good Quantity وError Quantity.
- Error Quantity يمكن تحميلها على المنتج الجيد أو ترحيلها لحساب Scrap حسب Scrap Method.
- قد يولّد Quality Order إذا كانت الجودة مفعّلة.
- Deferred Posting يجعل المنتج متاحًا مخزنيًا قبل معالجة القيود المحاسبية، ويحتاج تحقق إضافي من المصدر الأصلي عند التطبيق التفصيلي.

---

## 10. WIP في دقيقة واحدة

WIP هو الرابط المالي بين المواد المصروفة والمنتج النهائي.

- يزيد WIP عند صرف المواد وتسجيل وقت أو موارد على أمر الإنتاج.
- ينخفض WIP عند Report as Finished وعند إغلاق الأمر.
- عند الإغلاق، تُعكس القيود التقديرية وتُسجل القيود الفعلية والانحرافات.
- أوامر إنتاج مفتوحة لفترات طويلة تعني WIP متضخمًا ومخزونًا ماليًا غير محسوم.
- فهم WIP ضروري للمحاسبة، لأن الإنتاج تحت التشغيل يظهر كقيمة مالية يجب تفسيرها.
- لا يكفي تشغيل الإنتاج دون سياسة واضحة لكيفية متابعة WIP وإغلاق الأوامر.

---

## 11. Costing في دقيقة واحدة

تكلفة المنتج في Dynamics 365 تأتي من تفاعل عدة عناصر:

- Material Cost من BOM.
- Route / Operation Cost من Route.
- Resource / Labor Cost من الموارد وCost Categories.
- Overhead من Costing Sheet.
- Scrap Cost من الهالك.

المفاهيم الأساسية:

- Standard Cost: تكلفة معيارية تستخدم لتقييم المخزون وتحليل الانحرافات.
- Actual Cost: التكلفة الفعلية بناءً على الحركات المسجلة.
- Estimated Cost: تظهر عند Estimate بناءً على BOM وRoute والأسعار النشطة.
- Actual Cost النهائية تظهر عند Closing.
- BOM Calculation تجمع تكلفة المواد والعمليات والـ Overhead.
- Costing Version تحفظ التكاليف المعيارية أو المخططة، وتفرق بين Pending وActive Cost.

قاعدة مهمة: لا توجد تكلفة موثوقة إذا كانت BOM أو Route أو أسعار المواد أو Cost Categories غير دقيقة.

## 12. Variances في دقيقة واحدة

| نوع الانحراف | ماذا يعني؟ |
|---|---|
| Lot Size Variance | فرق بسبب اختلاف الكمية الفعلية المنتجة أو تأثير توزيع التكاليف الثابتة مثل Setup على حجم دفعة مختلف |
| Production Quantity Variance | فرق بسبب صرف مواد أو تسجيل وقت أو إنتاج كمية أكثر أو أقل من المخطط |
| Production Price Variance | فرق بسبب اختلاف سعر المادة أو تكلفة Cost Category عن التكلفة المعيارية |
| Production Substitution Variance | فرق بسبب استخدام مادة أو Cost Category غير موجودة في BOM أو المعيار |

الانحرافات ليست مشكلة محاسبية فقط. هي أداة رقابية تكشف مشاكل BOM، Route، الأسعار، الكميات، البدائل، والهالك. إذا تكررت الانحرافات دون تفسير، فهذا يعني أن البيانات أو التشغيل لا يعكسان الواقع.

---

## 13. Closing في دقيقة واحدة

إغلاق أمر الإنتاج هو الحدث المالي الحاسم في الدورة.

- يحول التكلفة من تقديرية إلى فعلية.
- يعكس القيود التقديرية.
- يسجل التكلفة الفعلية.
- يظهر الانحرافات.
- يمنع تراكم WIP.
- يجعل الأمر غير قابل لإضافة تكاليف جديدة.

لا يجب إغلاق الأمر قبل مراجعة:

- المواد المصروفة.
- ساعات العمل أو العمليات.
- الكمية المنجزة.
- الجودة.
- الهالك.
- المخزون.
- التكلفة.
- الاعتماد المالي أو التشغيلي.

## 15. أهم 15 سؤالًا يجب طرحها على عميل إنتاج

| السؤال | لماذا مهم؟ |
|---|---|
| ماذا تنتجون؟ | لتحديد نوع الإنتاج والمنتجات النهائية |
| هل الإنتاج للتخزين أم حسب الطلب؟ | يؤثر على التخطيط وأوامر الإنتاج |
| هل لكل منتج BOM موثقة؟ | BOM أساس الصرف والتكلفة |
| من يعتمد BOM؟ | لمنع التعديلات غير المنضبطة |
| هل توجد منتجات نصف مصنعة؟ | يؤثر على أوامر فرعية أو Phantom |
| هل توجد مراحل إنتاج Route؟ | لتحديد الجدولة والتكلفة التشغيلية |
| هل تسجلون وقت العمال أو الآلات؟ | يحدد هل تدخل تكلفة التشغيل في المنتج |
| متى تصرف المواد؟ | يحدد طريقة Picking / Flushing وتأثير WIP |
| هل تسمحون بصرف مواد بديلة؟ | يؤثر على الانحرافات والمخزون |
| متى تعتبرون المنتج منجزًا؟ | يحدد Report as Finished |
| هل يوجد فحص جودة بعد الإنتاج؟ | يؤثر على توفر المخزون |
| كيف تسجلون الهالك؟ | يؤثر على التكلفة والانحرافات |
| هل تريدون Standard Cost أم Actual Cost؟ | يحدد نموذج التقييم والتحليل |
| هل تتابعون WIP؟ | ضروري للمالية والإغلاق |
| متى تغلقون أوامر الإنتاج؟ | يمنع تراكم WIP والانحرافات غير المحسومة |

| صرف مواد أكثر أو أقل من BOM | انحرافات ومخزون غير مطابق |
| عدم تسجيل الوقت | تكلفة تشغيل ناقصة أو غير واقعية |
| غياب سياسة الهالك | تكلفة منتج غير دقيقة |
| عدم مشاركة المالية | فشل الربط المحاسبي وProduction Posting |
| UAT ضعيف | ظهور مشاكل جوهرية بعد Go-Live |
| تقارير غير كافية | الإدارة لا تستطيع الرقابة أو القرار |
| تشغيل نموذج تكلفة معقد لعميل غير جاهز | فشل تشغيلي ومحاسبي رغم صحة النظام نظريًا |

## 18. أهم الدروس لنظام ERP محلي مثل ناتج

- لا يجب نسخ Dynamics حرفيًا؛ المطلوب فهم منطق الإنتاج المؤسسي.
- الحد الأدنى لأي نظام إنتاج محلي يجب أن يغطي: BOM، أمر إنتاج، صرف مواد، استلام منتج، وإغلاق أو تسوية واضحة.
- التكلفة وWIP والانحرافات يمكن تطبيقها تدريجيًا حسب حجم العميل ونضج عملياته.
- لا يجوز إهمال التقارير؛ الشاشات دون تقارير لا تعطي رقابة حقيقية.
- UAT لدورة إنتاج كاملة أهم من اختبار شاشة واحدة.
- يجب التفريق بين ميزة أساسية وميزة متقدمة؛ مثل Batch Balancing أو Co-products قد لا تكون مطلوبة لكل عميل.
- معرفة Dynamics يجب أن تتحول إلى أسئلة تحليل، Checklist، تقارير، وضوابط تطبيق، لا إلى تعقيد زائد.
- مشاركة المالية والمستودعات والإنتاج ضرورية؛ الإنتاج ليس مسؤولية قسم واحد.

## 19. ماذا أراجع بسرعة قبل مقابلة عميل؟

1. ما دورة الإنتاج الفعلية لدى العميل؟
2. هل توجد BOM موثقة ومعتمدة؟
3. هل توجد Route أو مراحل تشغيل واضحة؟
4. متى وكيف يتم صرف المواد؟
5. كيف يتم استلام المنتج النهائي؟
6. هل يوجد هالك أو جودة أو منتجات مرفوضة؟
7. كيف تُحسب تكلفة المنتج؟
8. هل يوجد WIP؟ ومن يراجعه؟
9. متى يتم إغلاق أمر الإنتاج؟
10. ما التقارير التي تحتاجها الإنتاج والمستودعات والمالية؟

## 20. خلاصة نهائية

القيمة الحقيقية من دراسة Dynamics 365 ليست حفظ أسماء الشاشات أو الحالات، بل بناء عقلية Functional Consultant يفهم كيف يتحول الإنتاج إلى مخزون وتكلفة وقيود وتقارير. الإنتاج في ERP يبدأ من بيانات صحيحة وBOM دقيقة وRoute واقعية، ثم يمر بصرف واستلام وتكلفة وWIP، وينتهي بإغلاق وانحرافات وتقارير.

عند تحليل أي نظام إنتاج أو عميل تصنيع، السؤال الأساسي ليس: هل توجد شاشة Production Order؟ بل: هل يستطيع النظام تمثيل دورة الإنتاج كاملة، وربطها بالمخزون والتكلفة والمالية، وإعطاء الإدارة تقارير يمكن اتخاذ قرار بناءً عليها؟
