# 06_Production_Reports_and_Controls.md
# تقارير ورقابة الإنتاج في Microsoft Dynamics 365

## 1. مقدمة

تقارير الإنتاج في نظام ERP ليست مخرجات للطباعة فقط، بل أدوات رقابة تشغيلية ومالية تساعد الإدارة على فهم ما يحدث داخل دورة الإنتاج من لحظة تجهيز البيانات الأساسية وحتى إغلاق أمر الإنتاج. في Microsoft Dynamics 365 Supply Chain Management، ترتبط التقارير بمنطق النظام نفسه: أمر الإنتاج يمر بحالات، والمواد تُصرف من المخزون، والتشغيل يُسجل عبر Route / Job، والمنتج النهائي يُستلم عبر Report as Finished، ثم تُغلق التكلفة وتظهر الانحرافات.

التقرير الجيد يجب أن يجيب عن سؤال واضح، مثل:

- هل أمر الإنتاج جاهز للتنفيذ؟
- هل المواد صُرفت حسب BOM؟
- هل WIP منطقي أم متضخم؟
- لماذا زادت تكلفة المنتج؟
- هل يمكن إغلاق أمر الإنتاج الآن؟
- هل يوجد خلل في BOM أو Route أو Standard Cost؟
- هل القيود المالية الناتجة عن الإنتاج صحيحة؟

هذا الملف يحوّل ما ورد في المقال الأصلي إلى مرجع عملي لتقارير ورقابة الإنتاج، مع التمييز بين التقارير المذكورة صراحة في المقال، والتقارير المقترحة بناءً على منطق دورة الإنتاج في المقال.

## 2. خريطة التقارير حسب دورة الإنتاج

| مرحلة دورة الإنتاج | التقارير أو الرقابة المطلوبة | الهدف |
|---|---|---|
| قبل الإنتاج | تقارير Master Data، BOM، Route، Costing Version | التأكد أن المنتج قابل للإنتاج والتكلفة قبل إنشاء الأمر. |
| أثناء الإنتاج | All Production Orders، Picking List Journal، Route Card / Job Card، Resource Capacity | متابعة تنفيذ الأمر، الصرف، الوقت، والموارد. |
| بعد Report as Finished | Report as Finished Journal، Good/Error Quantity، Quality Order، المنتجات المستلمة | التأكد من صحة الكميات المنتجة وحالة المخزون. |
| قبل الإغلاق | WIP Statement، المواد المصروفة، الوقت المسجل، الهالك، الجودة، الانحرافات المتوقعة | تحديد هل الأمر جاهز للإغلاق المالي والتشغيلي. |
| بعد الإغلاق | Production Variance Report، Production Order Cost Analysis، Production Posting Page | مراجعة التكلفة الفعلية والانحرافات والقيود. |
| بعد Go-Live | أوامر مفتوحة، WIP متضخم، أخطاء Picking/RAF، تقارير الاستخدام | كشف مشاكل التطبيق والتدريب والبيانات. |

## 3. تقارير Master Data

هذه التقارير مقترحة بناءً على منطق المقال؛ لأن المقال يوضح أن ضعف Master Data يؤدي إلى أخطاء في المخزون، الإنتاج، التكلفة، والقيود.

| التقرير | الهدف | المستخدم | القرار الناتج |
|---|---|---|---|
| تقرير المنتجات المعتمدة Released Products | التأكد من أن المنتجات جاهزة للاستخدام داخل الشركة القانونية. | مستشار ERP، الإنتاج، المخزون | هل المنتج جاهز للدخول في دورة إنتاج؟ |
| تقرير المواد الخام | حصر المواد التي تدخل في BOM أو Formula. | المستودعات، الإنتاج | هل المواد معرفة ومربوطة بالمخزون؟ |
| تقرير المنتجات نصف المصنعة | تحديد الأصناف التي يمكن أن تكون Output لأمر وInput في BOM آخر. | الإنتاج، التخطيط | هل تحتاج أوامر إنتاج فرعية أم Phantom؟ |
| تقرير المنتجات النهائية | تحديد المنتجات التي ستدخل المخزون بعد RAF. | الإنتاج، المالية | هل المنتج النهائي له إعدادات تكلفة ومخزون صحيحة؟ |
| تقرير Item Groups | التأكد من أن مجموعات الأصناف تقود إلى حسابات صحيحة. | المالية، ERP | منع ترحيل الإنتاج إلى حسابات خاطئة. |
| تقرير Item Model Groups | التحقق من طريقة تقييم المخزون والتحديث المالي/المادي. | المالية، مراقب التكلفة | هل الصنف يدعم طريقة التكلفة المطلوبة؟ |
| تقرير وحدات القياس | كشف اختلاف وحدات الشراء والتخزين والإنتاج. | المخزون، الإنتاج | منع صرف كميات خاطئة بسبب التحويل. |
| تقرير الأصناف بدون BOM | تحديد منتجات لا يمكن إنتاجها عمليًا. | الإنتاج، ERP | استكمال BOM قبل Go-Live. |
| تقرير الأصناف بدون تكلفة | كشف أصناف لا يمكن احتساب تكلفة دقيقة لها. | مراقب التكلفة، المالية | استكمال Costing Version أو الأسعار. |
| تقرير الأصناف بدون إعدادات مخزون أو مالية صحيحة | كشف أخطاء Item Group / Item Model Group / Dimensions. | المالية، ERP | منع أخطاء مالية ومخزنية بعد التشغيل. |

## 4. تقارير BOM / Formula

BOM هي أساس الصرف والتكلفة. أي خلل فيها يظهر لاحقًا كصرف خاطئ أو تكلفة خاطئة أو Variance عند الإغلاق.

| التقرير/الرقابة | ماذا يكشف؟ | من يحتاجه؟ | أثره على التكلفة |
|---|---|---|---|
| BOM المعتمدة Approved | المنتجات التي تمت الموافقة على قوائم موادها. | الإنتاج، الهندسة، ERP | يمنع استخدام BOM غير رسمية. |
| BOM المفعّلة Active | BOM الافتراضية التي يستخدمها التخطيط والإنتاج. | التخطيط، الإنتاج | يحدد التكلفة المتوقعة والصرف المتوقع. |
| BOM Versions | النسخ حسب الفترة، الموقع، الكمية، أو المنتج. | الإنتاج، التكلفة | يمنع استخدام وصفة غير مناسبة. |
| BOM المنتهية الصلاحية | نسخ لم تعد صالحة زمنيًا. | ERP، الإنتاج | يمنع تكلفة أو صرف مبني على نسخة قديمة. |
| BOM غير المعتمدة | قوائم غير صالحة للتخطيط أو التصنيع. | ERP، الإنتاج | خطر مباشر على Go-Live. |
| المنتجات التي لها أكثر من BOM | تعدد الوصفات أو الاستخدامات. | الإنتاج، التكلفة | يحتاج تحديد أي BOM للتخطيط وأيها للتكلفة. |
| BOM التي تحتوي Phantom | المنتجات نصف المصنعة المدمجة داخل الأمر الأب. | الإنتاج، التخطيط | تؤثر على تفجير المكونات وليس على إنشاء أمر منفصل. |
| BOM التي تحتوي Pegged Supply | أسطر تولّد أوامر شراء أو إنتاج مرتبطة. | التخطيط، المشتريات | تؤثر على التوريد المرتبط بالأمر. |
| BOM التي تحتوي Vendor Line | عمليات تصنيع خارجية أو مقاول باطن. | الإنتاج، المشتريات | تؤثر على التكلفة والمورد الخارجي. |
| BOM التي تحتوي Scrap | وجود Constant أو Variable Scrap. | مراقب التكلفة، الإنتاج | يرفع كمية المواد المطلوبة والتكلفة المعيارية. |
| BOM التي لا تحتوي Scrap رغم وجود هالك فعلي | اختلاف بين الواقع والنظام. | الإنتاج، التكلفة | يؤدي إلى انحرافات متكررة. |
| BOM Costing / BOM Calculation | تكلفة المنتج من المواد والعمليات والـ Overhead. | مراقب التكلفة، المالية | أساس Standard Cost أو التكلفة المتوقعة. |

### مؤشرات خطر في تقارير BOM

| مؤشر الخطر | ماذا يعني؟ | الإجراء المطلوب |
|---|---|---|
| منتج يُنتج بدون BOM معتمدة | النظام لا يملك وصفة رسمية للصرف والتكلفة. | إيقاف اختبار الإنتاج لهذا المنتج حتى اعتماد BOM. |
| BOM غير مفعّلة | قد توجد BOM لكنها ليست الافتراضية للإنتاج أو التخطيط. | مراجعة النسخة وتفعيلها ضمن قيود الموقع والفترة والكمية. |
| اختلاف BOM عن الواقع | الصرف الفعلي سيختلف عن المتوقع. | مراجعة BOM مع الإنتاج والمستودع. |
| عدم وجود مسؤول اعتماد | لا توجد ملكية واضحة للتغييرات. | تحديد مسؤول اعتماد BOM قبل Go-Live. |
| عدم تعريف Scrap رغم وجود هالك | التكلفة المعيارية ستكون أقل من الواقع. | تعريف Constant / Variable Scrap حسب الحالة. |

## 5. تقارير Routes / Operations / Resources

Routes تحدد كيف يُصنع المنتج، وتؤثر على الجدولة وتكلفة العمل والموارد. التقارير التالية مقترحة بناءً على منطق المقال، مع وجود أصلها في مفاهيم Route وOperations وResources وCost Categories.

| التقرير | ماذا يراقب؟ | أثره على الجدولة | أثره على التكلفة |
|---|---|---|---|
| المنتجات بدون Route | منتجات ليس لها مراحل تشغيل. | لا يمكن جدولة العمليات بدقة. | لا تدخل تكلفة التشغيل في المنتج. |
| Routes غير مفعلة | Route موجودة لكنها غير صالحة للاستخدام الافتراضي. | قد يستخدم النظام مسارًا غير صحيح أو يفشل. | تكلفة التشغيل قد تكون غير دقيقة. |
| Operations غير معرفة | غياب خطوات التصنيع التفصيلية. | ضعف التحكم في تسلسل العمل. | ضعف تحليل تكلفة العملية. |
| Resources بدون Calendar | موارد لا يمكن تحديد طاقتها. | فشل أو ضعف الجدولة، خاصة Finite Capacity. | وقت المورد غير قابل للتحميل بدقة. |
| Resources بدون Cost Category | مورد يعمل دون تكلفة واضحة. | لا يؤثر مباشرة على الجدولة. | تكلفة العمل أو الآلة لا تظهر بدقة. |
| Work Centers غير مرتبطة بعمليات | مورد أو مركز عمل غير مستخدم في Route. | طاقة غير مستغلة أو غير مرئية. | تكلفة غير مرتبطة بالإنتاج. |
| اختلاف الوقت الفعلي عن وقت Route | الفرق بين المخطط والفعلي. | مؤشر ضعف Route أو ضعف التنفيذ. | ينتج Production Quantity Variance. |
| تقرير الطاقة الإنتاجية | الطاقة المتاحة مقابل المستخدمة. | قرار تحميل أو تأجيل الأوامر. | يؤثر على تكلفة التأخير والكفاءة. |
| تقرير Setup Time وRun Time | أوقات الإعداد والتشغيل. | تحديد مدة الأمر. | تدخل في تكلفة التشغيل عند تفعيلها. |
| تقرير استخدام الموارد | الموارد المستخدمة فعليًا. | قياس كفاءة الموارد. | تحليل تكلفة المورد والعملية. |

## 6. تقارير أوامر الإنتاج Production Orders

| التقرير | الهدف | المستخدم | الخطر إذا لم يُراجع |
|---|---|---|---|
| أوامر الإنتاج حسب الحالة | متابعة Created, Estimated, Scheduled, Released, Started, RAF, Ended. | مدير الإنتاج، ERP | بقاء أوامر في حالة غير مناسبة دون انتباه. |
| أوامر الإنتاج المفتوحة | تحديد الأوامر غير المغلقة. | الإنتاج، المالية | WIP متضخم وتشويه مالي. |
| أوامر الإنتاج المتأخرة | كشف الأوامر التي لم تُنجز في موعدها. | التخطيط، الإنتاج | تأخر تسليم أو نقص مخزون. |
| أوامر الإنتاج غير المجدولة | كشف أوامر لم تُحجز لها طاقة. | التخطيط | ضعف استخدام الموارد وتأخير. |
| أوامر لم يتم Estimate لها | أوامر لم تُحسب تكلفتها أو احتياجاتها. | ERP، الإنتاج | بدء عمل دون تقدير مواد وتكلفة. |
| أوامر لم يتم Close لها | أوامر بقيت بعد RAF دون End. | المالية، مراقب التكلفة | لا تظهر التكلفة الفعلية والانحرافات. |
| أوامر مرتبطة بطلبات بيع أو Master Planning | تتبع مصدر الطلب. | التخطيط، المبيعات | ضعف ربط الإنتاج بالطلب. |
| أوامر إنتاج فرعية Pegged Orders | متابعة الأوامر التابعة للأمر الأب. | التخطيط، الإنتاج | تأخير أمر فرعي يوقف الأمر الرئيسي. |

## 7. تقارير صرف المواد Material Consumption

| التقرير | ماذا يكشف؟ | أثره على المخزون | أثره على التكلفة |
|---|---|---|---|
| Picking List Journal | الصرف الرسمي للمواد إلى الإنتاج. | يخفض المخزون المتاح. | يزيد WIP بقيمة المواد. |
| تقرير المواد المطلوبة حسب BOM | الكمية المتوقعة للصرف. | يحدد الاحتياج قبل التنفيذ. | أساس التكلفة المقدرة. |
| تقرير المواد المصروفة فعليًا | الاستهلاك الحقيقي. | يوضح Deducted / Issued. | أساس Actual Cost عند الإغلاق. |
| مقارنة المطلوب بالمصروف | الفرق بين BOM والواقع. | يكشف نقص أو فائض. | ينتج Quantity أو Substitution Variance. |
| تقرير صرف أكثر من BOM | استهلاك زائد. | يخفض المخزون أكثر من المخطط. | يرفع تكلفة الأمر. |
| تقرير صرف أقل من BOM | استهلاك ناقص. | قد يخفي نقصًا في الواقع. | يقلل التكلفة ظاهريًا وقد يسبب مشكلة جودة. |
| تقرير المواد البديلة | أصناف صُرفت بدل مواد BOM. | يؤثر على تتبع الأصناف. | ينتج Substitution Variance. |
| تقرير مرتجعات المواد من الإنتاج | مواد عادت من الإنتاج. | يعيد المخزون المتاح. | يخفض WIP. |
| تقرير مواد لم تصرف رغم بدء الأمر | نقص في تنفيذ الصرف أو Auto-consumption. | المخزون لا يعكس الواقع. | تكلفة الأمر ناقصة. |
| تقرير Batch/Serial للمواد المصروفة | تتبع الدفعات أو الأرقام التسلسلية. | يحسن التتبع. | يحتاج تحقق إضافي من المصدر الأصلي لتفاصيل التقرير. |

### قرارات يمكن اتخاذها من تقارير صرف المواد

| القرار | التقرير الداعم | ما الذي نبحث عنه؟ |
|---|---|---|
| هل BOM دقيقة؟ | مقارنة المطلوب بالمصروف | تكرار صرف أكثر أو أقل. |
| هل نسمح بالصرف التلقائي؟ | Picking List / فروقات الصرف | هل الواقع مستقر بما يكفي؟ |
| هل توجد مشكلة مستودعات؟ | مواد لم تصرف رغم بدء الأمر | نقص مواد، خطأ موقع، أو تأخير صرف. |
| هل توجد مواد بديلة متكررة؟ | تقرير المواد البديلة | تحديث BOM أو سياسة التوريد. |

## 8. تقارير Route / Job Reporting

| التقرير | الهدف | أثره على تكلفة الإنتاج |
|---|---|---|
| Route Card Journal | تسجيل استهلاك العمليات على مستوى Route. | يدخل تكلفة العمليات عند الترحيل. |
| Job Card Journal | تسجيل Jobs التفصيلية مثل الإعداد والتشغيل. | يعطي تكلفة أدق للوقت والموارد. |
| تقرير الوقت المخطط | عرض Setup / Run / Queue / Wait / Transport حسب Route. | يمثل أساس التكلفة المعيارية. |
| تقرير الوقت الفعلي | ما تم تسجيله فعليًا. | يدخل Actual Cost عند الإغلاق. |
| الفرق بين الوقت المخطط والفعلي | قياس كفاءة العملية. | ينتج Production Quantity Variance. |
| تقرير استخدام الموارد | الموارد أو الآلات المستخدمة. | يكشف تكلفة المورد وكفاءته. |
| العمليات غير المكتملة | عمليات لم تنته داخل الأمر. | تمنع إغلاقًا نظيفًا للأمر. |
| عمليات لم يتم تسجيل وقت لها | تشغيل تم دون تسجيل. | تكلفة ناقصة وانحرافات غير مفهومة. |
| تكلفة التشغيل حسب Operation أو Resource | توزيع تكلفة العمل والآلة. | يساعد على تحسين Route وCost Category. |

## 9. تقارير Report as Finished

| التقرير | ماذا يوضح؟ | من يستخدمه؟ | القرار الناتج |
|---|---|---|---|
| Report as Finished Journal | كميات المنتج النهائي المبلّغ عنها. | الإنتاج، المخزون | هل المنتج دخل المخزون؟ |
| تقرير الكميات المنجزة | الكمية المنتجة مقابل المخططة. | مدير الإنتاج | هل الأمر اكتمل أو يحتاج استمرار؟ |
| تقرير الإنتاج الجزئي | كميات RAF الجزئية. | الإنتاج، التخطيط | هل يُغلق جزء أم يستمر الأمر؟ |
| تقرير Good Quantity | الكمية السليمة. | الإنتاج، الجودة | قياس الناتج المقبول. |
| تقرير Error Quantity | الكمية المعيبة. | الجودة، التكلفة | تحديد Scrap Method والتكلفة. |
| تقرير المنتجات المستلمة في المخزون | المنتجات بحالة Received. | المستودعات، المالية | هل المخزون متاح أو تحت الجودة؟ |
| تقرير المنتجات التي تنتظر الجودة | مخزون Received لكنه Unavailable. | الجودة، المستودعات | هل يمكن إتاحته للبيع أو الاستخدام؟ |
| تقرير المنتجات المرفوضة أو المعاد تشغيلها | الكميات غير المقبولة أو Rework. | الجودة، الإنتاج | قرار إعادة تصنيع أو رفض. |
| العلاقة بين المواد المصروفة والكمية المنتجة | كفاءة التحويل من مدخلات إلى مخرجات. | مراقب التكلفة | كشف هالك أو ضعف BOM. |

## 10. تقارير الجودة والهالك

| التقرير | ماذا يكشف؟ | أثره على المخزون | أثره على التكلفة |
|---|---|---|---|
| تقرير Scrap | كميات الهالك المسجلة. | يقلل الكميات الصالحة. | قد يحمل على المنتج أو حساب Scrap. |
| تقرير Constant / Variable Scrap | الهالك المحدد في BOM أو Route أو Resource. | يرفع الاحتياج المتوقع. | يدخل في Standard Cost. |
| تقرير Error Quantity | الكمية المعيبة عند RAF. | لا تعامل ككمية جيدة. | حسب Scrap Method. |
| تقرير الهالك الطبيعي وغير الطبيعي | تصنيف رقابي للهالك. | يحتاج تحقق إضافي من المصدر الأصلي لتفاصيل تقرير جاهز. | يحدد هل يتحمل المنتج الجيد أو حساب مستقل. |
| تقرير المنتجات تحت الفحص | المنتجات غير المتاحة بسبب Quality Order. | يقيّد المخزون. | يؤخر الاعتراف بالتوفر التشغيلي. |
| Quality Order | فحص الجودة بعد الاستلام. | يحول المخزون إلى Unavailable حتى الفحص. | قد يقود إلى رفض أو قبول. |
| تقرير المنتجات المرفوضة | كميات لم تقبلها الجودة. | يمنع إتاحتها. | يرفع الهالك أو Rework. |
| تقرير Rework | أوامر إعادة تصنيع المنتجات المعيبة. | يعيد إدخال المنتج في دورة إنتاج. | يضيف تكلفة جديدة. |
| تقرير أثر الهالك على التكلفة | قيمة Scrap في الأمر. | غير مباشر. | يوضح أثر الهالك على تكلفة الوحدة. |
| تقرير Batch Attributes / Batch Balancing | خصائص الدفعة وضبط الكميات في التصنيع العملياتي. | يؤثر على اختيار الدفعات. | يحتاج تحقق إضافي من المصدر الأصلي لتفاصيل التقرير. |

## 11. تقارير WIP

WIP هو الجسر المالي بين المواد المصروفة والمنتج التام. المقال يذكر صراحة **Production WIP Statement** كأداة تعرض قيمة WIP الحالية لأي أمر.

| تقرير WIP | الهدف | المستخدم | القرار الناتج |
|---|---|---|---|
| WIP Statement | عرض قيمة WIP الحالية لأمر معين. | المحاسب، المالية | هل WIP منطقي أم يحتاج إغلاق/تصحيح؟ |
| أوامر الإنتاج المفتوحة التي تحتوي WIP | تحديد أوامر تحمل تكلفة معلقة. | المالية، ERP | متابعة الإغلاق. |
| WIP حسب أمر الإنتاج | تحليل تكلفة كل أمر. | مراقب التكلفة | هل الأمر يستحق الإغلاق أو المراجعة؟ |
| WIP حسب المنتج | معرفة المنتجات ذات تكلفة تحت التشغيل عالية. | المالية، الإدارة | قرار مراجعة دورة إنتاج المنتج. |
| WIP حسب الموقع أو المستودع | توزيع WIP جغرافيًا. | المالية، العمليات | يحتاج تحقق إضافي من المصدر الأصلي لتفاصيل التقرير. |
| WIP قبل الإغلاق | التأكد من كل التكاليف المعلقة. | المحاسب | عدم إغلاق قبل اكتمال الصرف والوقت. |
| WIP بعد RAF | قياس ما تبقى في WIP بعد استلام المنتج. | المالية | كشف RAF غير مكتمل أو مواد/وقت غير مغلقة. |
| WIP المتضخم بسبب أوامر غير مغلقة | مؤشر خلل في إجراءات الإغلاق. | CFO، مراقب التكلفة | فرض سياسة إغلاق دورية. |

### لماذا يهم WIP المحاسبة؟

لأن WIP يمثل أصولًا قيد التحويل. عند صرف المواد أو تسجيل العمل تزيد قيمته، وعند Report as Finished ينخفض لصالح المنتج التام، وعند Close تُعكس القيود التقديرية وتُرحل الفعلية والانحرافات. ترك أوامر الإنتاج مفتوحة يضخم WIP ويشوّه الميزانية وتقارير المخزون.

## 12. تقارير تكلفة الإنتاج

| التقرير | ماذا يعرض؟ | من يستخدمه؟ | القرار الناتج |
|---|---|---|---|
| Production Order Cost Analysis | تحليل تكلفة أمر الإنتاج والانحرافات. | مراقب التكلفة، المالية | هل تكلفة الأمر مقبولة؟ |
| Cost Estimates and Costings Report | مقارنة التكلفة التقديرية والفعلية. | مراقب التكلفة | هل التقدير قريب من الواقع؟ |
| BOM Calculation Report | تكلفة المنتج من BOM وRoute وOverhead ضمن Costing Version. | مراقب التكلفة | اعتماد أو مراجعة Standard Cost. |
| تقرير Estimated Cost | التكلفة المحسوبة عند Estimate. | الإنتاج، التكلفة | هل الأمر منطقي قبل التنفيذ؟ |
| تقرير Actual Cost | التكلفة بعد الصرف والتشغيل والإغلاق. | المالية | هل التكلفة الفعلية مقبولة؟ |
| تقرير Material Cost | تكلفة المواد الداخلة من BOM والصرف. | المستودعات، التكلفة | هل المواد سبب ارتفاع التكلفة؟ |
| تقرير Route / Resource Cost | تكلفة العمليات والموارد. | الإنتاج، التكلفة | هل الوقت أو المورد سبب الانحراف؟ |
| تقرير Overhead | المصاريف غير المباشرة المحملة. | المالية | هل التحميل صحيح؟ |
| تقرير Cost Groups | تقسيم التكلفة إلى مواد، عمل، Overhead. | مراقب التكلفة | أين تتركز تكلفة المنتج؟ |
| تقرير تكلفة الوحدة | تكلفة المنتج النهائي للوحدة. | المالية، الإدارة | تسعير أو مراجعة ربحية. |
| مقارنة التكلفة المقدرة والفعلية | الفرق بين Estimate وActual. | مراقب التكلفة | هل يجب تعديل BOM أو Route أو الأسعار؟ |

### أسئلة يجب أن يجيب عنها تقرير تكلفة الإنتاج

- كم تكلفة المواد؟
- كم تكلفة التشغيل والموارد؟
- كم Overhead محمّل؟
- ما تكلفة الهالك؟
- ما تكلفة الوحدة؟
- هل التكلفة الفعلية أعلى من Standard Cost؟
- ما مصدر الفرق: كمية، سعر، حجم دفعة، أم استبدال؟

## 13. تقارير الانحرافات Variances

| نوع الانحراف | ماذا يكشف؟ | المسؤول عن مراجعته | الإجراء المتوقع |
|---|---|---|---|
| Lot Size Variance | اختلاف الكمية الفعلية المنتجة عن كمية الحساب القياسي أو اختلاف تغطية التكاليف الثابتة. | الإنتاج، مراقب التكلفة | مراجعة أحجام الدُفعات وسياسة التخطيط. |
| Production Quantity Variance | صرف مواد أو تسجيل وقت أو استلام كمية أكثر/أقل من المخطط. | الإنتاج، المستودعات، التكلفة | مراجعة BOM وRoute والتنفيذ الفعلي. |
| Production Price Variance | اختلاف سعر المادة أو Cost Category عن Standard Cost. | المالية، المشتريات، التكلفة | مراجعة الأسعار أو تحديث Standard Cost. |
| Production Substitution Variance | استخدام صنف غير موجود في BOM أو Cost Category مختلفة. | الإنتاج، المشتريات، ERP | تحديث BOM أو ضبط سياسة البدائل. |

Production Variance Report المذكور في المقال يعرض الانحرافات الأربعة لكل أمر مغلق. قيمته ليست محاسبية فقط؛ بل يستخدم لتحسين BOM وRoute وStandard Cost وسلوك الصرف.

## 14. تقارير Production Posting والقيود المالية

| التقرير/الرقابة | الهدف المالي | المستخدم | الخطر إذا لم يُراجع |
|---|---|---|---|
| Production Posting Page | عرض كل الحركات المحاسبية لأمر معين. | المحاسب، ERP | عدم اكتشاف قيود خاطئة. |
| قيود Picking List | مراجعة أثر صرف المواد على WIP ومخزون المواد. | المالية، مراقب التكلفة | WIP أو مخزون مواد غير صحيح. |
| قيود Route / Job Card | مراجعة تحميل تكلفة الموارد والعمل. | المالية | تكلفة عمل ناقصة أو مبالغ فيها. |
| قيود Report as Finished | مراجعة دخول المنتج التام وخفض WIP. | المالية، المستودعات | مخزون منتج تام أو WIP غير صحيح. |
| قيود End Journal | مراجعة عكس التقديري وترحيل الفعلي والانحرافات. | المالية | بقاء تقديرات أو عدم ظهور Actual Cost. |
| Physical Posting vs Financial Posting | فهم الفرق بين التحديث المادي والمالي. | المالية، ERP | سوء تفسير الأرصدة. |
| Ledger Posting Levels | تحديد مصدر الحسابات: Item/Resource أو Item/Category أو Production Group. | المالية، ERP | مستوى تحليل غير مناسب أو حسابات خاطئة. |
| الحسابات حسب Item and Resource | تحليل دقيق حسب المورد أو مجموعة الموارد. | مراقب التكلفة | تعقيد زائد إن لم تكن الشركة جاهزة. |
| الحسابات حسب Item and Category | تحليل حسب Cost Category. | المالية | مناسب لتحليل العملية. |
| الحسابات حسب Production Group | تحليل مجمع. | المالية | قد يخفي التفاصيل إذا احتاج العميل دقة أعلى. |

## 17. تقارير UAT وGo-Live

| تقرير الاختبار | الهدف | قرار Go/No-Go |
|---|---|---|
| اختبار دورة إنتاج كاملة | تأكيد أن الدورة تعمل من المنتج إلى Close. | لا Go-Live دون نجاحها. |
| اختبار BOM | التأكد من الصرف والتكلفة. | Go إذا BOM صحيحة ومعتمدة. |
| اختبار Material Issue | التأكد من Picking والتأثير المخزني والمالي. | No-Go إذا الصرف لا يطابق الواقع. |
| اختبار Report as Finished | التأكد من استلام المنتج النهائي. | No-Go إذا RAF لا يرحل صحيحًا. |
| اختبار WIP | التأكد من دخول وخروج WIP. | No-Go إذا المالية لا تفهم أو توافق. |
| اختبار Variances | التأكد من ظهور الانحرافات. | Go مع آلية مراجعة واضحة. |
| اختبار Financial Posting | التأكد من القيود والحسابات. | لا Go-Live دون اعتماد المالية. |
| اختبار Production Reports | التأكد من توفر التقارير الأساسية. | No-Go إذا الإدارة لا تستطيع الرقابة. |
| تقرير نتائج UAT | توثيق النجاح والفجوات. | قرار نهائي. |
| قائمة الفجوات قبل Go-Live | تحديد ما يؤجل وما يمنع التشغيل. | Go/No-Go حسب خطورة الفجوات. |

## 21. علامات الخطر في التقارير

| علامة الخطر | ماذا تعني؟ | الإجراء المقترح |
|---|---|---|
| WIP متضخم | أوامر مفتوحة أو إغلاق ضعيف. | مراجعة أوامر الإنتاج المفتوحة وتطبيق سياسة Close. |
| أوامر إنتاج مفتوحة منذ فترة طويلة | ضعف متابعة أو مشاكل تنفيذ. | تقرير يومي للأوامر المفتوحة. |
| Variance مرتفع ومتكرر | BOM أو Route أو Standard Cost غير دقيقة. | تحليل نوع الانحراف وتحديث مصدره. |
| صرف مواد لا يطابق BOM | خطأ وصفة أو تنفيذ أو بدائل غير مضبوطة. | مراجعة BOM وسياسة الصرف. |
| Route وقتها الفعلي دائمًا أعلى من المخطط | Route غير واقعية أو ضعف تنفيذ. | تعديل أوقات Route أو تدريب الفريق. |
| هالك متكرر وغير معرف | Scrap غير مضبوط في النظام. | تعريف Scrap Method وScrap في BOM/Route/Resource. |
| منتجات تستلم دون جودة | خطر إتاحة مخزون غير مفحوص. | تفعيل Quality Order عند الحاجة. |
| قيود مالية غير متطابقة | إعداد Posting أو الحسابات غير صحيح. | مراجعة Production Posting مع المالية. |
| تقارير لا تستخدمها الإدارة | النظام ينتج بيانات بلا قرار. | ربط كل تقرير بقرار ومسؤول واضح. |

- تقرير أوامر مفتوحة.
- تقرير WIP أو قيمة تكلفة معلقة.
- تقرير تكلفة أمر الإنتاج.
- تقرير انحرافات أو فروقات بين المخطط والفعلي.
- تقرير قيود أو أثر مالي قابل للمراجعة.

### كيف تساعد هذه التقارير في بناء Dashboard إنتاج عملي؟

Dashboard إنتاج فعّال يجب أن يبدأ من أسئلة الإدارة، لا من شكل الرسوم. الحد الأدنى:

| محور Dashboard | المؤشرات |
|---|---|
| حالة الأوامر | Created, Started, RAF, Ended, Open. |
| المواد | مطلوب، مصروف، فرق، بدائل، مرتجعات. |
| التكلفة | Estimated, Actual, Standard vs Actual. |
| WIP | قيمة مفتوحة، أوامر قديمة، WIP حسب منتج. |
| الجودة والهالك | Good/Error، Scrap، Rework. |
| الانحرافات | Quantity, Price, Substitution, Lot Size. |

## 23. ملخص تنفيذي

تقارير الإنتاج في Dynamics 365 هي أداة رقابة وليست مجرد مخرجات. النظام لا يعطي قيمة حقيقية من Module الإنتاج إذا كانت الشاشات تعمل لكن الإدارة لا تستطيع رؤية المواد المصروفة، WIP، تكلفة الأمر، الانحرافات، أثر القيود، وحالة الإغلاق.

مدير الإنتاج يحتاج تقارير أوامر الإنتاج، Route/Job، الكميات المنتجة، الهالك، والموارد. مدير المستودعات يحتاج تقارير Picking List، الصرف، المرتجعات، وتتبع الدفعات. المحاسبة تحتاج WIP Statement، Production Posting، Cost Analysis، وVariance Report. الإدارة العليا تحتاج صورة مختصرة عن التكلفة، WIP، الانحرافات، والكفاءة.

عند تقييم أي ERP محلي مثل ناتج، لا يكفي وجود شاشة Production Order. يجب التأكد أن النظام يغطي الرقابة التشغيلية والمخزنية والمالية: BOM، Route، Material Issue، RAF، WIP، Costing، Variances، Posting، UAT، وتقارير ما بعد Go-Live. التقارير الضعيفة تعني أن نظام الإنتاج سيبقى إدخال بيانات لا منظومة إدارة.
