# قاموس مصطلحات الإنتاج في Oracle Fusion Cloud Manufacturing

## 1. مقدمة قصيرة

هذا الملف مستخرج من دراسة Oracle Fusion Cloud Manufacturing، والهدف منه بناء قاموس عملي يساعد مستشار ERP أو محلل الأعمال على فهم مصطلحات Module الإنتاج وربطها بدورة العمل، المخزون، التكلفة، والمحاسبة.

الفكرة الأساسية في Oracle أن الإنتاج لا يُدار كأمر تصنيع فقط، بل كمنظومة تفصل بين أربع طبقات رئيسية:

- **Work Definition**: تعريف طريقة الإنتاج قبل التنفيذ.
- **Work Order Execution**: تنفيذ أمر العمل وصرف المواد وتسجيل الموارد وإكمال المنتج.
- **Cost Accounting**: تسعير حركات الإنتاج وتوليد Cost Distributions.
- **Subledger Accounting / General Ledger**: تحويل أحداث التكلفة إلى قيود محاسبية قابلة للترحيل.

هذا الفصل مهم وظيفيًا؛ لأنه يمنع خلط التصميم بالتنفيذ، ويمنع أن تصبح المحاسبة نتيجة يدوية غير قابلة للتتبع.

| Inventory Management | وحدة إدارة المخزون في Oracle. | عام ERP بصياغة Oracle | تستقبل حركات Material Issue وProduct Completion وScrap وReturn. | كل حدث إنتاجي له أثر مخزني يمر عبر Inventory. |
| Cost Management | وحدة إدارة التكلفة. | عام ERP بصياغة Oracle | تحدد مناهج التكلفة، Cost Profiles، Cost Books، وCost Processor. | تحدد كيف تُسعّر حركات الإنتاج وكيف تظهر في المحاسبة. |

| Supply Chain Planning | التخطيط الذي يولد توصيات أوامر إنتاج. | عام ERP بصياغة Oracle | يحول الحاجة إلى Planned Orders قد تصبح Work Orders. | يؤثر على توقيت وكمية الإنتاج وتجديد المخزون. |
| Supply Chain Orchestration | وسيط تنسيقي يحول طلبات البيع أو التخطيط إلى أوامر توريد/إنتاج. | خاص Oracle | مهم في Back-to-Back وConfigure to Order. | يربط الطلب بأمر العمل والحجز والتسليم. |
| Order Management | إدارة أوامر البيع. | عام ERP بصياغة Oracle | مصدر الطلب في سيناريوهات Back-to-Back وConfigure to Order. | قد ينشئ أمر عمل مرتبطًا بطلب بيع محدد. |
| Global Order Promising | وظيفة وعد الطلب وتحديد إمكانية التوريد. | خاص Oracle | تحدد كيف ومتى يمكن تلبية الطلب. | تؤثر على قرارات إنتاج حسب الطلب. |
| Maintenance | إدارة الصيانة. | عام ERP بصياغة Oracle | تتكامل مع الاستثناءات ومشاكل الموارد والآلات. | تعطّل مورد قد يؤثر على الجدولة والتنفيذ. |
| Quality Management | إدارة الجودة والفحوصات والقضايا. | عام ERP بصياغة Oracle | تدعم Inspection وQuality Issues وProduction Exceptions. | تمنع تمرير منتج معيب وتؤثر على Scrap/Rework. |
| Procurement | الشراء. | عام ERP | يدخل في Outside Processing وشراء المواد لأمر عمل. | يؤثر على تكلفة المورد الخارجي وتوفر المواد. |
| Subledger Accounting | طبقة تحويل Cost Distributions إلى قيود محاسبية حسب قواعد محاسبية. | خاص Oracle | تُولّد القيود قبل ترحيلها إلى GL. | القيود التفصيلية تعتمد على قواعد SLA وليس على Manufacturing مباشرة. |
| General Ledger | الأستاذ العام. | عام ERP | يستقبل القيود النهائية. | يجب أن يطابق Cost Accounting وInventory Valuation. |

## 3. مصطلحات بيانات الإنتاج الأساسية Master Data

| المصطلح | الشرح | النوع | دوره في الإنتاج | أثر الخطأ فيه |
|---|---|---|---|---|
| Item | الصنف المستخدم كمادة أو منتج أو مخرج. | عام ERP | أساس كل Work Definition وWork Order. | تعريف خاطئ يؤدي لصرف أو تسعير أو تتبع غير صحيح. |
| Raw Material | مادة خام تدخل في تصنيع المنتج. | عام ERP | تظهر ضمن Item Structure وOperation Items. | خطأ الكمية أو التكلفة يؤثر على WIP وتكلفة المنتج. |
| Semi-Finished Good / Subassembly | منتج نصف مصنع قد يكون له Work Definition مستقل أو يُستخدم كـ Phantom. | عام ERP | يربط مستويات الإنتاج المتعددة. | عدم وضوحه يسبب أوامر ناقصة أو تفجير مواد خاطئ. |
| Finished Good / Assembly | المنتج النهائي الذي ينتجه Work Order. | عام ERP | رأس Work Definition ومخرَج Product Completion. | خطأ تعريفه يؤثر على الاستلام، التكلفة، والحجز. |
| Ad hoc Item | صنف غير موجود في Item Structure يمكن إضافته إلى Work Definition بصلاحية خاصة. | خاص Oracle | يستخدم لإضافة مواد غير قياسية على عملية. | استخدامه دون ضوابط يسبب Substitution/Usage Variance أو تكلفة غير متوقعة. |
| Item Structure | هيكل المنتج في Product Management ويُمثل مصدر المكونات. | خاص Oracle بصياغته | يغذي Work Definition بالمواد. | إذا لم يتزامن مع Work Definition، قد ينتج النظام ببيانات قديمة. |
| BOM | قائمة مكونات المنتج. في Oracle تُدمج عمليًا مع Routing داخل Work Definition. | عام ERP | تحدد المواد المطلوبة. | BOM غير دقيقة تؤدي لصرف وتكلفة وانحرافات غير صحيحة. |
| Unit of Measure — UOM | وحدة القياس للصنف أو المورد أو العملية. | عام ERP | تحدد حساب الكمية والوقت والتكلفة. | وحدة خاطئة تسبب كميات أو ساعات أو تكلفة غير منطقية. |
| Inventory Organization | منظمة المخزون. | عام ERP بصياغة Oracle | تمثل نطاق الأصناف والمستودعات والحركات. | الخطأ فيها يسبب ترحيلًا وتشغيلًا في نطاق خاطئ. |
| Manufacturing Plant | منظمة مخزون مفعلة للتصنيع. | خاص Oracle | المكان التنظيمي لتنفيذ الإنتاج. | إعداد غير صحيح يمنع أو يشوه عمليات الإنتاج. |
| Plant Parameters | معاملات المصنع مثل Enable Process Manufacturing وPrevent Issue of Expired Lots. | خاص Oracle | تتحكم في سلوك الإنتاج داخل المصنع. | خطأ فيها قد يؤدي لصرف Lots منتهية أو تفعيل نموذج غير مناسب. |
| Subinventory | تقسيم منطقي داخل منظمة المخزون. | خاص Oracle/عام كمستودع فرعي | مصدر صرف المواد أو مكان استلام المنتج. | Subinventory خاطئ يسبب أرصدة وتكلفة في مكان غير صحيح. |
| Locator | موقع تفصيلي داخل Subinventory. | عام ERP بصياغة Oracle | يحدد الرف أو الموقع الدقيق. | خطأه يسبب صعوبة في التتبع والجرد. |
| Work Area | منطقة عمل داخل المصنع. | خاص Oracle | تحتوي Work Centers. | عدم تعريفها يمنع بناء هيكل المصنع. |
| Work Center | مركز عمل يحتوي موارد وله Shifts. | عام ERP بصياغة Oracle | ينفذ العمليات ويحدد الطاقة والموارد. | خطأه يؤثر على الجدولة وتكلفة الموارد. |
| Workstation | موقع مادي داخل Work Center، يمكن ربطه بمورد أو Resource Instance. | خاص Oracle | يدعم Check-in/Check-out وتنفيذ أدق. | يحتاج تحقق إضافي من المصدر الأصلي عند تطبيقه محليًا. |
| Resource | مورد عمالة أو معدات يُستخدم في عملية. | عام ERP | يسجل وقت أو استخدام ويؤثر على التكلفة. | Resource غير معرف أو بلا Rate يضعف تكلفة التشغيل. |
| Resource Instance | نسخة محددة من مورد، مثل آلة معينة. | خاص Oracle | يعطي تتبعًا أدق للمورد الفعلي. | عدم ضبطه يضعف تتبع الأداء أو الأعطال. |
| Labor Resource | مورد عمالة. | عام ERP | يمثل العمالة المباشرة. | معدل خاطئ يؤثر على Resource Cost. |
| Equipment Resource | مورد معدات أو آلة. | عام ERP | يمثل استخدام الآلات في الإنتاج. | يسبب تكلفة وجدولة غير دقيقة إذا لم يضبط. |
| Standard Operation | عملية قياسية تُعرّف مرة ثم تُستخدم في Work Definitions. | خاص Oracle بصيغته | توحيد العمليات والموارد عبر المنتجات. | تعديل غير مضبوط يؤثر على منتجات متعددة. |
| Operation | خطوة إنتاجية داخل Work Definition أو Work Order. | عام ERP | تحدد أين ومتى تُصرف مواد أو تُسجل موارد. | ترتيب أو تعريف خاطئ يسبب تنفيذ وتكلفة خاطئة. |
| Operation Sequence | تسلسل العمليات. | عام ERP | يحدد ترتيب التنفيذ. | تسلسل خاطئ يؤدي لصرف أو إكمال غير منطقي. |
| Shift | فترة عمل للمورد أو Work Center. | عام ERP | تحدد الطاقة الزمنية المتاحة. | ضعفها يؤدي لجدولة غير واقعية. |
| Calendar | تقويم العمل. | عام ERP | يستخدم لحساب توفر الوقت والموارد. | Calendar خاطئ يسبب مواعيد تنفيذ غير صحيحة. |
| Costing Enabled | خاصية تحدد قابلية الصنف للتكلفة. | خاص Oracle | ضرورية لتسعير الحركات. | إهمالها يؤدي لحركات غير مسعرة أو تكلفة ناقصة. |
| Inventory Asset Value | خاصية تحدد هل الصنف أصل مخزني أم لا. | خاص Oracle | تؤثر على Cost Profile والتقييم. | خطأها يسبب معالجة مالية غير صحيحة. |
| Lot Control | تتبع الصنف بالدفعات. | عام ERP | مطلوب للمواد أو المنتجات المتتبعة. | غيابه يمنع Genealogy وسحب المنتجات المعيبة. |
| Serial Control | تتبع الصنف بالأرقام التسلسلية. | عام ERP | مهم للصناعات المنظمة. | غيابه يضعف التتبع لكل وحدة. |
| Supply Type | طريقة توريد/صرف المكون مثل Push وOperation Pull وPhantom. | خاص Oracle بصياغته | يحدد سلوك صرف المواد. | نوع خاطئ يسبب صرفًا مبكرًا أو مفقودًا أو غير نظامي. |

## 4. مصطلحات Work Definitions

| المصطلح | الشرح | علاقته بـ Work Definition | أثره على Work Order | أثره على التكلفة |
|---|---|---|---|---|
| Work Definition | قالب طريقة تصنيع المنتج، يجمع Item Structure وRouting وResources وOutputs وYield. | هو الكيان المركزي للتصميم. | تُنسخ منه Operations وItems وResources إلى Work Order. | أساس Cost Rollup وتكلفة العمل والموارد والهالك. |
| Work Definition Header | رأس التعريف ويحتوي Work Method والصنف والأولويات ومكان الإكمال. | يحدد الإطار العام للتعريف. | يوجه الأمر إلى طريقة إنتاج محددة. | يؤثر على اختيار التعريف لأغراض الإنتاج أو التكلفة. |
| Work Definition Version | إصدار بتاريخ سريان. | يحفظ تغييرات طريقة الإنتاج عبر الزمن. | الأمر يستخدم الإصدار الفعال عند الإنشاء. | يسمح بإعادة تسعير أوامر تاريخية حسب الإصدار الصحيح. |
| Standard Work Definition | تعريف إنتاج قياسي. | النوع الافتراضي للإنتاج العادي. | أساس Standard Work Order. | يدخل في Cost Rollup. |
| Primary Work Definition | التعريف الأساسي بأولوية إنتاج 1. | مطلوب كمرجع أساسي للصنف. | يستخدمه Planning غالبًا لتوليد أوامر. | قد لا يكون دائمًا نفس التعريف المستخدم للتكلفة إذا اختلفت Costing Priority. |
| Alternate Work Definition | تعريف بديل للمواد أو الخط أو العملية أو المورد. | يمثل بدائل تصنيع. | يمكن اختيار بديل في الأمر. | يؤثر على التكلفة إذا استخدم للـ Cost Rollup أو التنفيذ. |
| Rework Work Definition | تعريف لإعادة تشغيل منتج معيب. | نوع خاص في Discrete. | يستخدم لأمر Rework. | تكلفة Rework منفصلة وتضاف أو تؤثر على الصنف. |
| Transform Work Definition | تعريف لتحويل منتج إلى منتج آخر. | نوع خاص في Discrete. | يستخدم في Transform Work Order. | يحتاج تحقق إضافي من المصدر الأصلي لتفاصيل التكلفة. |
| Flow Manufacturing Work Definition | تعريف للإنتاج المتدفق بمعدل ثابت. | خاص بنمط Flow/Lean. | قد يرتبط بإنتاج Orderless. | مناسب للتكلفة المتدفقة؛ التفاصيل تحتاج تحقق إضافي من المصدر الأصلي. |
| Process Work Definition | تعريف للتصنيع العملياتي والدفعات. | يدعم Operation Outputs متعددة. | يستخدم في Batch/Process Work Orders. | يدعم Primary Output وCo-products وBy-products وتأثير Yield. |
| Work Method | طريقة العمل مثل Discrete أو Process أو غيرها. | تحدد نوع التعريف. | تحدد سلوك الأمر الناتج. | تؤثر على المخرجات والهالك وطريقة الإكمال. |
| Production Priority | أولوية استخدام التعريف في الإنتاج. | تحدد Primary/Alternate من ناحية الإنتاج. | تساعد التخطيط والتنفيذ على اختيار التعريف. | غير مباشرة؛ قد تختلف عن Costing Priority. |
| Costing Priority | أولوية استخدام التعريف في Cost Rollup. | تحدد التعريف المستخدم للتكلفة. | قد لا تغير التنفيذ مباشرة. | مهمة جدًا لحساب تكلفة معيارية صحيحة. |
| Completion Subinventory | المستودع الفرعي لاستلام المنتج النهائي. | في Discrete يكون على رأس Work Definition. | يحدد مكان Product Completion. | مكان الإكمال يؤثر على المخزون والتقييم. |
| Completion Locator | الموقع التفصيلي لاستلام المنتج. | يكمل Completion Subinventory. | يحدد Location الإكمال. | يفيد في تتبع المخزون. |
| Operation Output | مخرج العملية، خصوصًا في Process Manufacturing. | يحدد Primary/Co/By-products. | ينتج مخرجات متعددة من عملية. | يحدد توزيع أو تسجيل تكلفة المخرجات. |
| Operation Item | المادة المطلوبة في عملية. | تربط المكون بعملية محددة. | تصبح Material Requirement في Work Order. | تؤثر على Material Cost وUsage/Substitution Variance. |
| Operation Resource | المورد المطلوب في عملية. | يربط العملية بعمالة أو معدات. | يصبح Resource Requirement في Work Order. | يؤثر على Resource Cost وResource Variances. |
| Item Structure Synchronization | مزامنة التغييرات من Item Structure إلى Work Definition. | تحافظ على اتساق المكونات. | تمنع استخدام أمر بتعريف قديم. | تؤثر على دقة Cost Rollup. |
| Effective Start Date | تاريخ بداية سريان الإصدار. | يحدد متى يبدأ استخدام التعريف. | يتحكم في الإصدار الذي ينسخ للأمر. | مهم للتكلفة التاريخية والحديثة. |
| Effective End Date | تاريخ نهاية السريان. | يغلق صلاحية الإصدار. | يمنع استخدام تعريف منتهي. | يمنع Cost Rollup على تعريف غير صالح. |
| Serial Tracking in Work Definition | فرض تتبع السيريال بدءًا من عملية محددة إذا كان الصنف Serial-controlled. | يربط التنفيذ بالتتبع التفصيلي. | يلزم العامل بإدخال Serial عند الصرف أو الإكمال. | يدعم Genealogy والتكلفة حسب الوحدة عند الحاجة. |

### أهم ما يجب فهمه عن Work Definition في Oracle

1. Work Definition ليس BOM فقط، بل يجمع المواد والعمليات والموارد والمخرجات.
2. هو القالب الذي تُنشأ منه Work Orders.
3. وجود Primary Work Definition مهم لتخطيط الإنتاج.
4. Costing Priority قد تجعل تعريف التكلفة مختلفًا عن تعريف التشغيل.
5. Versions ضرورية لإدارة التغييرات التاريخية.
6. في Process Manufacturing، المخرجات قد تكون متعددة.
7. المواد والموارد تُربط بعمليات محددة لا برأس الأمر فقط.
8. Supply Type داخل Operation Item يحدد متى وكيف تُصرف المواد.
9. Work Definition الخاطئة تعني تنفيذًا وتكلفة خاطئة.
10. تعديل Item Structure يجب أن ينعكس بوضوح على Work Definition.

## 5. مصطلحات Work Orders

| المصطلح | الشرح | متى يستخدم؟ | أثره على التنفيذ | أثره على المخزون/التكلفة |
|---|---|---|---|---|
| Work Order | أمر عمل لإنتاج كمية من صنف في تاريخ معين. | عند تنفيذ الإنتاج. | يمثل الطلب التنفيذي الفعلي. | كل حركاته تُسعّر في Cost Accounting وتؤثر على WIP. |
| Work Order Header | رأس أمر العمل ويحمل الصنف والكمية والتواريخ وWork Definition. | عند إنشاء الأمر. | يحدد طبيعة العمل والكمية والمواعيد. | يصبح Cost Object للحركات. |
| Standard Work Order | أمر يستند إلى Work Definition. | الإنتاج الطبيعي. | ينسخ العمليات والمواد والموارد من التعريف. | قابل للتكلفة والتحليل القياسي. |
| Nonstandard Work Order | أمر لا يستند إلى Work Definition. | إصلاح أو تجربة أو إنتاج لمرة واحدة. | مرن لكنه يحتاج ضبطًا. | التكلفة قد تكون أقل قابلية للمقارنة. |
| Rework Work Order | أمر لإعادة تشغيل منتج معيب. | عند إصلاح منتج مرفوض. | ينفذ مسار إعادة التشغيل. | تكلفة Rework تتبع الأمر. |
| Transform Work Order | أمر لتحويل منتج إلى آخر. | Upgrade/Downgrade أو تحويل. | يغير مخرج الإنتاج. | يحتاج تحقق إضافي من المصدر الأصلي. |
| Orderless Manufacturing | إنتاج دون أمر عمل، يُسجل بالحركة. | الإنتاج المتدفق Flow. | يقلل عبء أوامر العمل. | يحتاج ضبطًا قويًا للتكلفة والتتبع. |
| Contract Manufacturing | تصنيع تعاقدي لدى مصنع خارجي. | عند إسناد تصنيع كامل لطرف ثالث. | يتطلب تتبع خارجي. | يؤثر على المخزون والتكلفة عبر علاقات المورد. |
| Outside Processing | تنفيذ عملية أو أكثر لدى مورد. | عندما تكون عملية ضمن الأمر خارجية. | يربط Operation بالمورد. | تكلفة العملية الخارجية تدخل في تكلفة المنتج. |
| Supplier Operation | عملية ينفذها المورد. | ضمن Outside Processing. | تُدار كتوريد خدمة مرتبطة بأمر. | تؤثر على Cost Accounting. |
| Back-to-Back Work Order | أمر مرتبط بطلب بيع محدد. | عند الإنتاج حسب طلب البيع. | يحجز الناتج لذلك الطلب. | يربط المخزون النهائي بالحجز والشحن. |
| Configure to Order Work Order | أمر لمنتج قابل للتكوين حسب الطلب. | عندما يحدد العميل التكوين. | ينشأ عبر Orchestration/Order Management. | يحتاج تحقق إضافي من المصدر الأصلي للتكلفة التفصيلية. |
| Project-Specific Work Order | أمر مرتبط بمشروع. | عند وجود Project-driven supply chain. | يفصل التوريد حسب المشروع. | يؤثر على التتبع والتكلفة للمشروع. |
| Planned Order | أمر مخطط من Supply Chain Planning. | قبل التحويل إلى Work Order. | يمثل توصية إنتاج. | لا يسبب تكلفة تنفيذ حتى يتحول لأمر. |
| Work Order Reservations | حجز المنتج أو المواد لأمر/طلب. | خاصة Back-to-Back أو الطلبات المحددة. | يحافظ على الربط بين الإنتاج والطلب. | يؤثر على توفر المخزون. |
| Dispatch List | قائمة تشغيل تظهر أوامر Released للمشغلين. | بعد Release. | توجه العامل لما يجب تنفيذه. | لا أثر مالي مباشر، لكنها تضبط التنفيذ. |

## 6. مصطلحات دورة حياة Work Order

| الحالة/المرحلة | الشرح | ما يسمح به؟ | ما يمنع؟ | الأثر المخزني/المالي |
|---|---|---|---|---|
| Unreleased | أمر منشأ وغير جاهز للتنفيذ. | تعديل تفاصيل الأمر. | تنفيذ حركات عليه. | لا أثر مخزني مباشر؛ يصبح Cost Object محتمل. |
| Released | أمر جاهز للتنفيذ. | صرف مواد، تسجيل موارد، إكمال عمليات. | حسب الإعداد: قد يمنع تعديلات معينة. | تبدأ حركات WIP والمخزون عند التنفيذ. |
| On Hold | أمر موقوف مؤقتًا. | الرجوع إلى Released أو الإلغاء. | الحركات محجوبة. | يمنع تراكم حركات غير مرغوبة. |
| Pending Approval | حالة تظهر عند تفعيل E-Records/E-Signatures لتغيير Release. | انتظار الاعتماد. | التنفيذ قبل الاعتماد. | يحتاج تحقق إضافي من المصدر الأصلي. |
| Completed | تم إكمال آخر عملية أو الكمية المطلوبة. | الانتقال إلى Closed. | غالبًا لا يستمر التنفيذ العادي. | المنتج مستلم، WIP قد يبقى حتى الإغلاق. |
| Closed | إغلاق نهائي. | لا يسمح بالتعديل الطبيعي. | تعديل أو تنفيذ جديد. | توليد الانحرافات/التسويات وتسوية WIP. |
| Canceled | إلغاء الأمر. | لا يسمح بالتعديل. | التنفيذ. | أثره يعتمد على وجود حركات سابقة؛ يحتاج تحقق إضافي. |
| Create Work Order | إنشاء الأمر. | نسخ العمليات والمواد والموارد. | التنفيذ قبل Release. | لا حركة مخزون مباشرة. |
| Release Work Order | إطلاق الأمر. | ظهور في Dispatch List وبدء التنفيذ. | التنفيذ قبل الإطلاق. | لا يعني الصرف؛ يفتح الباب للحركات. |
| Issue Materials | صرف المكونات إلى الأمر. | إنقاص المواد من Supply Subinventory. | الصرف لأمر غير صالح. | يزيد WIP بقيمة المواد. |
| Execute Operations | تنفيذ العمليات. | إكمال عمليات وتسجيل كميات. | تجاوز Count Point إن كان إلزاميًا. | يؤثر على تقدم الأمر. |
| Resource Charging | تسجيل استخدام الموارد. | إضافة تكلفة عمال/آلات. | تسجيل مورد غير مرتبط دون ضبط. | يزيد WIP بتكلفة Resource. |
| Material Return | إرجاع مادة مصروفة. | زيادة المخزون. | إرجاع أكثر من المصروف دون ضبط. | يخفض WIP. |
| Product Completion | استلام المنتج النهائي. | زيادة Finished Goods. | الإكمال دون بيانات مطلوبة مثل Lot/Serial. | يخفض WIP تقديريًا ويرفع مخزون المنتج. |
| Work Order Close | إغلاق الأمر. | حساب التكلفة النهائية والانحرافات. | الإغلاق مع حركات معلقة. | يسوي WIP ويولد Variances/Adjustments. |

## 7. مصطلحات Supply Types والمواد

| المصطلح | الشرح | أثره على صرف المواد | أثره على المخزون | ملاحظات |
|---|---|---|---|---|
| Push Supply | صرف يدوي صريح للمادة. | المستخدم يصرف المادة يدويًا. | ينقص المخزون عند الصرف. | مناسب للمواد الحساسة أو غير المتكررة. |
| Operation Pull | صرف آلي عند إكمال العملية المعنية. | Backflush مرتبط بعملية. | ينقص المخزون تلقائيًا عند Completion. | يحتاج Work Definition دقيقة. |
| Assembly Pull | صرف آلي عند إكمال المنتج النهائي. | Backflush عند Product Completion. | ينقص المواد مع إكمال المنتج. | مناسب لتدفقات متكررة. |
| Bulk | مادة تظهر للتخطيط أو العرض ولا تُصرف نظاميًا. | لا تُصرف كحركة WIP عادية. | لا أثر صرف مباشر. | سوء استخدامه يسبب تكلفة ناقصة. |
| Phantom | عنصر تُفجّر مكوناته داخل الأمر. | لا يصرف كعنصر مستقل. | مكوناته تظهر كمطلوبات. | مفيد للتركيبات الوسيطة غير المصنعة كأمر مستقل. |
| Supplier Supply | مواد يوفرها المورد في Outside Processing. | لا تأتي من مخزون الشركة بنفس المنطق. | أثرها يعتمد على OSP. | يحتاج تحقق إضافي عند التطبيق. |
| Component Issue | صرف مكون لأمر العمل. | يخلق WIP Material Issue. | ينقص المواد. | يدخل Cost Accounting. |
| Component Return | عكس صرف مكون. | يرجع مادة مصروفة. | يزيد المخزون. | يخفض WIP. |
| Material Issue | صرف المواد للإنتاج. | أساس إدخال المواد إلى WIP. | ينقص Supply Subinventory. | مفهوم عام ERP؛ في Oracle يمر عبر Inventory ثم Costing. |
| Material Return | إرجاع مواد من الإنتاج. | عكس حركة الصرف. | يزيد Supply Subinventory. | يجب تحديد Lot الأصلي عند Lot Control. |
| Backflush | صرف تلقائي بناءً على Completion. | يقلل الإدخال اليدوي. | ينقص المخزون تلقائيًا. | خطر إذا كانت Work Definition غير دقيقة. |
| Supply Subinventory | المستودع الفرعي الذي تُصرف منه المادة. | يحدد مصدر الصرف. | يتحكم في الرصيد المتأثر. | خطأه يشوه أرصدة المستودعات. |
| Supply Locator | الموقع التفصيلي لمصدر الصرف. | يحدد Location الصرف. | يؤثر على الجرد والتتبع. | يحتاج عندما تكون المنظمة Locator-controlled. |
| Expired Lots | دفعات منتهية الصلاحية. | يجب منع صرفها عند الحاجة. | تمنع استخدام مخزون غير صالح. | مرتبط بـ Prevent Issue of Expired Lots. |
| Prevent Issue of Expired Lots | إعداد يمنع صرف Lots منتهية. | يوقف الصرف غير المسموح. | يحسن سلامة المخزون. | خاص Oracle كـ Plant Parameter/تحسين حديث. |
| Lot / Serial Traceability | تتبع الدفعات أو السيريالات. | يلزم إدخال Lot/Serial عند الصرف. | يوفر مسارًا من المادة للمنتج. | مهم للامتثال والاستدعاء Recall. |
| Genealogy | شجرة ربط المواد والدفعات بالمنتج النهائي. | يربط Lot/Serial المكون بالمنتج. | يدعم تتبع كامل. | أداة رقابية قوية للجودة. |

## 8. مصطلحات العمليات والموارد

| المصطلح | الشرح | أثره التشغيلي | أثره على الجدولة | أثره على التكلفة |
|---|---|---|---|---|
| Count Point Operation | عملية يجب تسجيل إكمالها صراحة. | نقطة رقابة إلزامية. | تحدد تقدم الأمر. | عدم تسجيلها يمنع إكمال صحيح. |
| Auto-Transact Operation | عملية تُكمل تلقائيًا عند إكمال عملية لاحقة. | يقلل الإدخال اليدوي. | يسرع تقدم العمليات. | قد تُسجل تكلفة تلقائيًا حسب الموارد. |
| Optional Operation | عملية اختيارية يمكن تجاوزها. | مرونة في التنفيذ. | لا توقف الأمر إذا لم تُنفذ. | لا تُحتسب تكلفتها إذا لم تُسجل. |
| Resource Charging | تسجيل تكلفة استخدام مورد. | يثبت استخدام العمالة أو الآلة. | قد يرتبط بتقدم العملية. | يزيد WIP بتكلفة المورد. |
| Manual Resource Charging | تسجيل يدوي لاستخدام المورد. | يتطلب إدخال المستخدم. | أدق إذا تم الالتزام. | خطر الخطأ أو النسيان. |
| Automatic Resource Charging | تسجيل آلي لاستخدام المورد. | يقلل الإدخال اليدوي. | مرتبط بالتنفيذ. | يعتمد على دقة Work Definition. |
| Operator Check-in | تسجيل دخول العامل/المشغل على Workstation. | يثبت بداية العمل. | يدعم حساب الوقت. | يمكن أن يدعم Resource Usage. |
| Operator Check-out | تسجيل خروج العامل/المشغل. | يثبت نهاية العمل. | يحسب مدة العمل. | يدعم حساب تكلفة الموارد إذا مفعّل. |
| Resource Usage | كمية استخدام المورد، غالبًا وقت. | يحدد الجهد الفعلي. | يدخل في المدة. | Usage × Rate = Resource Cost. |
| Resource Rate | معدل تكلفة المورد. | لا يغير التشغيل مباشرة. | لا يغير الجدولة. | أساس تكلفة الموارد المنشورة في Cost Scenario. |
| Utilization | نسبة استخدام المورد. | تعبر عن القدرة العملية. | أقل من 100% تطيل المدة المجدولة. | أثرها غير مباشر على تكلفة التخطيط. |
| Efficiency | كفاءة المورد. | تحدد الإنتاجية. | تؤثر على مدة الجدولة. | قد تؤثر على تكلفة الاستخدام إذا زاد الزمن. |
| Operation Yield | نسبة الناتج المتوقع في العملية. | يحدد الفاقد المتوقع. | يؤثر على الكميات المطلوبة. | يولد Estimated Scrap وYield Variance. |
| Undercompletion Tolerance | السماح بإغلاق/اعتبار الأمر مكتملًا بكمية أقل. | يدعم الإنتاج الأقل من المخطط. | يؤثر على اعتبار الأمر مكتملًا. | قد يسبب Variances أو Adjustments. |
| Setup | وقت أو جهد تجهيز. | يحدث قبل التشغيل. | يزيد مدة العملية. | يدخل في Resource Cost إذا كان معرفًا. |
| Run Time | وقت التشغيل. | يمثل زمن الإنتاج. | أساس مدة العملية. | خطأه يسبب Resource Efficiency Variance. |
| Resource Availability | توفر المورد. | يحدد قابلية التنفيذ. | أساس جدولة الطاقة. | تعطله قد يسبب تأخيرًا. |
| Work Center Capacity | طاقة مركز العمل. | تحدد ما يمكن إنتاجه. | أساس التخطيط. | تأثيرها المالي غير مباشر عبر التأخير والكفاءة. |

## 9. مصطلحات Product Completion والهالك والجودة

| المصطلح | الشرح | أثره على الإنتاج | أثره على المخزون | أثره على التكلفة/الجودة |
|---|---|---|---|---|
| Product Completion | تأكيد إنتاج كمية وإيداعها في المخزون. | ينهي كمية من الأمر. | يزيد Finished Goods. | يُسعّر بتكلفة تقديرية ثم تُحسب الفعلية عند الإغلاق. |
| Provisional Completion Cost | تكلفة مؤقتة لإكمال المنتج قبل Close. | تسمح باستلام المنتج قبل التكلفة النهائية. | تزيد المخزون بقيمة مؤقتة. | تُعدل عند Work Order Close. |
| Completion Subinventory | المستودع الفرعي لاستلام المنتج. | يحدد وجهة الإكمال. | يزيد رصيد ذلك المستودع. | يؤثر على التقييم حسب Cost Profile/Valuation. |
| Operation Output | مخرج عملية في Process. | يسمح بمخرجات متعددة. | يزيد رصيد المخرجات. | يؤثر على تكلفة المخرجات. |
| Primary Output | المخرج الرئيسي في Process. | يمثل المنتج الأساسي. | يستلم كمخرج رئيسي. | أساس تكلفة الدفعة. |
| Co-product | منتج مشترك. | ينتج مع المخرج الأساسي. | يضاف للمخزون. | يحتاج توزيع تكلفة؛ التفاصيل تحتاج تحقق إضافي. |
| By-product | منتج ثانوي. | ينتج عرضيًا. | يضاف للمخزون إذا مُدار. | تكلفة/ائتمان حسب الإعداد؛ يحتاج تحقق إضافي. |
| Scrap | هالك غير قابل للإصلاح. | يقلل الناتج الجيد. | قد لا يدخل مخزون قابل للبيع. | إما يحمّل على المنتج أو يذهب كمصروف. |
| Scrap Transaction | حركة تسجيل الهالك. | تثبت كمية الهالك. | تخرج أو تخفض قيمة WIP. | تُعاد تكلفة Scrap عند Close. |
| Rework | إعادة تشغيل منتج معيب لإصلاحه. | يعيد المنتج لمسار إصلاح. | قد يعود للمنتج الجيد. | له Work Order وتكلفة منفصلة. |
| Reject | رفض كمية/منتج. | يوجه إلى Scrap أو Rework أو قرار آخر. | قد يمنع إتاحة المنتج. | يؤثر على الجودة والتكلفة. |
| Production Exception | استثناء إنتاجي لمشكلة في المصنع. | يوقف أو يعرقل تقدم العمل. | قد يؤخر الصرف/الإكمال. | يؤثر على قرارات المشرف. |
| Resource Exception | استثناء بسبب مورد مثل آلة أو عامل. | يعطل عملية. | لا أثر مخزني مباشر. | قد يسبب تأخيرًا واستخدام مورد بديل. |
| Material Exception | استثناء بسبب مادة مثل نقص المكون. | يمنع أو يؤخر التنفيذ. | يكشف نقص مخزون. | قد يؤدي لصرف بديل أو Variance. |
| Quality Issue | مشكلة جودة. | توجه للتحقيق أو المعالجة. | قد تمنع إتاحة المخزون. | تقود إلى Scrap/Rework/Reject. |
| Inspection | فحص جودة. | يتحقق من قبول المنتج أو العملية. | قد يحجب المخزون. | يحدد قبول أو رفض أو Rework. |
| Quality Check | تحقق جودة. | نقطة رقابة. | يمنع المنتج غير المقبول. | يدعم الامتثال. |
| Lot / Serial Inspection | فحص مرتبط بدفعة أو سيريال. | يدعم تتبع العيوب. | قد يحجب Lot/Serial معين. | مهم للامتثال والاستدعاء. |
| Output Yield | نسبة مخرجات العملية. | تحسب المتوقع من الجيد والهالك. | يؤثر على كمية الإكمال. | يؤدي إلى Yield Variance عند اختلاف الفعلي. |

## 10. مصطلحات Cost Accounting والتكلفة

| المصطلح | الشرح | متى يظهر؟ | أثره على WIP/التكلفة | أثره المحاسبي |
|---|---|---|---|---|
| Cost Accounting | طبقة تسعير وتحليل حركات الإنتاج والمخزون. | بعد نقل الحركات إلى Costing. | تحسب WIP وCost Distributions. | تمهد للقيود عبر Subledger Accounting. |
| Cost Management | منظومة إعداد وإدارة التكلفة. | عند تعريف Cost Profiles وBooks وMethods. | تضبط طريقة حساب التكلفة. | تؤثر على التقارير والقيود. |
| Cost Processor | معالج التكلفة الذي يسعر الحركات. | بعد Transfer to Costing. | يحسب التكلفة والانحرافات والتعديلات. | ينتج Cost Distributions. |
| Cost Distribution | توزيع تكلفة لحركة إنتاج/مخزون. | بعد تسعير الحركة. | يبين عناصر التكلفة. | مصدر Subledger Accounting. |
| Cost Accounting Event | حدث تكلفة ناتج عن حركة. | عند Material Issue أو Completion أو Close وغيرها. | ينقل أثر التشغيل للتكلفة. | يتحول إلى توزيعات ثم قيود. |
| Inventory Transaction Costing | تسعير حركة المخزون. | مع حركات Issue/Return/Completion/Scrap. | يحدد قيمة الحركة. | يدعم مطابقة المخزون والمالية. |
| WIP | Work in Process؛ قيمة تحت التشغيل. | بعد صرف مواد أو موارد وقبل الإغلاق. | يتراكم حتى Completion/Close. | أصل في الميزانية يحتاج مطابقة. |
| Work in Process | الاسم الكامل لـ WIP. | طوال دورة الأمر المفتوح. | يجمع مواد وموارد وأوفرهيد. | يجب تسويته عند Close. |
| Material Cost | تكلفة المواد. | عند Material Issue أو Rollup. | تزيد WIP. | تخفض Raw Materials وتزيد WIP. |
| Resource Cost | تكلفة العمالة أو المعدات. | عند Resource Charging. | تزيد WIP. | تمثل امتصاص تكلفة الموارد. |
| Overhead Cost | تكلفة غير مباشرة. | عند تطبيق Overhead Rules. | تزيد تكلفة المنتج أو WIP. | تمثل امتصاص مصاريف عامة. |
| Standard Cost | تكلفة معيارية مسبقة. | في منهج Standard Cost. | الفروق تظهر Variances. | يساعد على رقابة الانحرافات. |
| Average Cost | تكلفة متوسطية. | Perpetual/Periodic Average. | التكلفة تتحدث بالحركات أو الفترة. | فروقها تظهر كتعديلات لا كـ Standard Variances بنفس المنطق. |
| Actual Cost | تكلفة فعلية حسب طبقات/حركات. | عند استخدام Actual Cost. | تعكس تكلفة الواقع. | قد تولد Product Cost Adjustments عند Close. |
| Cost Organization | كيان محاسبة تكلفة يضم منظمات مخزون. | في إعدادات Cost Accounting. | يحدد نطاق التكلفة. | أساس Cost Book وValuation. |
| Cost Book | سجل تكلفة مالي. | في Cost Accounting. | يمكن أن يكون Primary أو Secondary. | يدعم أغراض مالية/قانونية متعددة. |
| Cost Element | عنصر تكلفة مثل Material أو Resource أو Overhead. | في التحليل والتوزيع. | يصنف تكلفة المنتج. | يظهر في التوزيعات والتقارير. |
| Cost Profile | ملف يحدد Cost Method وValuation وProvisional Completion وScrap Policy. | لكل صنف/مجموعة. | يحكم تسعير الحركات. | من أخطر إعدادات الربط المالي. |
| Cost Method | منهج التكلفة مثل Standard أو Actual أو Average. | عند إعداد Cost Profile. | يحدد طريقة حساب المنتج وWIP. | يحدد نوع الانحرافات/التعديلات. |
| Valuation Unit | مستوى تقييم التكلفة. | ضمن Cost Profile/Valuation Structure. | يحدد هل التكلفة حسب item أو lot أو subinventory. | يؤثر على دقة التقييم. |
| Cost Scenario | حاوية لأسعار الموارد والـ Overhead للـ Standard Cost. | قبل نشر التكاليف. | يحدد Resource Rates وOverheads. | أساس Cost Rollup. |
| Cost Rollup | تجميع تكلفة المنتج من المواد والموارد والأوفرهيد. | عند حساب Standard/Planned Cost. | ينتج تكلفة المنتج. | يعتمد على Work Definition. |
| Work Order Cost | تكلفة أمر العمل. | أثناء التنفيذ وعند Close. | تجمع المواد والموارد والهالك. | أساس تحليل الربحية والانحرافات. |
| Estimated Cost | تكلفة تقديرية. | عند الإكمال المؤقت أو التخطيط. | تستخدم مؤقتًا قبل التكلفة الفعلية. | تُعدل عند Close حسب المنهج. |
| Actual Work Order Cost | التكلفة الفعلية لأمر العمل. | عند الإغلاق. | تحسم تكلفة الأمر. | تظهر في Adjustments أو التقارير. |
| Product Completion Cost | تكلفة المنتج عند الإكمال. | عند Product Completion. | مؤقتة غالبًا. | تتحول أو تعدل عند Close. |
| Scrap Cost | تكلفة الهالك. | عند Scrap Transaction/Close. | قد تحمل على المنتج أو المصروف. | حسب Operation Scrap Accounting. |
| WIP Adjustment | تعديل رصيد WIP. | عند الإغلاق أو تغير التكلفة. | يصحح WIP. | يظهر كCost Adjustment. |
| Period Close | إغلاق الفترة المحاسبية. | نهاية الفترة. | يتحقق من أوامر غير مغلقة أو حركات معلقة. | يمنع ترحيل تكلفة بفترة خاطئة. |
| Accounting Period | الفترة المحاسبية. | في القيود والتقارير. | تحدد توقيت التكلفة. | عدم إغلاق الأوامر يسبب فروقات زمنية. |
| Subledger Accounting | طبقة إنشاء القيود من Cost Distributions. | بعد Cost Accounting. | لا تؤثر تشغيليًا. | تولد قيود GL حسب قواعد. |
| General Ledger Transfer | نقل القيود إلى الأستاذ العام. | بعد SLA. | لا يغير WIP تشغيليًا. | يثبت الأثر المالي النهائي. |

## 11. مصطلحات Variances والانحرافات

| المصطلح | الشرح | سبب ظهوره | ماذا يكشف؟ | من يراجعه؟ |
|---|---|---|---|---|
| Variance | فرق بين المتوقع/المعياري والفعلي. | اختلاف سعر أو كمية أو مورد أو عائد. | جودة التعريف والتنفيذ والتكلفة. | محاسب التكلفة ومدير الإنتاج. |
| Material Rate Variance | فرق سعر المادة. | السعر في Rollup لا يساوي سعر حركة الصرف. | مشكلة أسعار أو Cost Rollup. | محاسب التكلفة. |
| Material Usage Variance | فرق كمية المادة. | الكمية الفعلية لا تساوي Work Definition. | BOM/Work Definition أو استهلاك زائد/ناقص. | الإنتاج والمخزون والتكلفة. |
| Material Substitution Variance | فرق بسبب مادة بديلة أو غير موجودة في Work Definition. | صرف صنف خارج التعريف أو عدم صرف صنف مطلوب. | استبدال غير مضبوط أو تعريف غير واقعي. | الإنتاج والتكلفة. |
| Resource Rate Variance | فرق معدل المورد. | معدل Rollup لا يساوي معدل حركة المورد. | خطأ في Resource Rate أو تغير تكلفة. | محاسب التكلفة. |
| Resource Usage Variance | فرق استخدام المورد. | الاستخدام الفعلي لا يساوي Work Definition. | كفاءة تشغيل أو Route غير دقيق. | الإنتاج والتكلفة. |
| Yield Variance | فرق بسبب العائد/الفاقد. | الإنتاج الفعلي يختلف عن المتوقع حسب Operation Yield. | مشكلة جودة أو Yield غير واقعي. | الإنتاج والجودة والتكلفة. |
| Job Close Variance | فرق متبقٍ عند إغلاق الأمر. | فروقات غير مصنفة أو Standard Cost محددة يدويًا. | رصيد أو فرق يحتاج تفسيرًا. | محاسب التكلفة. |
| Scrap Variance | فرق متعلق بالهالك. | اختلاف الهالك الفعلي عن المتوقع أو إعادة حساب Scrap. | ضعف سياسة الهالك أو جودة الإنتاج. | الجودة والتكلفة. |
| Standard Cost Variance | انحراف في منهج Standard Cost. | الفرق بين Standard والفعلي. | كفاءة العمليات ودقة التكاليف. | المالية والتكلفة. |
| Actual vs Standard | مقارنة الفعلي بالمعياري. | عند تحليل الأداء. | أين انحرفت التكلفة. | الإدارة والتكلفة. |
| Target vs Actual | مقارنة الهدف بالفعلي. | عند مراقبة الأداء. | يحتاج تحقق إضافي من المصدر الأصلي إذا كان تقريرًا محددًا. | الإدارة. |

## 16. خاتمة قصيرة

فهم مصطلحات Oracle Manufacturing ضروري قبل دراسة دورة العمل والتكلفة؛ لأن Oracle يبني Module الإنتاج على فصل واضح بين التصميم، التنفيذ، المخزون، التكلفة، والمحاسبة.

المصطلحات المركزية مثل **Work Definition** و **Work Order** و **Material Issue** و **Product Completion** و **WIP** و **Cost Distribution** و **Work Order Close** ليست أسماء شاشات، بل عقد تشغيلية ومالية مترابطة.

أي مقارنة لاحقة مع نظام ERP محلي مثل ناتج يجب أن تبدأ من هذه المصطلحات: ما الذي يدعمه النظام؟ ما البديل؟ وما أثر غياب المفهوم على التشغيل والمخزون والتكلفة؟
