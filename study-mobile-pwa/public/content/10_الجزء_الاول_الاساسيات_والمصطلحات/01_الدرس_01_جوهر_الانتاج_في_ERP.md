# الدرس 01: جوهر الإنتاج في ERP

## لماذا هذا الدرس مهم

أكبر خطأ يقع فيه المبتدئ في موديول الإنتاج Production Module هو أنه يتخيل أن الإنتاج يساوي:

منتج + كمية + أمر تصنيع.

هذا فهم ناقص.

في الواقع، الإنتاج داخل نظام ERP هو Business System وليس مجرد شاشة Transaction Screen.

هو منظومة تربط بين:

- البيانات الأساسية Master Data
- هيكل المنتج Product Structure
- طريقة التصنيع Production Method
- أمر الإنتاج Production Order أو Work Order
- الحركات المخزنية Inventory Transactions
- التكلفة تحت التشغيل Work in Process (WIP)
- التكلفة الفعلية والانحرافات Actual Cost and Variances
- الإغلاق والرقابة Closing and Control

إذا فهمت هذا الدرس جيدًا، فستتغير طريقة قراءتك لأي نظام ERP محلي أو عالمي.

## الهدف من الدرس

بعد إنهاء هذا الدرس يجب أن تكون قادرًا على:

1. شرح معنى موديول الإنتاج Production Module بطريقة صحيحة.
2. فهم لماذا لا يكفي وجود شاشة أمر إنتاج فقط.
3. ربط الإنتاج بالمخزون Inventory والتكلفة Costing والرقابة Control.
4. تكوين نظرة أولية قوية عند تحليل أي نظام تصنيع محلي.

## الفكرة الكبرى

الإنتاج في ERP ليس Event واحدًا، بل Cycle كاملة.

الصورة الصحيحة هي:

Master Data
→ Product Structure
→ Production Method
→ Production Order / Work Order
→ Material Issue / Goods Issue
→ Operation Execution / Reporting
→ Finished Goods Receipt / Product Completion
→ WIP and Cost
→ Close and Variance
→ Reporting and Control

هذه السلسلة هي العمود الفقري لأي نظام إنتاج محترم، حتى لو اختلفت أسماء الكيانات بين Dynamics 365 وOracle Fusion Cloud وSAP S/4HANA.

## الركائز الستة لأي موديول إنتاج قوي

### 1. البيانات الأساسية Master Data

هذه هي القاعدة التي يقف عليها كل شيء.

وتشمل عادةً:

- المنتج النهائي Finished Good
- المواد الخام Raw Materials
- المنتجات نصف المصنعة Semi-finished Goods / Subassemblies
- وحدات القياس Units of Measure (UoM)
- المواقع والمستودعات Sites / Warehouses / Locations
- التتبع Tracking مثل Lot / Batch / Serial

إذا كانت Master Data ضعيفة، فكل ما بعدها سيتشوه:

- التخطيط Planning
- الصرف Consumption
- التكلفة Costing
- التقارير Reports

### 2. هيكل المنتج Product Structure

هذا هو منطق: ماذا يدخل في تصنيع المنتج؟

في الأنظمة الثلاثة قد يظهر عبر:

- BOM in Dynamics 365
- Item Structure or BOM in Oracle
- BOM in SAP

هذا المحور يحدد:

- المكونات Components
- الكميات Quantities
- وحدات القياس UoM
- الهالك Scrap
- المنتجات النصف مصنعة Subassemblies
- أحيانًا المنتجات الجانبية By-products أو المنتجات المشتركة Co-products

إذا كانت BOM أو Item Structure خاطئة، فثلاثة أشياء تتعطل فورًا:

1. التخطيط Planning
2. الصرف Material Issue
3. التكلفة Material Cost

### 3. طريقة التصنيع Production Method

هذا هو منطق: كيف يُصنع المنتج؟

ويظهر غالبًا عبر:

- Route / Routing / Operations / Resources
- Work Definition in Oracle

هذا المحور يشرح:

- ما الخطوات Steps / Operations
- أين تتم Where
- بأي مورد Resource
- بأي زمن Time
- وبأي تكلفة تشغيل Operation Cost

إذا غاب هذا المحور، يتحول النظام إلى نظام خصم مواد وإضافة منتج فقط، دون فهم حقيقي للتشغيل Manufacturing Execution.

### 4. أمر الإنتاج Production Order أو Work Order

هذا هو الكائن التنفيذي المركزي.

الأمر لا يمثل طلب تصنيع فقط، بل يجمع:

- المنتج Product
- الكمية Quantity
- المكونات Components
- العمليات Operations
- الموارد Resources
- الحركات Transactions
- الحالات States / Statuses
- التكلفة Cost

أمر الإنتاج هو مركز الدورة كلها.

### 5. الحركات التنفيذية الفعلية Execution Transactions

أي نظام إنتاج قوي يجب أن يدعم على الأقل:

- صرف المواد Material Issue / Goods Issue
- تسجيل التشغيل Operation Reporting / Resource Charging / Confirmation
- استلام المنتج النهائي Finished Goods Receipt / Product Completion / Report as Finished
- معالجة الهالك Scrap
- الإرجاع Returns عند الحاجة

السؤال الحاسم هنا ليس: هل توجد الشاشة؟

السؤال الحاسم هو:

- هل الحركة تغير المخزون؟
- هل تؤثر على WIP؟
- هل تنعكس على التكلفة؟
- هل تترك أثرًا رقابيًا واضحًا؟

### 6. التكلفة والإغلاق والرقابة Costing, Closing, and Control

هنا يبدأ الفرق الحقيقي بين النظام القوي والنظام الضعيف.

لا يكفي أن تنتج.

بل يجب أن تعرف:

- ما التكلفة المقدرة Estimated Cost؟
- ما التكلفة الفعلية Actual Cost؟
- ما قيمة WIP؟
- ما سبب الانحراف Variance؟
- هل الأمر Closed فعلًا أم ما زال Open؟

الأنظمة الكبيرة لا تعتبر أمر الإنتاج منتهيًا عند اكتمال التشغيل فقط.

بل تعتبره منتهيًا عندما يصبح مفهوم التكلفة واضحًا ويغلق الأمر بالشكل الصحيح.

## كيف يظهر هذا الجوهر في الأنظمة الثلاثة

| المحور العام | Dynamics 365 | Oracle Fusion Cloud | SAP S/4HANA |
| --- | --- | --- | --- |
| البيانات الأساسية | Released Products, Item Groups, Dimensions | Items, Inventory Organizations, Plant Parameters | Material Master, Plant, Storage Location |
| هيكل المنتج | BOM / Formula | Item Structure / Work Definition inputs | BOM |
| طريقة التصنيع | Route / Operations / Resources | Work Definition / Operations / Resources | Routing / Work Center / Activity Type |
| الكيان التنفيذي | Production Order | Work Order | Production Order / Process Order |
| استلام الناتج | Report as Finished | Product Completion | Goods Receipt |
| الإغلاق | Ended / Closed | Work Order Close / Closed | TECO / Settlement / CLSD بحسب السياق |

هذه الأسماء تختلف، لكن المنطق الأساسي متقارب جدًا.

## منظور المستشار Functional Consultant View

المستشار الضعيف يسأل:

- هل توجد شاشة أمر إنتاج؟
- هل يمكن طباعة أمر؟
- هل يمكن خصم المواد؟

المستشار القوي يسأل:

- من أين جاءت هذه المكونات Components؟
- هل BOM معتمدة وصحيحة؟
- هل توجد مراحل تصنيع Operations حقيقية؟
- متى يرتفع WIP؟
- متى ينخفض WIP؟
- متى يصبح المنتج النهائي متاحًا رسميًا؟
- هل الإغلاق تشغيلي فقط أم تشغيلي ومالي؟
- ما التقارير التي تراقب ذلك؟

## كيف يبدو النظام المحلي الضعيف غالبًا

غالبًا تظهر بعض هذه العلامات:

- أمر إنتاج بدون حالات واضحة Statuses
- BOM ضعيفة أو غير معتمدة
- لا يوجد Route أو Work Definition أو منطق مراحل واضح
- صرف المواد لا ينعكس على WIP بشكل سليم
- استلام المنتج النهائي يتم دون منطق جودة Quality أو تكلفة Cost
- الأوامر تبقى مفتوحة Open Orders لفترات طويلة
- لا توجد تقارير Variance أو WIP مفيدة
- الربط بين التشغيل والمالية ضعيف أو غير واضح

## الأخطاء الشائعة Common Mistakes

1. الاعتقاد أن وجود شاشة أمر إنتاج يعني وجود Module إنتاج محترم.
2. الخلط بين Product Structure وProduction Method.
3. التفكير أن نهاية التشغيل تعني نهاية كل الأثر المالي.
4. التعامل مع WIP كرقم محاسبي فقط، لا كأثر تشغيلي أيضًا.
5. حفظ المصطلحات الإنجليزية دون فهمها داخل الدورة.

## المراجعة السريعة Quick Review

- موديول الإنتاج Production Module هو دورة مترابطة لا مجرد أمر.
- أي نظام قوي يحتاج Master Data وProduct Structure وProduction Method وOrder وExecution وCosting.
- BOM أو ما يعادلها تجيب عن سؤال: ماذا يدخل في المنتج؟
- Route أو Work Definition تجيب عن سؤال: كيف يُصنع المنتج؟
- WIP هو قيمة ما دخل في الإنتاج ولم يغلق بعد.
- الإغلاق Closing هو نقطة حاسمة، وليس خطوة شكلية.
