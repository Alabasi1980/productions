# 05_Production_Implementation_Checklist.md
# قائمة تطبيق وتسليم نظام الإنتاج في Microsoft Dynamics 365

## 1. مقدمة

هذا الملف يحوّل دراسة إدارة الإنتاج في Microsoft Dynamics 365 Supply Chain Management إلى قائمة تنفيذية تساعد فريق ERP على تطبيق وتسليم Module الإنتاج بطريقة منظمة، من مرحلة جاهزية العميل وحتى ما بعد التشغيل.

الهدف من هذه الـ Checklist هو تقليل مخاطر التطبيق المرتبطة بـ:

- ضعف بيانات المنتجات.
- BOM غير دقيقة أو غير معتمدة.
- Routes غير واضحة أو غير قابلة للجدولة.
- تكلفة إنتاج غير مضبوطة.
- WIP غير مفهوم أو غير مطابق ماليًا.
- أوامر إنتاج لا تُغلق.
- تقارير غير كافية.
- فشل UAT أو Go-Live.

> ملاحظة تنفيذية: تطبيق الإنتاج لا ينجح بمجرد إنشاء شاشة Production Order. النجاح يعتمد على تكامل Master Data وBOM وRoute والمخزون والتكلفة والمالية والتقارير والإغلاق.

## 2. طريقة استخدام هذا الملف

يُستخدم هذا الملف كأداة رقابة تنفيذية في كل مرحلة من مراحل تطبيق Module الإنتاج. لا يُستخدم كقائمة قراءة فقط، بل كقائمة قرار: هل هذا البند جاهز؟ من اعتمده؟ ما الخطر إن لم يكن جاهزًا؟

| المرحلة | الهدف من استخدام الـ Checklist | الأطراف المطلوبة |
|---|---|---|
| قبل بدء التطبيق | قياس جاهزية العميل وفهم إن كان لديه دورة إنتاج قابلة للتطبيق داخل ERP | الإدارة، الإنتاج، المستودعات، المالية |
| أثناء تحليل العميل | تحويل الواقع التشغيلي إلى متطلبات وإعدادات واضحة | Functional Consultant، Business Analyst، الإنتاج، المالية |
| أثناء إعداد النظام | التأكد من ضبط البيانات والإعدادات المؤثرة على الإنتاج والمخزون والتكلفة | فريق التطبيق، Master Data، المالية |
| أثناء الاختبار | اختبار دورة إنتاج كاملة من المنتج إلى الإغلاق | الإنتاج، المستودعات، الجودة، المالية |
| قبل Go-Live | اتخاذ قرار Go/No-Go بناءً على جاهزية فعلية لا افتراضات | مدير المشروع، المستخدمون الرئيسيون، المالية |
| بعد Go-Live | مراقبة أول أوامر إنتاج وتصحيح أخطاء البيانات أو الاستخدام | الدعم، الإنتاج، المالية، المستودعات |

## 3. Checklist جاهزية العميل قبل تطبيق الإنتاج

| بند التحقق | لماذا مهم؟ | المسؤول | الخطر إذا لم يتحقق |
|---|---|---|---|
| هل لدى العميل دورة إنتاج موثقة؟ | لأن Dynamics يعامل الإنتاج كدورة مترابطة تمر بحالات من Created إلى Ended | الإنتاج / التحليل | بناء النظام على فهم شفهي غير مستقر |
| هل المنتجات النهائية معروفة؟ | المنتج النهائي هو مخرج أمر الإنتاج ويجب أن يكون Released Product | Master Data / الإنتاج | أوامر إنتاج بلا مخرجات واضحة |
| هل المواد الخام معروفة؟ | المواد الخام هي مدخلات BOM والصرف | المستودعات / الإنتاج | صرف خاطئ وتكلفة غير صحيحة |
| هل توجد منتجات نصف مصنعة؟ | لأنها قد تكون Output لأمر وInput لأمر آخر أو Phantom أو Pegged Supply | الإنتاج / التخطيط | تفكك غير صحيح لـ BOM أو أوامر فرعية ناقصة |
| هل توجد BOM موثقة؟ | BOM تحدد المواد والكميات والهالك وتؤثر على الصرف والتكلفة | الهندسة / الإنتاج | تكلفة ناقصة أو صرف غير مطابق |
| هل توجد مراحل إنتاج Route؟ | Route يحدد العمليات والموارد والأوقات والتكلفة | الإنتاج | فشل الجدولة وعدم احتساب تكلفة التشغيل |
| هل المستودعات واضحة؟ | Site/Warehouse/Location تؤثر على الحركات المخزنية والتكلفة | المستودعات | خلط بين مواد خام ومنتج تام أو مواقع خاطئة |
| هل طريقة التكلفة معروفة؟ | النظام يدعم Standard Cost وActual Cost، ولكل منهما أثر على المخزون والانحرافات | المالية | تقارير تكلفة غير قابلة للاعتماد |
| هل المالية مشاركة في التحليل؟ | لأن الإنتاج يولد WIP وPosting وVariances | المالية | فجوة بين التشغيل والدفتر العام |
| هل توجد سياسة هالك؟ | Scrap يؤثر على المواد المطلوبة والتكلفة والانحرافات | الإنتاج / الجودة / المالية | انحرافات متكررة وتكلفة غير واقعية |
| هل توجد سياسة إغلاق أوامر الإنتاج؟ | الإغلاق يحول التكلفة من تقديرية إلى فعلية ويحسب الانحرافات | الإنتاج / المالية | WIP متضخم وأوامر مفتوحة تشوه المالية |

| Finished Goods | تحديد المنتج النهائي وموقع استلامه ومجموعة الصنف | استلام منتج في مخزون أو حساب خاطئ | الإنتاج / المستودعات |
| Item Groups | التأكد أنها تعكس التصنيف المالي الصحيح للأصناف | حركات الإنتاج تذهب لحسابات محاسبية خاطئة | المالية / Master Data |
| Item Model Groups | التأكد من طريقة تقييم المخزون والتحديثات Physical/Financial | عدم تسجيل حركات أو عدم احتساب تكلفة | المالية / المخزون |
| Units of Measure | التأكد من وحدات المخزون والشراء والإنتاج والتحويل بينها | صرف بالمضاعفات أو الكسور وتكلفة غير منطقية | Master Data |
| Tracking Dimensions | تفعيل Batch/Serial عند الحاجة | فقدان تتبع الدفعات أو السيريال عند الصرف والاستلام | الجودة / المستودعات |
| Storage Dimensions | ضبط Site/Warehouse/Location | حركات مخزون في مواقع غير صحيحة | المستودعات |
| Site | ربط الإنتاج بالموقع الصحيح | BOM/Route/Cost قد تختلف حسب الموقع | الإنتاج / المالية |
| Warehouse | تحديد مستودعات المواد والمنتج التام | خلط بين المخزون الخام والنهائي | المستودعات |
| Location | تحديد المواقع الداخلية عند الحاجة | صعوبة تنفيذ السحب والوضع وتتبع الحركة | المستودعات |

### أخطاء BOM التي يجب منعها قبل Go-Live

| الخطأ | الأثر التشغيلي | الأثر المالي |
|---|---|---|
| نسيان مكوّن في BOM | لا يُصرف من المخزون وقد يظهر نقص فعلي | تكلفة المنتج أقل من الواقع |
| تضمين مكوّن غير مطلوب | صرف مواد زائدة | تكلفة المنتج أعلى من الواقع |
| كمية BOM خاطئة | نقص أو فائض في الصرف | Variances كبيرة عند الإغلاق |
| وحدة قياس خاطئة | صرف مضاعف أو ناقص | تكلفة غير منطقية |
| نسيان الهالك | عجز متكرر في المواد | انحرافات سالبة متكررة |
| BOM غير معتمدة | توقف استخدام القائمة | تأخير في الإنتاج والتخطيط |
| BOM غير مفعلة | النظام لا يختارها تلقائيًا | اعتماد يدوي يعرض العملية للخطأ |

## 6. Checklist إعداد Routes / Operations / Resources

| بند التحقق | الهدف | أثره على الجدولة | أثره على التكلفة |
|---|---|---|---|
| Route موجود لكل منتج يحتاج مراحل تصنيع | تعريف كيف يُصنع المنتج | بدون Route لا توجد جدولة عمليات | لا تُحسب تكلفة التشغيل |
| Route Version صحيحة | ربط Route بالمنتج والموقع والكمية والفترة | اختيار مسار غير مناسب | تكلفة معيارية خاطئة |
| Operations معرفة | تحديد خطوات التصنيع | تحديد بداية ونهاية كل عملية | احتساب تكلفة كل عملية |
| ترتيب العمليات صحيح | عكس التسلسل الفعلي | منع جدولة غير واقعية | توزيع وقت وتكلفة صحيح |
| العمليات التسلسلية أو المتوازية واضحة | تحديد هل المسار Simple أو Network | حجز موارد متزامنة أو متتابعة | احتساب صحيح للوقت |
| Resources / Work Centers معرفة | تحديد الآلات أو العمال أو الأدوات أو المواقع | حجز طاقة الموارد | احتساب تكلفة الموارد |
| Resource Groups معرفة | تجميع الموارد لأغراض الطاقة | رؤية الطاقة المتاحة | تحليل تكلفة حسب مجموعة |
| Calendars معرفة | تحديد أيام وساعات العمل | منع فشل الجدولة بسبب طاقة غير معرفة | يؤثر على وقت العملية |
| Setup Time معرف | وقت إعداد الآلة أو العملية | يؤثر على تاريخ البداية والنهاية | تكلفة ثابتة توزع على الكمية |
| Run Time معرف | وقت التشغيل لكل وحدة | يؤثر على مدة الإنتاج | تكلفة متغيرة حسب الكمية |
| Queue / Wait / Transport Time إن كان مستخدمًا | تمثيل الانتظار والنقل | جدولة أكثر واقعية | يحتاج تحقق إضافي من المصدر الأصلي |
| Cost Categories معرفة | تحديد سعر الإعداد والتشغيل | لا يؤثر مباشرة إلا عبر العمليات | أساس احتساب تكلفة Route |
| Route Group صحيح | تحديد ما يُحتسب ماليًا من الوقت والكمية | يضبط علاقة العملية بالترحيل | إن أُغلق الخيار لا يولد Job/Route card قيدًا |
| الموارد التي تدخل في التكلفة محددة | التمييز بين موارد تشغيلية ومالية | قد تُجدول دون تكلفة أو العكس | منع تكلفة ناقصة |
| الموارد التي تدخل في الجدولة محددة | تحديد الموارد التي تحجز طاقة | منع تضارب الطاقة | أثر غير مباشر على التكلفة |

## 7. Checklist إعداد المخزون والمستودعات للإنتاج

| بند التحقق | لماذا مهم؟ | الخطر إذا لم يتحقق |
|---|---|---|
| مستودعات المواد الخام واضحة | لأن Picking List تسحب المواد منها | صرف من مستودع خاطئ |
| مستودع الإنتاج إن وجد واضح | يساعد على فصل مواد الإنتاج عن باقي المخزون | عدم وضوح المواد قيد التشغيل |
| مستودع المنتج التام واضح | RAF يدخل المنتج النهائي إلى المخزون | استلام منتج في موقع خاطئ |
| Locations واضحة | تؤثر على حركة السحب والوضع | صعوبة تنفيذ المستودع للحركة |
| طريقة صرف المواد محددة | يدوي أو تلقائي أو Backflush | صرف غير مراقب أو تأخير في الإنتاج |
| طريقة استلام المنتج النهائي محددة | عبر Report as Finished | عدم تحديث المنتج النهائي في المخزون |
| Batch / Serial Tracking مفعل عند الحاجة | لضبط الدفعات والسيريالات في الصرف والاستلام | فقدان التتبع |
| المرتجعات من الإنتاج واضحة | يمكن تسجيلها بكميات سالبة في Picking List | بقاء WIP أو مخزون غير صحيح |
| Warehouse Work أو Picking Work واضح إن ورد | Released قد يولد عمل مستودعي عند تفعيل Warehouse management | المستودع لا يعرف ما يجب سحبه |
| أثر الإنتاج على On-hand inventory مفهوم | الصرف يخفض المواد وRAF يزيد المنتج التام | فرق بين الواقع والنظام |

## 8. Checklist إعداد أوامر الإنتاج Production Orders

| المرحلة | ما يجب ضبطه؟ | المسؤول | الملاحظات |
|---|---|---|---|
| Created | طريقة إنشاء الأمر: يدوي، Master Planning، Sales Order، أمر إنتاج أعلى | الإنتاج / التخطيط | لا يوجد أثر مخزني أو مالي في هذه المرحلة |
| Created | صلاحية إنشاء الأمر | إدارة النظام | منع أوامر غير معتمدة |
| Estimated | صلاحية التقدير | الإنتاج / المالية | يحسب Estimated Cost ويولد حركات On order |
| Scheduled | صلاحية الجدولة | التخطيط | تعتمد على Route والموارد والتقويم |
| Released | صلاحية Release | الإنتاج / المستودعات | يجعل الأمر متاحًا للصالة والمستودع |
| Started | صلاحية البدء | الإنتاج | يسمح بتسجيل الصرف والوقت |
| Report as Finished | صلاحية RAF | الإنتاج / الجودة / المستودعات | يدخل المنتج النهائي إلى المخزون |
| Ended | صلاحية Close | المالية / الإنتاج | يحسم التكلفة الفعلية والانحرافات |
| تعديل الأمر بعد الإنشاء | تحديد متى يسمح بالتعديل | الإنتاج / المالية | منع تغيير غير مضبوط لـ BOM/Route |
| تعديل BOM أو Route داخل الأمر | تحديد السياسة | الإنتاج | قد يؤثر على التكلفة والـ Variances |
| سياسة الإلغاء | تحديد متى وكيف يُلغى الأمر | الإنتاج / المالية | يحتاج تحقق إضافي من المصدر الأصلي |

## 9. Checklist Material Issue / Consumption

| بند التحقق | أثره المخزني | أثره على التكلفة | الخطر |
|---|---|---|---|
| Picking List Journal جاهزة | تخفيض مخزون المواد | زيادة WIP بتكلفة المواد | عدم تسجيل الصرف |
| طريقة الصرف محددة: يدوي أو تلقائي | تحديد توقيت خفض المخزون | توقيت ظهور التكلفة | صرف غير مناسب لطبيعة المواد |
| Auto-consumption / Flushing مضبوط | صرف تلقائي حسب الإعداد | يقلل العمل اليدوي | لا يكتشف الفروقات إلا في Variance |
| الصرف من BOM صحيح | ربط الكميات المتوقعة بالمواد | أساس مقارنة الفعلي بالمخطط | صرف مواد غير مطابقة |
| الكميات المصروفة مطابقة أو مراقبة | يعكس الاستهلاك الفعلي | فروقات تظهر عند الإغلاق | Variances غير مفسرة |
| السماح بصرف أكثر أو أقل محدد | يضبط مرونة التشغيل | يؤثر على Production quantity variance | فتح الباب لصرف غير منضبط |
| السماح بالبدائل محدد | يتيح مادة بديلة عند الحاجة | قد يسبب Substitution variance | استخدام بدائل بلا رقابة |
| طريقة إرجاع المواد من الإنتاج واضحة | يعيد المادة للمخزون | يخفض WIP | بقاء تكلفة غير حقيقية في الإنتاج |
| أثر الصرف على WIP مفهوم | WIP يزيد عند الصرف | قيد تقديري حتى الإغلاق | فهم مالي خاطئ |
| أثر الصرف على Variances مفهوم | فرق الكمية/الصنف يظهر لاحقًا | يفسر انحرافات الإنتاج | عدم تحليل أسباب الانحراف |

## 22. الدروس المستفادة لنظام ERP محلي مثل ناتج

يمكن استخدام هذه الـ Checklist لاحقًا عند تقييم أو تطوير Module الإنتاج في نظام ERP محلي مثل ناتج، بشرط عدم نسخ تعقيد Dynamics كاملًا دون حاجة. المطلوب هو تحويل المنطق إلى حد أدنى قابل للتطبيق.

### 22.1 البنود التي يجب أن يدعمها النظام كحد أدنى

| البند | لماذا لا يجوز تجاهله؟ |
|---|---|
| تعريف منتج نهائي ومواد خام | أساس أي دورة إنتاج |
| BOM مرتبطة بالمنتج | تحدد الصرف والتكلفة |
| أمر إنتاج بحالات واضحة | يمنع الفوضى التشغيلية |
| صرف مواد مرتبط بالأمر | يربط الإنتاج بالمخزون |
| استلام منتج نهائي | يحول الإنتاج إلى مخزون |
| إغلاق أمر الإنتاج | يمنع بقاء تكلفة معلقة |
| تقرير تكلفة أمر | يتيح الرقابة |
| تقرير استهلاك مواد | يكشف الفروقات |

### 22.2 البنود المتقدمة التي يمكن تأجيلها

| البند | سبب التأجيل المحتمل |
|---|---|
| Route Network والعمليات المتوازية | يحتاج نضجًا في بيانات العمليات |
| Capabilities بدل الموارد المحددة | مناسب لمنشآت أكبر وأكثر نضجًا |
| Deferred Posting | مفيد للحجم العالي، وليس أولوية أولى |
| Co-products وBy-products | فقط للصناعات العملياتية التي تحتاجها |
| Batch Balancing | تخصصي ويحتاج تحقق إضافي من المصدر الأصلي |
| Cost Groups متقدمة | يمكن البدء بتصنيف أبسط ثم التوسع |

### 22.3 البنود المالية التي لا يجوز تجاهلها

| البند | السبب |
|---|---|
| طريقة التكلفة | بدونها لا معنى لتكلفة المنتج |
| WIP أو بديل محاسبي واضح | يوضح أين تذهب تكلفة المواد قبل الاستلام |
| إغلاق أمر الإنتاج | يحسم التكلفة والانحرافات |
| حسابات المواد والمنتج التام | تمنع ترحيل خاطئ |
| Variances | تكشف أخطاء BOM وRoute والصرف |
| مشاركة المالية في UAT | تمنع نجاحًا تشغيليًا وفشلًا ماليًا |

### 22.4 البنود التي يجب أن تكون جزءًا من UAT

| البند | سبب الاختبار |
|---|---|
| دورة كاملة من BOM إلى Close | اختبار تكامل النظام |
| صرف مواد مطابق وغير مطابق | اختبار Variances |
| إنتاج جزئي | تمثيل الواقع |
| هالك أو Error Quantity | اختبار Scrap Method |
| Report as Finished | اختبار المخزون النهائي |
| WIP | اختبار الربط المالي |
| تقارير التكلفة | اختبار الرقابة |
| القيود أو المنطق المالي | اعتماد المالية |

### 22.5 كيف تساعد هذه القائمة في تسليم العملاء بشكل أفضل؟

- تجعل التسليم مبنيًا على بنود قابلة للتحقق لا انطباعات.
- تكشف مبكرًا إن كان العميل غير جاهز لتكلفة إنتاج دقيقة.
- تمنع تشغيل الإنتاج قبل ضبط BOM وRoute.
- تربط الإنتاج بالمستودعات والمالية.
- تحول UAT من تجربة شاشة إلى اختبار دورة مؤسسية كاملة.
- تعطي قرار Go/No-Go مبنيًا على مخاطر واضحة.

## 23. ملخص تنفيذي

تطبيق الإنتاج لا ينجح بمجرد تشغيل شاشة Production Order. شاشة الأمر ليست سوى نقطة داخل دورة أكبر تبدأ من Master Data وتنتهي بالإغلاق المالي وتحليل الانحرافات.

نجاح Module الإنتاج يعتمد على:

- دقة Released Products وItem Groups وItem Model Groups ووحدات القياس.
- BOM معتمدة ومفعلة وصحيحة من حيث المواد والكميات والهالك.
- Route واضحة ومربوطة بعمليات وموارد وأوقات وتكلفة.
- سياسة صرف مواد واضحة، سواء يدوية أو تلقائية.
- Report as Finished مضبوط لاستلام المنتج النهائي.
- WIP مفهوم ومقبول من المالية.
- Costing Version وStandard/Actual Cost محددة بوضوح.
- Production Posting وقيود الإنتاج مفهومة.
- تقارير تكلفة وانحرافات جاهزة.
- UAT يغطي دورة إنتاج كاملة من المنتج إلى الإغلاق.
- Go-Live لا يتم قبل اعتماد الإنتاج والمستودعات والمالية.

هذه الـ Checklist هي أداة تنفيذ ورقابة، وليست ملخصًا نظريًا. قيمتها الحقيقية تظهر عندما تُستخدم قبل التطبيق، أثناء الإعداد، في UAT، وقبل قرار Go/No-Go.
