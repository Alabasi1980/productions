# مصفوفة المفاهيم الأساسية بين الأنظمة

## الهدف من هذا الملف

هذا الملف هو Cross-System Concept Matrix.

الغرض منه أن يعطيك ترجمة ذهنية سريعة بين:

- المفهوم العام Generic Concept
- لغة Dynamics 365
- لغة Oracle Fusion Cloud
- لغة SAP S/4HANA

استخدمه عند الدراسة وعند المراجعة وعند تحليل أي نظام محلي.

## قاعدة القراءة

لا تقرأ العمود على أنه ترجمة حرفية دائمًا.

بل اسأل:

1. ما المفهوم العام؟
2. ما الاسم في كل نظام؟
3. هل الفكرة واحدة تمامًا أم أن هناك فرقًا في العمق أو المنطق؟

## المصفوفة الأساسية

| --- | --- | --- | --- | --- |
| البيانات الأساسية | Master Data / Released Products | Items / Plant Data | Material Master | كلها تمثل الأساس الذي تبنى عليه الدورة |
| المنتج النهائي | Finished Good | Finished Good / Output Item | Finished Product / Material | اختلافات لفظية أكثر من كونها جذرية |

| قائمة المواد | BOM / Formula | Item Structure / BOM | BOM | Oracle يفصل بين Item Structure وWork Definition |
| طريقة التصنيع | Route | Work Definition | Routing | Oracle يميل لدمج عناصر أكثر داخل Work Definition |
| العملية | Operation | Operation | Operation | مفهوم مشترك بوضوح |
| المورد | Resource | Resource | Activity Type / Resource logic / Work Center context | SAP أعمق في ربط المورد بالتكلفة |
| مركز العمل | Resource Group / Work Center context | Work Center | Work Center | اللفظ متقارب لكن التكامل مع التكلفة يختلف |
| أمر الإنتاج | Production Order | Work Order | Production Order / Process Order | Oracle يفضل Work Order، SAP يفرق بين Discrete وProcess |
| إنشاء الأمر | Created | Unreleased | Created / CRTD | بداية وجود الأمر لا تعني بدء الحركات |
| إطلاق الأمر | Released | Released | Released / REL | السماح بالتنفيذ يبدأ هنا غالبًا |

| تكلفة مقدرة | Estimated Cost | Estimated / preliminary order cost logic | Planned / target cost context | ليست دائمًا بنفس الآلية لكنها فكرة مشتركة |

| تكلفة فعلية | Actual Cost | Actual Work Order Cost | Actual Cost | لا تظهر قيمتها النهائية دائمًا قبل الإغلاق |

| انحراف | Variance | Variance | Variance | الاختلاف في الأنواع والعمق، لا في أصل الفكرة |
| إغلاق الأمر | End / Close | Work Order Close | TECO / Settlement / CLSD بحسب السياق | SAP يوزع الإغلاق على أكثر من مفهوم وحالة |
| إغلاق الفترة | Period Close | Period Close / GL Transfer readiness | Period-End Closing | نقطة رقابية ومالية حساسة جدًا |
| توزيع التكلفة | Posting / cost logic | Cost Distribution | Settlement / cost postings | Oracle يبرز Cost Distribution كمفهوم مستقل بقوة |
| مقارنة النظام الحالي بالمرجع | Gap Analysis mindset | Gap Analysis mindset | Gap Analysis mindset | هذا مفهوم استشاري مشترك أكثر منه مصطلح نظام |

## كيف تستخدم هذه المصفوفة عمليًا

### الحالة الأولى: أثناء الدراسة

إذا قرأت مصطلحًا جديدًا في أحد الأنظمة، حاول ربطه فورًا بالمفهوم العام الموجود في العمود الأول.

### الحالة الثانية: أثناء الحوار

إذا قال شخص في الشركة:

"نريد استلام المنتج التام"

فأنت يجب أن تفهم أن هذا قد يظهر في المراجع العالمية كواحد من المصطلحات التالية:

- Report as Finished
- Product Completion
- Goods Receipt

### الحالة الثالثة: أثناء التحليل الاستشاري

إذا وجدت نظامًا محليًا لا يملك مفهومًا واضحًا لما يقابل أحد هذه الصفوف، فهذه إشارة إلى فجوة وظيفية أو رقابية محتملة.

## ملاحظة مهمة

بعض الصفوف هنا تقوم على Conceptual Equivalence أكثر من التطابق الحرفي.

وهذا طبيعي.

لأن الغرض من الملف ليس فرض تطابق صوري بين الأنظمة، بل مساعدتك على بناء Concept Translation Layer داخل عقلك.
