# الدرس 04: الصورة الكاملة لدورة الإنتاج

## لماذا هذا الدرس مهم

الآن بعد أن فهمت الجوهر العام والمصطلحات والكيانات، تحتاج أن ترى الدورة الكاملة End-to-End Production Cycle بصورة واحدة واضحة.

هذا الدرس هو الجسر بين الفهم النظري وبين الدخول لاحقًا في تفاصيل التنفيذ والتكلفة.

## الهدف من الدرس

بعد إنهاء هذا الدرس يجب أن تكون قادرًا على:

1. شرح الدورة الكاملة للإنتاج من البداية إلى النهاية.
2. معرفة أين تبدأ البيانات وأين تبدأ الحركات الفعلية.
3. فهم متى يدخل الأثر المخزني ومتى يدخل الأثر التكاليفي.
4. رؤية الفروق اللفظية بين الأنظمة الثلاثة دون أن تضيع الفكرة الأساسية.

## الدورة العامة Universal Production Cycle

الصورة العامة المشتركة بين الأنظمة الثلاثة تكون غالبًا كالتالي:

1. تعريف البيانات الأساسية Master Data Setup
2. تعريف بنية المنتج Product Structure Definition
3. تعريف طريقة التصنيع Production Method Definition
4. إنشاء أمر الإنتاج Production Order / Work Order Creation
5. تفعيل الأمر أو إطلاقه Release
6. بدء التنفيذ Start / Execution Begin
7. صرف المواد Material Issue / Goods Issue
8. تنفيذ العمليات Operation Execution / Confirmation / Resource Charging
9. إثبات الناتج Finished Goods Receipt / Report as Finished / Product Completion
10. معالجة الهالك أو الإرجاع Scrap / Return / Rework if needed
11. تجميع ومراجعة التكلفة WIP / Actual Cost / Variance Review
12. إغلاق الأمر Close / Settlement / Final Status
13. إصدار التقارير الرقابية Reporting and Control

## أين تبدأ البيانات وأين تبدأ الحركات؟

هذه نقطة محورية جدًا.

### منطقة البيانات التعريفية Definition Zone

تشمل عادة:

- Product
- Material
- BOM / Formula / Item Structure
- Route / Work Definition
- Resources / Work Centers

هذه المنطقة لا تنفذ الإنتاج بعد، لكنها تبني المنطق الذي سيُنفذ لاحقًا.

### منطقة التنفيذ Transaction Zone

تشمل عادة:

- Production Order / Work Order
- Material Issue
- Operation Execution
- Finished Goods Receipt
- Returns / Scrap

هذه المنطقة هي التي تبدأ فيها الحركة الفعلية على المخزون والتكلفة.

### منطقة الرقابة والإغلاق Control Zone

تشمل عادة:

- WIP Review
- Variance Analysis
- Closing
- Period Control
- Reporting

هذه المنطقة هي التي تحكم على مدى صحة ما حدث.

## متى يتأثر المخزون Inventory Impact؟

المخزون لا يتأثر لمجرد تعريف Product أو BOM أو Route.

يتأثر عادة عندما تبدأ Transactions حقيقية مثل:

- Material Issue / Goods Issue
- Finished Goods Receipt / Product Completion / Report as Finished
- Material Return
- Scrap Movements

هذا التفريق مهم جدًا، لأن بعض المبتدئين يخلط بين Setup Data وبين Inventory Impact.

## متى يتأثر WIP وCost؟

الأثر التكاليفي Cost Impact يبدأ منطقيًا مع الأحداث التنفيذية، مثل:

- صرف المواد Materials Consumed
- تسجيل العمالة أو الآلة Resource Charging / Confirmation
- تحميل التكاليف غير المباشرة Overhead عند الحاجة
- إدخال الناتج النهائي مع استمرار بقاء جزء من الأمر غير مغلق

أما الحسم النهائي Final Cost Resolution فعادة يرتبط بالإغلاق Closing.

## الخريطة المقارنة بين الأنظمة الثلاثة

| الخطوة العامة | Dynamics 365 | Oracle Fusion Cloud | SAP S/4HANA |

| تعريف المنتج | Released Product | Item | Material Master |
| بنية المنتج | BOM / Formula | Item Structure | BOM |

## أين يخطئ الناس في فهم الدورة؟

### الخطأ الأول

الاعتقاد أن إنشاء الأمر Creation يعني أن الدورة بدأت فعليًا على المخزون.

الصحيح:

إنشاء الأمر مهم، لكنه لا يساوي دائمًا Material Issue أو Finished Goods Receipt.

### الخطأ الثاني

الاعتقاد أن Finished Goods Receipt أو Product Completion يعني أن كل شيء انتهى.

الصحيح:

قد يكتمل التشغيل، لكن يبقى:

- WIP يحتاج مراجعة
- Variance تحتاج تفسير
- Closing يحتاج تنفيذ

### الخطأ الثالث

الخلط بين خطوة التنفيذ وخطوة الرقابة.

مثلًا:

- Material Issue = خطوة تنفيذية
- Variance Analysis = خطوة رقابية
- Closing = خطوة حسم وتحكم

## منظور المستشار Functional Consultant View

عندما تنظر إلى النظام المحلي، حاول أن تحدد:

1. ما الذي هو Setup فقط؟
2. ما الذي هو Transaction فعلية؟
3. ما الذي هو Control / Closing؟

إذا كان النظام يملك Transaction بدون Control، فسيبدو جيدًا ظاهريًا لكنه ضعيف رقابيًا.

إذا كان يملك Setup ضعيفًا، فكل التنفيذ سيخرج مضطربًا.

إذا كان يملك Execution جيدًا لكن Closing ضعيفًا، فالمشاكل ستظهر في WIP والتكلفة والتقارير.

## مثال ذهني سريع

تخيل أنك تصنع 100 وحدة من منتج معين.

- BOM حددت المواد المطلوبة.
- Route أو Work Definition حددت العمليات.
- Work Order أو Production Order أنشئ للكمية.
- صرفت المواد Material Issue.
- سجلت العمليات Operation Execution.
- استلمت 95 وحدة Finished Goods Receipt.
- بقي 5 وحدات Scrap أو Loss.
- هنا تبدأ أسئلة WIP وVariance وClosing.

هذا المثال البسيط يوضح لماذا لا تكفي خطوة واحدة لفهم الدورة كلها.

## الأخطاء الشائعة Common Mistakes

1. الخلط بين Definition Zone وTransaction Zone.
2. الخلط بين Finished Goods Receipt وClosing.
3. نسيان أن Quality وScrap وReturns قد تغير شكل الدورة.
4. الاعتقاد أن اختلاف أسماء الخطوات بين الأنظمة يعني اختلاف الدورة من جذورها.

## المراجعة السريعة Quick Review

- الدورة تبدأ من التعريف، لا من الحركة.
- المخزون يتأثر عند Transactions، لا عند Setup Data.
- WIP وActual Cost يرتبطان بالأحداث التنفيذية.
- Closing ليس مجرد خطوة شكلية، بل نقطة حسم.
- Dynamics وOracle وSAP تختلف في الأسماء والتفاصيل، لكن تشترك في منطق دورة عامة متقاربة.

## مهمة الدرس Study Task

اكتب دورة الإنتاج العامة من ذاكرتك في 12 خطوة.

ثم تحت كل خطوة اكتب مثالًا واحدًا من الإنجليزية يطابقها، مثل:

- صرف مواد = Material Issue
- إغلاق = Closing
- أمر إنتاج = Production Order
