# الدرس 02: الإطلاق والبدء والصرف Release Start Material Issue

## لماذا هذا الدرس مهم

لأن أول أثر حقيقي في أرض المصنع لا يبدأ عند إنشاء الأمر، بل عندما يصبح جاهزًا للتنفيذ ثم يبدأ الصرف الفعلي للمواد. هذه هي النقطة التي ينتقل فيها النظام من نية التصنيع إلى استهلاك حقيقي للمخزون وبدء ظهور Work in Process أو WIP.

## الهدف من الدرس

بعد هذا الدرس يجب أن تكون قادرًا على:

- شرح الفرق بين Release وStart.
- فهم معنى Material Issue وGoods Issue وComponent Issue.
- تفسير متى يبدأ استهلاك المخزون ومتى يبدأ أثر WIP.
- التمييز بين الصرف اليدوي Manual issue والصرف التلقائي Backflush أو Auto issue.

## الشرح المفاهيمي الأساسي

### Release

Release يعني أن الأمر أصبح معتمدًا للتنفيذ.

عمليًا، هذا يعني أن النظام يعتبر أن:

- البيانات الأساسية الأساسية صحيحة بما يكفي.
- العمليات والمواد الموجودة على الأمر قابلة للاستخدام.
- الأمر يمكن أن يظهر للمشغلين أو للمخزن أو للمتابعة اليومية.

لكن Release وحده لا يعني أن المواد صُرفت ولا أن التشغيل بدأ.

### Start

Start يعني بدء التنفيذ الفعلي. في بعض الأنظمة تكون هذه المرحلة صريحة وواضحة مثل Dynamics، وفي أنظمة أخرى يكون الانتقال عمليًا مرتبطًا ببدء الحركات نفسها، مثل أول Issue أو أول Operation transaction.

### Material Issue

Material Issue أو Goods Issue هو حركة إخراج مواد من المخزون إلى أمر الإنتاج.

هذا الحدث مهم جدًا لأنه:

- يخفض On-hand Inventory.
- يرفع استهلاك المواد على الأمر.
- يبدأ تكوين WIP أو Cost Collection بشكل أوضح.
- يكشف الفروقات بين الاستهلاك المعياري Actual consumption.

## طرق الصرف

| الطريقة | المعنى | متى تستخدم؟ |
| --- | --- | --- |
| Manual Issue | المستخدم يحدد الصرف صراحة | عندما تريد رقابة عالية أو مرونة أكبر |
| Backflush | الصرف يتم تلقائيًا عند مرحلة لاحقة | عندما تكون العملية مستقرة ومتكررة |
| Pre-flush / Auto Issue | الصرف يتم تلقائيًا عند Start أو Release أو Operation step | عندما ترغب في تقليل الإدخال اليدوي |

لا توجد طريقة أفضل مطلقًا. الطريقة الصحيحة تعتمد على:

- استقرار العملية الإنتاجية.
- دقة BOM.
- نضج الرقابة الداخلية.
- حساسية المواد وقيمتها.

## كيف يظهر المفهوم في الأنظمة الثلاثة

### Dynamics 365

في Dynamics يمر الأمر عادة عبر:

- Release
- Start
- Picking list journal أو Material consumption

وهنا يظهر الفصل بوضوح بين جعل الأمر جاهزًا وبين بدء التنفيذ فعليًا. كما أن Dynamics يدعم استهلاكًا يدويًا أو تلقائيًا وفق إعدادات flushing والـ journals.

### Oracle Fusion Cloud

في Oracle، Release يجعل Work Order جاهزًا للتنفيذ، ثم تبدأ معاملات Issue Materials. وقد يتأثر توقيت الصرف بتعريف Supply Type مثل Push وOperation Pull وAssembly Pull.

هذا مهم جدًا لأن Oracle يربط توقيت الصرف بمنطق استهلاك المكون داخل العملية، وليس فقط بقرار يدوي مباشر.

### SAP S/4HANA

في SAP يظهر المفهوم غالبًا عبر:

- Release للحالة REL
- Material Staging / Reservation
- Goods Issue للمكونات

كما أن Backflush في SAP قد يتم من خلال Confirmation أو إعدادات Material Master وRouting، لذلك يجب فهم الصرف في SAP من زاويتين:

- الصرف كحركة مستقلة Goods Issue
- الصرف كأثر ناتج عن Confirmation أو Backflush

## منظور المستشار Functional Consultant View

عند تحليل بيئة عميل، اسأل الأسئلة التالية:

- هل المواد تصرف كلها قبل التشغيل أم أثناءه أم بعده بالاعتماد على Backflush؟
- ما المواد التي يجب أن تكون Manual بسبب قيمتها أو حساسيتها؟
- هل يوجد فرق بين المواد الرئيسية والمواد الاستهلاكية Consumables؟
- من يملك صلاحية Release؟ ومن يملك صلاحية Issue؟
- ماذا يحدث عندما لا توجد كمية متاحة On-hand؟

هذه الأسئلة مهمة لأن كثيرًا من مشاكل التنفيذ ليست في Work Order نفسه، بل في سياسة الصرف Material issue policy.

## المخاطر العملية

| الخطر | النتيجة التشغيلية | النتيجة التكلفية |
| --- | --- | --- |
| Release مبكر قبل جاهزية المواد | أمر متوقف على الأرض | تأخير وقياس أداء مضلل |
| Issue زائد | استهلاك غير حقيقي | تضخم WIP أو تكلفة المنتج |
| Issue ناقص | إنتاج فعلي دون تسجيل كامل | تكلفة أقل من الواقع |
| Backflush مع BOM ضعيفة | استهلاك تلقائي خاطئ | Variances مرتفعة ومتكررة |
| الصرف من مخزن خاطئ | تتبع مخزني مشوش | تقارير inventory غير موثوقة |

## الأخطاء الشائعة Common Mistakes

- اعتبار Release حدثًا تنفيذيًا مكتملاً.
- اعتبار أول Goods Issue مجرد حركة مخزنية وليس حدثًا تكلفيًا.
- تفعيل Backflush على مواد لا تتحمل الخطأ.
- إهمال Material staging وreservations قبل بدء الأمر.
- عدم التفريق بين نقص المادة وبين فشل الترحيل أو الصلاحيات.

## المراجعة السريعة Quick Review

- Release يفتح الباب للتنفيذ، لكنه ليس التنفيذ نفسه.
- Start أو أول معاملة تنفيذية يعني أن الأمر دخل الصالة فعليًا.
- Material Issue يخفض المخزون ويربط المواد بالأمر.
- طريقة الصرف يجب أن تتماشى مع نضج العميل ودقة بياناته.

## مهمة الدرس Study Task

حلل حالة عميل يقول: نحن نريد Backflush لكل المواد لأن الإدخال اليدوي متعب.

اكتب ردًا استشاريًا يوضح:

- متى يكون هذا القرار مناسبًا.
- متى يكون خطرًا.
- ما الشروط التي يجب تحققها قبل تطبيقه.
