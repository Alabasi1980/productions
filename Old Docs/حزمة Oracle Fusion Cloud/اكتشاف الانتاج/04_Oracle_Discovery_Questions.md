# أسئلة تحليل عميل الإنتاج في Oracle Fusion Cloud Manufacturing

## 1. مقدمة

هذا الملف يحوّل مفاهيم **Oracle Fusion Cloud Manufacturing** إلى أسئلة تحليل عملية يستخدمها الـ **Functional Consultant** أو **Business Analyst** لفهم واقع الإنتاج لدى العميل قبل التطبيق. الهدف ليس تعبئة نموذج أسئلة، بل كشف منطق العمل الحقيقي خلف الإنتاج: كيف تُعرّف المنتجات، كيف تُبنى **Item Structures** و **Work Definitions**، كيف تُدار **Work Orders**، كيف تُصرف المواد، كيف تُسجل الموارد والعمليات، وكيف تتحول هذه الأحداث إلى مخزون وتكلفة و **Cost Distributions** ثم إلى أثر محاسبي عبر **Subledger Accounting** و **General Ledger**.

يجب أن تساعد هذه الأسئلة على كشف:

- طريقة الإنتاج الفعلية.
- طريقة تعريف المنتجات والمواد والمخرجات.
- طريقة تصميم **Work Definitions**.
- طريقة إنشاء وإدارة **Work Orders**.
- طريقة صرف المواد حسب **Supply Type**.
- طريقة تسجيل الموارد والعمليات.
- طريقة إكمال المنتج **Product Completion**.
- طريقة احتساب التكلفة و **WIP**.
- علاقة الإنتاج بالمخزون والمحاسبة.
- نقاط الخطر قبل **Go-Live**.

## 2. طريقة استخدام هذا الملف

لا تُطرح كل الأسئلة دفعة واحدة. يتم اختيار الأسئلة حسب نوع العميل وحجم التطبيق ودرجة تعقيد التصنيع. يجب عقد جلسات منفصلة أو متتابعة مع الإنتاج، المستودعات، التخطيط، الجودة، محاسبة التكاليف، والمالية. كل إجابة يجب أن تتحول إلى إعداد في النظام، متطلب تقرير، ضابط صلاحيات، سيناريو اختبار، أو قرار تأجيل لمرحلة لاحقة.

| نوع الاجتماع | الأقسام المطلوبة | الهدف |
|---|---|---|
| اجتماع فهم النشاط | الإدارة، الإنتاج، التخطيط | تحديد نوع الإنتاج: Discrete / Process / Mixed-Mode / Contract / Outside Processing. |
| اجتماع Master Data | الإنتاج، PIM/الأصناف، المستودعات، التكلفة | توثيق Items، Item Structures، UOM، Lot/Serial، Inventory Organizations. |
| اجتماع Work Definition | مهندس التصنيع، الإنتاج، الجودة، التكلفة | فهم Operations، Work Centers، Resources، Outputs، Versions، Priorities. |
| اجتماع Work Orders | الإنتاج، التخطيط، المستودعات | تحديد مصادر إنشاء Work Orders، الحالات، الصلاحيات، Dispatch List. |
| اجتماع Material Issue | المستودعات، الإنتاج، التكلفة | تحديد Supply Types، الصرف اليدوي/الآلي، Lot/Serial، المرتجعات. |
| اجتماع Execution & Resources | الإنتاج، المشغلون، الصيانة، التكلفة | تحديد Count Point، Auto-Transact، Optional Operations، Resource Charging. |
| اجتماع Cost Accounting | محاسب التكاليف، المالية، الإنتاج | تحديد Cost Method، Cost Profiles، WIP، Cost Distributions، Variances. |
| اجتماع Quality & Exceptions | الجودة، الإنتاج، الصيانة، المستودعات | تحديد Production Exceptions، Inspection، Scrap، Rework، Reject. |
| اجتماع Period Close | المالية، محاسبة التكاليف، الإنتاج | ضبط Close، Pending Transactions، WIP، GL Transfer، تقارير نهاية الفترة. |
| اجتماع UAT/Go-Live | كل الأطراف | اعتماد سيناريوهات دورة كاملة قبل التشغيل. |

## 3. أسئلة الفهم العام لطبيعة الإنتاج

| السؤال | الهدف من السؤال | أثر الإجابة على تصميم النظام |
|---|---|---|
| ماذا تنتجون؟ | تحديد طبيعة المنتج النهائي والمخرجات. | يحدد الأصناف، Work Definitions، التقارير، وطريقة التكلفة. |
| هل الإنتاج Discrete أم Process أم Mixed-Mode؟ | تحديد نمط التصنيع. | يؤثر على Work Method، Outputs، Co-products، By-products، وإعدادات المصنع. |
| هل الإنتاج للتخزين Make to Stock أم حسب الطلب Make to Order؟ | فهم علاقة الإنتاج بالطلب. | يؤثر على Planning، Orchestration، Reservations، وWork Order Source. |
| هل يوجد Configure to Order أو Back-to-Back؟ | تحديد هل الأمر مرتبط مباشرة بطلب بيع. | يؤثر على Supply Chain Orchestration، الحجز، ومواعيد التسليم. |
| هل يوجد Project-specific production؟ | معرفة إن كانت الأوامر مرتبطة بمشاريع. | يؤثر على تتبع التكلفة والتقارير حسب المشروع. |
| هل يوجد Contract Manufacturing؟ | تحديد وجود مصنع خارجي كامل. | يؤثر على العلاقة مع المورد، المخزون، والتكلفة. |
| هل يوجد Outside Processing؟ | تحديد عمليات معينة تنفذ عند مورد. | يؤثر على Supplier Operations، Procurement، والتكلفة. |
| هل لديكم أكثر من مصنع أو Inventory Organization؟ | فهم التعدد التنظيمي. | يؤثر على Plant Parameters، Cost Organization، Subinventory، وWork Definitions. |
| هل يوجد أكثر من خط إنتاج؟ | تحديد تعقيد الموارد والجدولة. | يؤثر على Work Areas، Work Centers، Alternate Work Definitions. |
| هل الإنتاج مستمر أم على دفعات؟ | تحديد طريقة التنفيذ والإكمال. | يؤثر على Work Order، Product Completion، Batch/Lot Tracking. |
| هل الإنتاج داخلي بالكامل أم يوجد موردون في مراحل معينة؟ | تحديد مسؤولية التنفيذ. | يؤثر على Supplier Supply وOutside Processing. |
| هل يوجد Orderless Manufacturing؟ | معرفة إن كان الإنتاج يسجل بدون Work Order. | يحتاج تحقق إضافي من المصدر الأصلي عند التطبيق التفصيلي. |
| هل توجد منتجات نصف مصنعة؟ | تحديد Subassemblies. | تؤثر على Work Definitions منفصلة أو Phantom داخل Work Definition. |
| هل توجد Co-products أو By-products؟ | تحديد مخرجات متعددة. | يؤثر على Process Work Definition وOperation Outputs والتكلفة. |

## 4. أسئلة المنتجات والأصناف Master Data

| السؤال | لماذا مهم؟ | أثره على الإنتاج/المخزون/التكلفة |
|---|---|---|
| ما المنتجات النهائية؟ | لأنها رأس Work Definition وWork Order. | تحدد Product Completion وFinished Goods Inventory. |
| ما المواد الخام؟ | لأنها تدخل في Item Structure وOperation Items. | تحدد Material Issue وWIP وMaterial Cost. |
| هل توجد منتجات نصف مصنعة أو Subassemblies؟ | لتحديد هل لها Work Definitions مستقلة أو Phantom. | تؤثر على التسلسل والتكلفة والتخطيط. |
| هل توجد Ad hoc Items؟ | المقال يذكر إمكانية إضافتها بصلاحية خاصة. | تؤثر على ضبط الصلاحيات والتكلفة إذا أضيفت خارج Item Structure. |
| هل توجد أصناف تُشترى وتُصنع في نفس الوقت؟ | لتحديد Supply/Make/Buy. | يؤثر على التخطيط والمخزون والتكلفة. |
| ما وحدات القياس المستخدمة؟ | Oracle يربط UOM بالأصناف والموارد والعمليات. | خطأ UOM يؤدي إلى صرف أو تكلفة غير دقيقة. |
| هل تختلف وحدة الشراء عن الإنتاج أو التخزين؟ | لضبط التحويلات. | يؤثر على الكميات والتكلفة. |
| هل توجد تحويلات بين الوحدات؟ | لمنع أخطاء كمية. | يؤثر على Material Requirements وCost Rollup. |
| هل يوجد Lot Control أو Serial Control؟ | للتتبع والصرف والإكمال. | يؤثر على Issue، Completion، Genealogy، والجودة. |
| هل خصائص Costing Enabled مضبوطة؟ | المقال يربطها بالتسعير. | خطأها يؤدي إلى تكلفة غير صحيحة أو حركات غير مسعرة. |
| هل Inventory Asset Value مضبوط؟ | يحدد المعاملة المحاسبية للصنف. | يؤثر على Cost Profile والتقييم. |
| هل تختلف المنتجات حسب المصنع أو Inventory Organization؟ | لأن Manufacturing يعمل داخل Inventory Organization. | يؤثر على Work Definition وPlant Parameters. |
| هل توجد مواد منتهية الصلاحية يجب منع صرفها؟ | Oracle لديه Prevent Issue of Expired Lots. | يمنع صرف Lots غير صالحة ويقلل مخاطر الجودة. |
| هل توجد أصناف تحتاج Traceability أو Genealogy؟ | مهم للصناعات المنظمة. | يربط Lot/Serial المكوّن بالمنتج النهائي. |

## 5. أسئلة Item Structure / BOM

| السؤال | الهدف | أثر الإجابة على Work Definition / Work Order / Costing |
|---|---|---|
| هل لكل منتج Item Structure موثقة؟ | معرفة هل توجد قاعدة مواد رسمية. | بدونها لا يمكن بناء Work Definition موثوقة. |
| من المسؤول عن إعداد واعتماد Item Structure؟ | ضبط الحوكمة. | يمنع تغييرات غير معتمدة تؤثر على الصرف والتكلفة. |
| هل المكونات ثابتة أم تتغير؟ | تحديد الحاجة للإصدارات أو التحديثات. | يؤثر على Work Definition Synchronization. |
| هل توجد بدائل مواد؟ | فهم Material Substitution. | قد تؤدي إلى Material Substitution Variance. |
| هل توجد منتجات نصف مصنعة داخل الهيكل؟ | تحديد Subassemblies أو Phantom. | يؤثر على Work Orders الفرعية أو تفجير المكونات. |
| هل توجد مواد Ad hoc أثناء التصميم أو التنفيذ؟ | ضبط الاستثناءات. | تؤثر على التكلفة والصلاحيات. |
| هل توجد مكونات لها Supply Type مختلف؟ | ربط المكونات بطريقة الصرف. | يؤثر على Push / Pull / Bulk / Phantom / Supplier. |
| هل توجد مكونات Phantom؟ | فهم التفجير داخل الأمر. | تؤثر على المواد المنسوخة إلى Work Order. |
| هل توجد مكونات Supplier Supply؟ | تحديد مواد يقدمها المورد. | مهم في Outside Processing. |
| هل توجد Co-products أو By-products؟ | فهم المخرجات المتعددة. | يؤثر على Process Work Definition والتكلفة. |
| هل توجد تغييرات متكررة على Item Structure؟ | تقييم مخاطر التغيير. | تؤثر على Work Definitions وأوامر تاريخية. |
| كيف يتم إشعار Manufacturing Engineer أو Cost Accountant عند التغيير؟ | المقال يذكر Change Notification. | مهم لتحديث Work Definition وCosting. |
| هل تتم مزامنة التغييرات مع Work Definitions؟ | التأكد من عدم انفصال التصميم عن التنفيذ. | يمنع استخدام Work Definition قديمة. |
| هل توجد صلاحية واضحة لمن يغيّر المكونات؟ | ضبط الحوكمة. | يمنع أثر مالي وتشغيلي غير مقصود. |

### علامات الخطر في Item Structure

| علامة الخطر | ماذا تعني؟ | الإجراء المقترح |
|---|---|---|
| Item Structure غير موثقة | الإنتاج يعتمد على معرفة شفوية. | توثيق واعتماد الهياكل قبل التطبيق. |
| الاعتماد شفهي | لا توجد حوكمة للتغييرات. | تحديد مالك اعتماد رسمي. |
| تغييرات كثيرة دون إشعار | Work Definitions قد تصبح غير متزامنة. | تفعيل آلية مراجعة التغييرات. |
| عدم وضوح المنتجات نصف المصنعة | قد تختلط بين Phantom وأمر مستقل. | تحديد سياسة Subassembly. |
| عدم وضوح Supply Type | الصرف قد يكون خاطئًا. | مراجعة كل Operation Item. |
| اختلاف الواقع عن النظام | تكلفة ومخزون غير صحيحين. | تنفيذ UAT بمنتجات فعلية. |

## 6. أسئلة Work Definitions

| السؤال | الهدف | أثر الإجابة على النظام |
|---|---|---|
| هل لكل منتج Work Definition واضحة؟ | التأكد من وجود قالب إنتاج. | شرط أساسي لإنشاء Work Order قياسي. |
| هل توجد Primary Work Definition؟ | Oracle يحتاج Primary بأولوية إنتاج 1. | تؤثر على التخطيط وLead Times. |
| هل توجد Alternate Work Definitions؟ | تمثيل خطوط أو مواد أو عمليات بديلة. | تؤثر على التخطيط والتكلفة. |
| هل تختلف Work Definition حسب المصنع أو العميل أو الدفعة؟ | تحديد الحاجة لتعدد الإصدارات أو البدائل. | يؤثر على اختيار Work Definition عند إنشاء الأمر. |
| هل توجد Work Definition Versions؟ | تتبع تغييرات طريقة الإنتاج عبر الزمن. | يؤثر على إعادة التسعير التاريخي والدقة. |
| من يملك صلاحية إنشاء Work Definition؟ | ضبط الفصل بين التصميم والتنفيذ. | يحمي من تغييرات غير معتمدة. |
| من يعتمد Work Definition؟ | تحديد مسؤولية الحوكمة. | يؤثر على جودة الإنتاج والتكلفة. |
| هل توجد Effective Start Date وEffective End Date؟ | ضبط صلاحية الإصدار. | يمنع استخدام طريقة إنتاج غير سارية. |
| هل توجد Production Priority؟ | تحديد Primary/Alternate للإنتاج. | يؤثر على Planning وWork Order. |
| هل توجد Costing Priority؟ | تحديد التعريف المستخدم للتكلفة. | قد تختلف عن Production Priority. |
| هل Work Definition تجمع Item Structure + Operations + Resources + Outputs؟ | تأكيد اكتمال القالب. | بدونها لا تكتمل صورة التكلفة. |
| هل تختلف Work Definition المستخدمة للتكلفة عن الإنتاج؟ | فهم سيناريوهات Cost Rollup. | يؤثر على Standard Cost. |
| هل توجد Rework أو Transform Work Definitions؟ | تحديد سيناريوهات إصلاح أو تحويل. | يؤثر على جودة وتكلفة إعادة التشغيل. |
| هل توجد Process Work Definitions للمخرجات المتعددة؟ | دعم Co-products وBy-products. | يؤثر على Operation Outputs. |
| هل Serial Tracking يبدأ من عملية معينة؟ | دعم التتبع التفصيلي. | يؤثر على الصرف والإكمال والتقارير. |
| هل يتم تحديث Work Definition عند تغيير Item Structure؟ | منع الانفصال بين التصميم والتنفيذ. | يؤثر على Work Orders الجديدة والتكلفة. |
| هل Completion Subinventory / Locator واضحة؟ | تحديد مكان استلام المنتج. | يؤثر على Product Completion والمخزون. |

### أسئلة حرجة قبل إعداد Work Definition

1. ما المنتج النهائي الذي سيمثل رأس Work Definition؟
2. هل Item Structure معتمدة ومحدثة؟
3. ما العمليات الفعلية التي يمر بها المنتج؟
4. ما Work Centers المستخدمة في كل عملية؟
5. ما Resources المطلوبة: Labor / Equipment؟
6. ما Supply Type لكل مادة؟
7. هل توجد Outputs متعددة؟
8. هل توجد Versions أو Alternate Work Definitions؟
9. ما Completion Subinventory / Locator؟
10. هل التعريف سيستخدم للتخطيط فقط أم للتكلفة أيضًا؟

## 7. أسئلة Operations / Work Centers / Resources

| السؤال | الهدف | أثر الإجابة على الجدولة والتكلفة |
|---|---|---|
| هل لديكم مراحل إنتاج واضحة؟ | تحديد Operations. | يؤثر على Work Definition والتنفيذ. |
| ما العمليات الرئيسية؟ | توثيق التسلسل. | يؤثر على Count Point وCompletion. |
| هل العمليات تسلسلية أم يمكن تجاوز بعضها؟ | تحديد Optional أو Auto-Transact. | يؤثر على التسجيل والتكلفة. |
| هل توجد Count Point Operations؟ | معرفة العمليات الإلزامية. | عدم تسجيلها يؤثر على الإكمال. |
| هل توجد Auto-Transact Operations؟ | معرفة العمليات التي تسجل تلقائيًا. | يقلل الإدخال اليدوي لكنه يحتاج ضبط. |
| هل توجد Optional Operations؟ | معرفة العمليات القابلة للتجاوز. | قد لا تدخل التكلفة إذا لم تنفذ. |
| هل كل عملية مرتبطة بـ Work Center؟ | Oracle يربط Operation بمركز عمل. | يؤثر على الموارد والجدولة. |
| هل Work Centers معرفة داخل Work Areas؟ | ضبط هيكل المصنع. | يؤثر على التنظيم والتقارير. |
| هل توجد Workstations فعلية؟ | ربط التنفيذ بموقع مادي. | مفيد لـ Check-in/Check-out. |
| هل توجد Resources من نوع Labor أو Equipment؟ | تحديد تكلفة الموارد. | يؤثر على Resource Charging وWIP. |
| هل توجد Resource Instances؟ | تتبع آلة أو عامل محدد. | مفيد للتشغيل التفصيلي. |
| هل لكل Resource UOM صحيحة؟ | المقال يوضح أهمية UOM للمورد. | خطأها يؤثر على الجدولة والتكلفة. |
| هل توجد Shifts أو Calendars؟ | تحديد الطاقة المتاحة. | يؤثر على Scheduling. |
| هل توجد Utilization وEfficiency؟ | ضبط مدة الموارد. | أقل من 100% يطيل المدة المجدولة. |
| هل توجد Resource Rates؟ | شرط لتكلفة الموارد. | تؤثر مباشرة على WIP والتكلفة. |
| هل يتم تسجيل استخدام الموارد يدويًا أم تلقائيًا؟ | تحديد أسلوب التنفيذ. | يؤثر على الدقة والرقابة. |
| هل يوجد Operator Check-in / Check-out؟ | تسجيل الاستخدام من محطة العمل. | يؤثر على وقت الموارد. |
| هل توجد عمليات خارجية Supplier Operations؟ | تحديد Outside Processing. | يؤثر على Procurement والتكلفة. |

## 8. أسئلة Work Orders

| السؤال | الهدف | أثر الإجابة على دورة العمل |
|---|---|---|
| من ينشئ Work Order؟ | تحديد المسؤولية. | يؤثر على الصلاحيات والحوكمة. |
| هل مصدره يدوي؟ | معرفة الحاجة لشاشة إدخال. | يؤثر على إجراءات الإنتاج. |
| هل يأتي من Supply Chain Planning؟ | ربط Plan-to-Produce. | يؤثر على التخطيط والتوصيات. |
| هل يأتي من Order Management / Orchestration؟ | دعم Back-to-Back أو CTO. | يؤثر على الحجز والعميل. |
| هل يأتي من Min-Max Planning؟ | دعم تجديد المخزون. | يؤثر على MTS. |
| هل يستخدم REST API / FBDI؟ | معرفة الحاجة للتحميل أو التكامل. | يؤثر على التكامل والتحكم. |
| هل توجد Standard وNonstandard Work Orders؟ | تحديد الحالات خارج Work Definition. | Nonstandard يحتاج ضوابط خاصة. |
| هل توجد Rework أو Transform Work Orders؟ | دعم الإصلاح أو التحويل. | يؤثر على الجودة والتكلفة. |
| هل يوجد Orderless Manufacturing؟ | فهم سيناريوهات Flow. | يحتاج تحقق إضافي من المصدر الأصلي. |
| هل Work Order ينسخ Operations وItems وResources من Work Definition؟ | تأكيد منطق Oracle. | يجعل الأمر كائنًا تنفيذيًا مستقلًا. |
| هل يسمح بتعديل Work Order بعد الإنشاء؟ | ضبط المرونة مقابل الحوكمة. | يؤثر على التكلفة والتحليل. |

## 28. ملخص تنفيذي

أسئلة التحليل في Oracle Manufacturing ليست أسئلة تشغيلية فقط، بل أداة لاكتشاف الواقع التشغيلي والمالي معًا. نجاح التطبيق يعتمد على وضوح **Item Structure** و **Work Definition** و **Work Order Lifecycle**، وعلى فهم أثر **Material Issue** و **Resource Charging** و **Product Completion** على **WIP** و **Cost Accounting**.

لا يجب تطبيق الإنتاج قبل وضوح:

- طريقة تعريف المنتجات والمواد.
- طريقة بناء Work Definitions.
- سياسة Supply Types.
- طريقة تنفيذ Work Orders وحالاتها.
- سياسة Product Completion.
- سياسة WIP وCost Distributions.
- سياسة Work Order Close وPeriod Close.
- التقارير المطلوبة للرقابة.
