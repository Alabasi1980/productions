# مخاطر وفشل تطبيق نظام الإنتاج في Microsoft Dynamics 365

## 1. مقدمة

تطبيق نظام الإنتاج في ERP من أكثر التطبيقات حساسية، لأنه لا يمس شاشة Production Order فقط، بل يمس سلسلة مترابطة تشمل المواد، المستودعات، التصنيع، التكلفة، WIP، القيود المالية، التقارير، وقرارات الإدارة.

في Microsoft Dynamics 365، الإنتاج منظومة مترابطة بين Product information management وInventory management وWarehouse management وProduction control وCost management وGeneral ledger. لذلك فإن أي خلل في بيانات المنتج أو BOM أو Route أو تكلفة الموارد أو سياسة الإغلاق يظهر لاحقًا كخلل في المخزون أو التكلفة أو المالية.

فشل تطبيق الإنتاج لا يظهر عادةً في الشاشة فقط، بل يظهر في:

- تكلفة منتج غير صحيحة.
- مخزون غير منطقي.
- أوامر إنتاج مفتوحة لفترات طويلة.
- WIP متضخم.
- انحرافات Variances غير مفسرة.
- قيود مالية في حسابات خاطئة.
- تقارير لا تخدم الإدارة ولا تساعد على اتخاذ القرار.

هذا الملف يحوّل ما ورد في المقال الأصلي إلى مرجع عملي لفهم مخاطر تطبيق Module الإنتاج وأسباب الفشل الشائعة وكيفية الوقاية منها.

## 2. خريطة المخاطر حسب دورة الإنتاج

| المرحلة | الخطر الرئيسي | الأثر المحتمل |
|---|---|---|
| Master Data | بيانات الأصناف أو مجموعات الأصناف أو أبعاد المخزون غير صحيحة | أخطاء في المخزون، التقييم، الحسابات، والتقارير |
| BOM / Formula | BOM غير دقيقة أو غير معتمدة أو لا تعكس الواقع | صرف مواد خاطئ وتكلفة منتج غير صحيحة |
| Routes / Operations / Resources | Route أو الموارد أو أوقات التشغيل غير دقيقة | جدولة غير واقعية وتكلفة عمالة/موارد خاطئة |
| Production Order Creation | إنشاء أمر بدون بيانات صالحة أو مصدر واضح | أمر إنتاج غير قابل للتنفيذ أو غير قابل للتكلفة |
| Estimate | تقدير مبني على BOM أو Route خاطئة | Estimated Cost غير موثوقة وحركات On order مضللة |
| Scheduling | Calendar أو Capacity أو Resource غير مضبوط | تأخير أوامر الإنتاج وجدولة لا تعكس الواقع |
| Release | إطلاق الأمر قبل جاهزية المواد أو المستودعات | تعطل صالة الإنتاج أو Picking غير صحيح |
| Start | بدء الأمر قبل ضبط الصرف أو WIP | صرف تلقائي خاطئ أو قيود WIP غير مفهومة |
| Material Issue | صرف أكثر/أقل من BOM أو صرف بدائل غير مضبوطة | Variances ومخزون غير مطابق للواقع |
| Route / Job Reporting | عدم تسجيل الوقت أو تسجيله خطأ | تكلفة تشغيل وانحرافات غير صحيحة |
| Report as Finished | استلام كمية غير صحيحة أو قبل الجودة | مخزون منتج تام غير موثوق وWIP غير صحيح |
| Scrap / Quality | هالك غير معرف أو جودة غير مربوطة بالمخزون | تحميل تكلفة خاطئ وإتاحة مخزون غير صالح |
| Close / End | ترك أوامر مفتوحة أو إغلاق بدون مراجعة | WIP متضخم وانحرافات غير مكتشفة |
| Variance Analysis | الانحرافات لا تُراجع | استمرار أخطاء BOM وRoute والتكلفة |
| Financial Posting | حسابات الترحيل أو Ledger posting غير صحيحة | قيود خاطئة في دفتر الأستاذ |
| Reports | غياب تقارير WIP والتكلفة والانحرافات | الإدارة لا ترى المخاطر مبكرًا |
| UAT / Go-Live | اختبار شاشة بدل دورة إنتاج كاملة | فشل بعد التشغيل وعودة المستخدمين إلى Excel |

## 3. مخاطر Master Data

| الخطر | سبب ظهوره | أثره على الإنتاج | أثره على المخزون | أثره المالي | طريقة الوقاية |
|---|---|---|---|---|---|

| Item Model Groups خاطئة | عدم ضبط طريقة تقييم المخزون أو Physical/Financial update | تكلفة الإنتاج لا تُحتسب كما هو مطلوب | حركات مخزون لا تظهر أو لا تُقيّم كما يجب | تقييم مخزون وتكلفة غير صحيحة | اعتماد طريقة تقييم المخزون لكل صنف مع محاسب التكاليف |

| Tracking Dimensions غير مفعلة عند الحاجة | عدم تفعيل Batch أو Serial للمواد الحساسة | صعوبة تتبع المواد المستخدمة في الإنتاج | ضعف Traceability للدفعات | صعوبة تفسير تكلفة دفعة محددة | تحديد الأصناف التي تحتاج Batch/Serial قبل الصرف |
| Storage Dimensions غير صحيحة | Site/Warehouse/Location غير مضبوطة | أوامر تسحب أو تستلم من أماكن خاطئة | On-hand غير موثوق | تكلفة قد تختلف حسب الموقع | إعداد Site وWarehouse وLocation وربطها بدورة الإنتاج |
| Site / Warehouse / Location غير واضحة | غياب نموذج مستودعي للإنتاج | تعطل الصرف والاستلام | مواد في مواقع خاطئة | تقارير مخزون وإنتاج غير قابلة للاعتماد | رسم خريطة مستودعات المواد والإنتاج والمنتج التام |
| أصناف بدون تكلفة | عدم إعداد Costing Version أو تكلفة نشطة | تقدير الأمر غير مكتمل | تقييم حركة المنتج غير صحيح | انحرافات أو تكلفة صفرية | تقرير أصناف بدون تكلفة قبل UAT |
| أصناف بدون BOM | منتج معرف كإنتاج دون وصفة | لا يمكن إنتاجه بمنطق صحيح | لا توجد مواد مطلوبة | تكلفة غير قابلة للحساب | تقرير منتجات Production بدون BOM معتمدة |
| أصناف بدون إعدادات مالية صحيحة | عدم مراجعة Posting profiles | الإنتاج ينتج حركات بلا ترحيل صحيح | حركة مخزون بلا أثر محاسبي صحيح | قيود خاطئة أو ناقصة | اختبار Posting من Picking وRAF وEnd قبل Go-Live |

## 4. مخاطر BOM / Formula

BOM الخاطئة مذكورة في المقال كأحد أهم أسباب فشل أنظمة الإنتاج. إذا لم يستطع العميل تقديم BOM دقيقة لكل منتج، فهذا مؤشر خطر يستدعي تأجيل تطبيق الإنتاج إلى حين توثيق BOM ومراجعتها.

| الخطر | كيف يظهر؟ | أثره التشغيلي | أثره على التكلفة | كيف نكتشفه؟ | كيف نمنعه؟ |
|---|---|---|---|---|---|
| BOM غير موثقة | الاعتماد على معرفة الموظفين أو Excel غير معتمد | صرف مواد غير موحد بين الأوامر | تكلفة متغيرة وغير موثوقة | اختلاف الصرف بين أوامر لنفس المنتج | توثيق BOM واعتمادها رسميًا |
| BOM غير معتمدة Approved | النظام لا يسمح باستخدامها أو يستخدم بدائل يدوية | تعطل إنشاء أوامر الإنتاج | لا يمكن حساب التكلفة بشكل صحيح | تقرير BOM غير معتمدة | وضع مسؤول اعتماد BOM |
| BOM غير مفعلة Active | BOM موجودة لكن ليست الافتراضية | Master planning أو أوامر الإنتاج لا تلتقطها تلقائيًا | تكلفة مبنية على نسخة غير صحيحة | تقرير Active BOM Versions | اعتماد وتفعيل النسخة الصحيحة |
| BOM Version غير صحيحة | نسخة زمنية أو كمية أو موقع غير مناسب | استخدام مكونات غير مناسبة للمنتج | Cost roll-up خاطئ | مقارنة BOM Version المستخدمة مع المنتج والموقع والتاريخ | ضبط قيود الموقع والتاريخ والكمية |
| مكونات ناقصة | مادة مستخدمة فعليًا غير موجودة في BOM | نقص مواد أثناء الإنتاج أو صرف يدوي متكرر | تكلفة المنتج أقل من الواقع وربح متضخم | Variance متكرر أو صرف خارج BOM | مراجعة BOM مع الإنتاج والمستودعات |
| مكونات زائدة | مواد لا تُستخدم فعليًا موجودة في BOM | صرف مواد غير مطلوبة | تكلفة المنتج أعلى من الواقع | مواد مرتجعة أو فائض متكرر | اختبار إنتاج تجريبي UAT |
| كميات خاطئة | كمية BOM لا تطابق الواقع | نقص أو فائض مستمر في الصرف | Production quantity variance | مقارنة المطلوب بالمصروف الفعلي | اعتماد كميات BOM من بيانات تشغيل فعلية |
| وحدات قياس خاطئة | استخدام كغ بدل غرام أو قطعة بدل كرتون | صرف مضاعف أو ناقص | تكلفة غير منطقية | تقرير فروقات صرف كبيرة | اختبار UoM conversions قبل Go-Live |
| عدم تعريف الهالك | BOM لا تحتوي Constant/Variable Scrap رغم وجود هالك | نقص مواد متكرر | انحرافات سالبة مستمرة | مراجعة الهالك الفعلي مقابل BOM | تعريف Scrap على مستوى BOM أو Route عند الحاجة |
| Constant Scrap غير واضح | الهالك الثابت غير معرف لكل أمر | نقص في مواد الإعداد أو الاختبار | تكلفة الدفعات الصغيرة غير صحيحة | مقارنة الهالك بين أحجام أوامر مختلفة | توثيق الهالك الثابت |
| Variable Scrap غير واضح | نسبة الهالك غير مضبوطة | نقص يتناسب مع حجم الإنتاج | تكلفة معيارية غير دقيقة | مراجعة الهالك كنسبة من الكمية | اعتماد نسب Scrap بعد مراجعة الإنتاج |
| Phantom مستخدم بطريقة خاطئة | منتج نصف مصنع يُفجّر داخل BOM بدل أمر منفصل أو العكس | فقدان تتبع الإنتاج نصف المصنع | تكلفة المرحلة الوسيطة غير واضحة | مراجعة المنتجات نصف المصنعة | تحديد سياسة Phantom مقابل أمر إنتاج فرعي |
| Pegged Supply غير مفهوم | إنشاء أوامر شراء/إنتاج فرعية تلقائيًا دون توقع | تشابك أوامر إنتاج فرعية | تكلفة وربط توريد غير مفهوم | مراجعة الأوامر الفرعية الناتجة عن Estimate | استخدام Pegged Supply فقط عند الحاجة الواضحة |
| Vendor Line غير مضبوط | تصنيع خارجي غير مرتبط بإجراء واضح | عدم وضوح مسؤولية المورد الخارجي | تكلفة المقاول من الباطن غير دقيقة | مراجعة أسطر Vendor في BOM | توثيق التصنيع الخارجي وسعره وإجراءه |
| Formula غير دقيقة | وصفة صناعات عملياتية لا تعكس النسب أو Yield | ناتج غير صحيح أو Co-products غير مضبوطة | توزيع تكلفة غير صحيح | مراجعة Formula مع الإنتاج والجودة | اعتماد Formula وFormula Version |
| Co-products وBy-products غير مضبوطة | المنتجات المشتركة أو الثانوية غير معرفة | استلام مخرجات ناقصة | توزيع تكلفة غير صحيح | مراجعة مخرجات Batch order | تحديد Cost distribution للمخرجات |
| BOM لا تعكس الواقع التشغيلي | المستخدمون يعدلون يدويًا عند كل أمر | ضعف الثقة في النظام | تكلفة وانحرافات لا تنتهي | ارتفاع التعديلات اليدوية | UAT بأوامر إنتاج حقيقية |

### أعراض فشل BOM بعد Go-Live

| العرض | ماذا يعني غالبًا؟ | التصرف المطلوب |
|---|---|---|
| صرف مواد غير صحيحة | BOM لا تعكس الواقع أو وحدة القياس خاطئة | مراجعة BOM والـ UoM |
| تكلفة منتج غير منطقية | BOM أو الأسعار أو الهالك غير مضبوط | إعادة BOM Calculation بعد التصحيح |
| مواد تنقص دائمًا | مكون ناقص أو Scrap غير معرف | تعديل BOM/Scrap |
| Variance متكرر | كمية BOM أو Route أو السعر غير دقيق | تحليل Production Quantity/Price Variance |
| المستخدمون يعدلون يدويًا كثيرًا | BOM غير موثوقة أو غير مرنة | ضبط إصدارات BOM والبدائل |

## 5. مخاطر Routes / Operations / Resources

| الخطر | أثره على الجدولة | أثره على التكلفة | أثره على التقارير | طريقة الوقاية |
|---|---|---|---|---|
| Route غير موجود | لا توجد مراحل تصنيع قابلة للجدولة | تكلفة العمل/الموارد لا تدخل | تقارير العملية غير متاحة | تعريف Route لكل منتج يحتاج مراحل تصنيع |
| Route غير مفعّل | الأمر لا يلتقط Route الصحيحة | Cost calculation غير صحيح | مقارنة المخطط بالفعلي غير صالحة | تفعيل Route Version الصحيحة |
| ترتيب Operations خاطئ | تسلسل الإنتاج غير واقعي | أوقات وتكلفة غير منطقية | متابعة العمليات مضللة | مراجعة Route مع مشرف الإنتاج |
| Operations غير مكتملة | عمليات فعلية لا تُسجل | تكلفة ناقصة | فجوة بين الإنتاج الفعلي والنظام | توثيق كل العمليات الرئيسية |
| Resource غير معرف | الجدولة لا تجد المورد | تكلفة المورد غير محسوبة | تقرير استخدام الموارد ناقص | تعريف Resources وResource groups |
| Resource بدون Calendar | فشل الجدولة أو طاقة غير واقعية | تحميل وقت غير مناسب | Capacity reports غير صحيحة | إعداد Calendars وWorking time templates |
| Work Center غير صحيح | حجز مورد خاطئ | Cost Category خاطئة | استخدام الموارد غير موثوق | مراجعة Work Centers/Resources |
| Setup Time غير دقيق | تاريخ بداية/نهاية غير واقعي | تكلفة الدفعة غير دقيقة | Lot size variance محتمل | قياس Setup time فعليًا |
| Run Time غير دقيق | Capacity غير واقعية | تكلفة الوحدة غير صحيحة | Production quantity variance بسبب الوقت | مراجعة Run time من بيانات تشغيل حقيقية |
| Cost Category غير صحيحة | لا أثر مباشر على الجدولة | سعر ساعة العمل/المورد خاطئ | تحليل التكلفة غير صحيح | اعتماد Cost Categories مع المالية |
| Route Group غير مناسب | قد لا تُسجل بعض الأوقات كما يجب | قيود Route/Job قد لا تظهر | تقارير تكلفة التشغيل ناقصة | ضبط Route Group وفق سياسة الترحيل |
| موارد تدخل في الجدولة ولا تدخل في التكلفة أو العكس | جدول صحيح ظاهريًا لكن تكلفة ناقصة أو مبالغ فيها | تكلفة غير متسقة | تقارير متناقضة | تحديد مورد لكل غرض: جدولة، تكلفة، أو الاثنين |
| Capacity غير واقعية | أوامر تتأخر أو تظهر قابلة للتنفيذ خطأ | تكلفة التأخير لا تظهر مباشرة | تخطيط غير موثوق | اختبار Finite/Infinite capacity |
| Parallel Operations غير مفهومة | تعارض في حجز الموارد | تحميل تكلفة موارد متعددة خطأ | تقارير العمليات المتوازية مضللة | عدم استخدامها إلا إذا كانت العملية مفهومة وموثقة |

## 6. مخاطر Production Orders

|---|---|---|---|---|
| إنشاء أمر إنتاج بدون بيانات صحيحة | عند Create قبل ضبط المنتج/BOM/Route | أمر غير قابل للتنفيذ | Estimated/Actual cost غير موثوقة | منع إنشاء الأوامر قبل جاهزية Master Data |
| إنشاء أمر من مصدر غير واضح | يدوي، Master Planning، Sales Order، أو Pegged Supply غير محدد | تكرار أوامر أو أوامر غير مطلوبة | التزام WIP ومخزون غير مبرر | تحديد مصادر إنشاء الأوامر وصلاحياتها |
| عدم وجود صلاحيات واضحة | كل مستخدم ينشئ أو يعدل | فقدان السيطرة على دورة الأمر | قيود وحركات غير معتمدة | توزيع صلاحيات Created/Estimate/Release/Start/RAF/End |

| أوامر ملغاة بدون معالجة واضحة | عند إلغاء أمر بعد حركات | حركات مخزون غير معكوسة | WIP أو قيود معلقة | إجراء إلغاء موثق ومختبر |

| Production Substitution Variance متكرر | استخدام مواد أو فئات تكلفة ليست في BOM/Standard | ضبط البدائل ومنع الاستبدال غير المعتمد |
| الانحرافات لا تُراجع | النظام يسجل الفشل ولا أحد يعالجه | اجتماع دوري لمراجعة الانحرافات |

| الإدارة لا تفهم دلالة الانحرافات | تقارير بلا قرار | تدريب الإدارة على معنى كل Variance |
| الانحرافات تظهر بعد فوات الأوان عند Closing | المشكلة لا تظهر أثناء التشغيل | تقارير مقارنة Estimated/Actual قبل الإغلاق؛ مستنتج من منطق المقال |

### ضوابط لا يجوز تجاهلها في نظام محلي

- حالة واضحة لأمر الإنتاج من الإنشاء إلى الإغلاق.
- BOM معتمدة ومفعلة.
- ربط الصرف بـ BOM.
- ربط الاستلام بـ Report as Finished.
- مفهوم WIP أو بديل محاسبي واضح.
- إغلاق مالي لأمر الإنتاج.
- تقارير تكلفة وانحرافات.
- UAT كامل بدورة إنتاج حقيقية.

## 25. ملخص تنفيذي

فشل نظام الإنتاج غالبًا لا يكون بسبب شاشة Production Order، بل بسبب البيانات والتكلفة والمالية. الشاشة قد تعمل، لكن المنتج لا يُحسب بشكل صحيح، والمخزون لا يطابق الواقع، وWIP يتضخم، والانحرافات تظهر دون تفسير.

BOM وRoute هما أساس النجاح. BOM تحدد ما سيُصرف وما سيدخل في تكلفة المواد. Route يحدد كيف يُصنع المنتج، وما الوقت والموارد الداخلة في التكلفة والجدولة. أي خطأ فيهما ينتقل مباشرة إلى المخزون والتكلفة والانحرافات.

WIP والإغلاق المالي من أخطر مناطق الفشل. إذا صُرفت المواد ولم تُغلق أوامر الإنتاج، تبقى التكلفة معلقة. وإذا لم تُراجع الانحرافات، يتحول النظام إلى أداة تسجيل أخطاء بدل أداة رقابة.

UAT يجب أن يغطي دورة كاملة وليس شاشة واحدة: تعريف منتج، BOM، Route، أمر إنتاج، Estimate، Schedule، Release، Start، صرف مواد، تسجيل وقت، Report as Finished، جودة/هالك، Close، WIP، Variances، وقيود مالية.

التقارير والانحرافات تكشف الفشل مبكرًا. لذلك يجب أن تكون WIP Statement وProduction Variance Report وProduction Order Cost Analysis وتقارير الصرف والهالك جزءًا من التطبيق، لا إضافة لاحقة.

تطبيق الإنتاج يحتاج تعاون الإنتاج والمستودعات والمالية، وليس فريق ERP فقط. إذا لم تفهم المالية WIP وPosting، ولم تفهم المستودعات الصرف والاستلام، ولم يوثق الإنتاج BOM وRoute، فالتطبيق سيبقى ضعيفًا حتى لو كان النظام قويًا.
