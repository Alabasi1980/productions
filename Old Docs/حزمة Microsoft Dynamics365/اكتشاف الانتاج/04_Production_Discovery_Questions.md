# 04_Production_Discovery_Questions.md
# أسئلة تحليل عميل الإنتاج في Microsoft Dynamics 365

## 1. مقدمة

هذا الملف يحوّل مفاهيم **Microsoft Dynamics 365 Production Control** إلى أسئلة تحليل عملية تساعد المستشار الوظيفي أو محلل الأعمال على فهم دورة الإنتاج لدى العميل قبل التطبيق.

الهدف من الأسئلة ليس تعبئة نموذج شكلي، بل كشف الواقع التشغيلي والمالي الذي سيُبنى عليه تصميم النظام، خصوصًا في المحاور التالية:

- طريقة الإنتاج الفعلية.
- طريقة إدارة المواد والمخزون.
- طريقة إعداد BOM وRoutes.
- طريقة احتساب التكلفة.
- العلاقة بين الإنتاج والمالية.
- سياسة WIP والإغلاق.
- التقارير المطلوبة.
- المخاطر قبل Go-Live.

في Dynamics 365، الإنتاج ليس شاشة صرف مواد واستلام منتج فقط. الإنتاج دورة مترابطة تبدأ من **Released Products وBOM وRoute**، ثم تمر بأمر الإنتاج وحالاته، وتنتهي بالتكلفة الفعلية والانحرافات والإغلاق المالي.

## 2. طريقة استخدام هذا الملف

يُستخدم هذا الملف كأداة مقابلات وتحليل، لا كقائمة تُقرأ مرة واحدة على العميل. يجب أن يختار المستشار الأسئلة المناسبة حسب نوع العميل ونضجه وطبيعة إنتاجه.

قواعد الاستخدام العملية:

- لا تُطرح كل الأسئلة دفعة واحدة.
- يتم اختيار الأسئلة حسب نوع التصنيع: Discrete، Process، Lean، Project/Engineer to Order.
- يجب حضور ممثلين من الإنتاج، المستودعات، المالية، والمحاسبة.
- يجب توثيق الإجابات وتحويلها إلى إعدادات أو متطلبات نظام أو فجوات.
- يجب التمييز بين متطلبات المرحلة الأولى وما يمكن تأجيله.
- أي إجابة غير واضحة في BOM أو التكلفة أو WIP يجب اعتبارها مخاطرة، لا مجرد نقص معلومات.

| نوع الاجتماع | الأقسام المطلوبة | الهدف |
|---|---|---|
| اجتماع فهم عام للإنتاج | الإنتاج، التخطيط، ERP | فهم طبيعة الإنتاج وأنواعه ومصادر أوامر الإنتاج. |
| اجتماع Master Data | الإنتاج، المستودعات، المالية | تحديد المنتجات، المواد، الوحدات، الأبعاد، مجموعات الأصناف. |
| اجتماع BOM / Formula | الإنتاج، الهندسة، الجودة، التكلفة | توثيق وصفات المنتجات واعتماداتها والهالك والبدائل. |
| اجتماع Routes / Resources | الإنتاج، الصيانة، التخطيط، التكلفة | فهم العمليات والموارد والأوقات وأثرها على الجدولة والتكلفة. |
| اجتماع المخزون والصرف | المستودعات، الإنتاج، المالية | تحديد متى وكيف تُصرف المواد وكيف تُعالج المرتجعات. |
| اجتماع التكلفة والمالية | المالية، المحاسبة، مراقبة التكلفة، الإنتاج | تحديد Standard/Actual Cost وWIP وPosting وVariances. |
| اجتماع الجودة والهالك | الجودة، الإنتاج، المالية | تحديد الفحص والهالك والمنتجات المرفوضة والمعالجة المالية. |
| اجتماع التقارير وUAT | الإدارة، الإنتاج، المالية، ERP | تحديد التقارير وسيناريوهات الاختبار قبل التشغيل. |

## 3. أسئلة الفهم العام لطبيعة الإنتاج

| السؤال | الهدف من السؤال | أثر الإجابة على تصميم النظام |
|---|---|---|
| ماذا تنتجون بالضبط؟ منتج واحد أم عائلة منتجات؟ | تحديد نطاق Module الإنتاج. | يحدد هيكل المنتجات وBOM وRoutes والتقارير. |
| هل الإنتاج للتخزين Make to Stock أم حسب الطلب Make to Order؟ | فهم علاقة الإنتاج بالطلب والمخزون. | يؤثر على Master Planning، المخزون، وأوامر الإنتاج. |
| هل يوجد Configure to Order؟ | معرفة هل المنتج يتكوّن حسب خيارات العميل. | قد يتطلب ضبط Product variants أو منطق تكوين. |
| هل يوجد Engineer to Order؟ | معرفة هل هناك مرحلة هندسية قبل الإنتاج. | قد يتطلب ربط الإنتاج بالمشاريع أو مراحل هندسية. |
| هل لديكم أكثر من مصنع أو موقع إنتاج؟ | تحديد مفهوم Site. | يؤثر على المخزون، التكلفة، BOM/Route حسب الموقع. |
| هل يوجد أكثر من خط إنتاج؟ | فهم الطاقة والتوزيع التشغيلي. | يؤثر على الموارد والجدولة والتقارير. |
| هل الإنتاج مستمر أم على دفعات؟ | تحديد نمط الأمر المناسب. | قد يفرق بين Production order وBatch order. |
| هل الإنتاج يدوي، آلي، أو مختلط؟ | معرفة دور العمال والآلات. | يؤثر على Route، الموارد، Cost Categories. |
| هل لديكم منتجات نصف مصنعة؟ | فهم تعدد مستويات الإنتاج. | يؤثر على BOM متعدد المستويات أو Phantom أو Pegged supply. |
| هل يوجد إنتاج داخلي وخارجي؟ | تحديد وجود Subcontracting. | قد يتطلب Vendor line أو مورد خارجي ضمن العملية. |

## 4. أسئلة المنتجات والأصناف Master Data

| السؤال | لماذا مهم؟ | أثره على الإنتاج/المخزون/التكلفة |
|---|---|---|
| ما المنتجات النهائية التي يتم إنتاجها؟ | تحديد Released Products المستخدمة كمخرجات إنتاج. | يحدد أوامر الإنتاج والتكلفة والمخزون النهائي. |
| ما المواد الخام الداخلة في الإنتاج؟ | تحديد مدخلات BOM. | يؤثر على الصرف والمخزون وتكلفة المنتج. |
| هل توجد منتجات نصف مصنعة؟ | معرفة هل المنتج يدخل كمخرج وكمدخل. | يؤثر على BOM متعدد المستويات أو Phantom أو أوامر فرعية. |
| هل توجد أصناف تُشترى وتُصنع في نفس الوقت؟ | تحديد سياسة التوريد لكل صنف. | يؤثر على التخطيط، الشراء، والإنتاج. |
| ما وحدات القياس المستخدمة؟ | ضمان صحة الكميات. | خطأ الوحدة قد يؤدي إلى صرف غير منطقي وتكلفة خاطئة. |
| هل توجد تحويلات بين الوحدات؟ | ربط الشراء والتخزين والإنتاج. | يحد من أخطاء الكمية والتكلفة. |
| هل تختلف وحدة الشراء عن وحدة الإنتاج أو التخزين؟ | كشف الحاجة إلى تحويلات واضحة. | يؤثر على BOM والصرف ومطابقة المخزون. |
| هل يوجد تتبع Batch أو Serial؟ | تحديد أبعاد التتبع. | يؤثر على الصرف، الاستلام، الجودة، والتتبع. |
| هل تختلف المنتجات حسب الموقع أو المصنع؟ | تحديد أثر Site. | قد يتطلب BOM/Route/Cost مختلفة حسب الموقع. |
| هل توجد مجموعات أصناف تؤثر على الحسابات المالية؟ | تحديد Item Groups. | تؤثر على حسابات المخزون وقيود الإنتاج. |
| هل Item Model Group محددة لكل صنف؟ | تحديد طريقة تقييم المخزون. | تؤثر على Standard/FIFO/Moving average والتكلفة. |
| هل توجد Storage Dimensions مثل Warehouse وLocation؟ | تحديد مستوى إدارة المخزون. | تؤثر على الصرف والاستلام والمواقع الفعلية. |

## 5. أسئلة BOM / Formula

| السؤال | الهدف | أثر الإجابة على النظام |
|---|---|---|
| هل لكل منتج BOM موثقة؟ | التأكد من وجود وصفة إنتاج قابلة للنظام. | بدون BOM دقيقة، تكلفة وصرف المواد سيكونان غير موثوقين. |
| من المسؤول عن إعداد واعتماد BOM؟ | تحديد مالك البيانات. | يؤثر على الصلاحيات ودورة الاعتماد. |
| هل BOM ثابتة أم تتغير؟ | قياس استقرار وصفة المنتج. | يؤثر على Versioning والتدقيق. |
| هل توجد BOM متعددة لنفس المنتج؟ | معرفة البدائل أو اختلاف المواقع/الفترات. | يتطلب BOM Versions وقيود تفعيل. |
| هل تختلف BOM حسب الموقع أو الكمية أو الفترة؟ | تحديد شروط صلاحية BOM. | يؤثر على Site والكمية وتاريخ السريان. |
| هل توجد BOM للتخطيط وأخرى للإنتاج وأخرى للتكلفة؟ | فهم مستوى النضج. | قد يتطلب فصل Planning BOM وProduction BOM وCosting BOM. |
| هل توجد مواد بديلة؟ | معرفة التعامل مع نقص المواد. | يؤثر على الاستبدال والانحرافات. |
| هل توجد منتجات نصف مصنعة داخل BOM؟ | تحديد مستويات الإنتاج. | يحدد هل نستخدم أمر فرعي أو Phantom أو Pegged supply. |
| هل المنتجات نصف المصنعة تُنتج بأمر منفصل أم تعامل كـ Phantom؟ | تحديد طريقة التفجير والإنتاج. | يؤثر على عدد أوامر الإنتاج والتكلفة والتتبع. |
| هل توجد Pegged Supply؟ | معرفة هل هناك توريد مرتبط مباشرة بالأمر. | قد ينشئ أوامر إنتاج أو شراء فرعية مرتبطة. |
| هل يوجد تصنيع خارجي Vendor Line؟ | تحديد عمليات المورد الخارجي. | يؤثر على الشراء والتكلفة وسير العمل. |
| هل توجد Co-products أو By-products؟ | تحديد وجود أكثر من مخرج من نفس الأمر. | يؤثر على Formula وتوزيع التكلفة. |
| هل تستخدمون Formula بدل BOM في بعض المنتجات؟ | تحديد نمط Process manufacturing. | يؤثر على وصفة المنتج وBatch orders. |
| هل توجد صلاحية زمنية لـ BOM؟ | تحديد تاريخ بداية ونهاية الاستخدام. | يمنع استخدام وصفة قديمة أو غير صحيحة. |
| هل توجد كميات Minimum / Maximum مرتبطة بالـ BOM؟ | معرفة اختلاف الوصفة حسب حجم الأمر. | يؤثر على اختيار Version المناسبة. |
| هل توجد نسبة هالك على مستوى BOM؟ | قياس الفاقد المتوقع. | يؤثر على احتياج المواد والتكلفة والانحرافات. |
| هل الهالك ثابت Constant أم متغير Variable؟ | تحديد طريقة احتساب الفاقد. | يؤثر على الكمية المطلوبة وتكلفة المنتج. |

### علامات الخطر في BOM

| علامة الخطر | ماذا تعني؟ | الأثر المحتمل |
|---|---|---|
| BOM غير موثقة | الإنتاج يعتمد على خبرة أشخاص لا بيانات نظام. | صرف وتكلفة غير دقيقة. |
| اعتماد شفهي للوصفة | لا توجد مسؤولية واضحة عن صحة BOM. | نزاعات وتشوهات تكلفة. |
| كثرة التغييرات دون Versioning | لا يمكن معرفة أي وصفة استُخدمت. | صعوبة تفسير الانحرافات. |
| عدم معرفة الهالك | المواد المطلوبة والتكلفة ناقصة. | عجز مخزون وانحرافات متكررة. |
| عدم وضوح المنتجات نصف المصنعة | لا يمكن تصميم دورة إنتاج صحيحة. | أوامر ناقصة أو تعقيد غير مضبوط. |
| اختلاف الواقع عن النظام | BOM النظام لا تمثل الإنتاج الفعلي. | تقارير مضللة وفشل UAT. |

## 6. أسئلة Routes / Operations / Resources

| السؤال | الهدف | أثر الإجابة على الجدولة والتكلفة |
|---|---|---|
| هل لديكم مراحل إنتاج واضحة؟ | تحديد Route. | بدون Route واضحة تصبح الجدولة والتكلفة التشغيلية ضعيفة. |
| ما العمليات الرئيسية؟ | تحديد Operations. | يؤثر على تتبع التقدم والتكلفة. |
| هل العمليات تسلسلية أم متوازية؟ | تحديد Simple route أو Route network. | يؤثر على الجدولة وحجز الموارد. |
| هل يوجد Route لكل منتج؟ | ربط المنتج بمراحل التصنيع. | يؤثر على Estimated Cost والتخطيط. |
| هل يمكن استخدام Route مختلفة لنفس المنتج؟ | معرفة بدائل التشغيل. | قد يتطلب Route versions. |
| هل توجد Operations مشتركة بين أكثر من منتج؟ | إعادة استخدام العمليات. | يساعد في التوحيد والتحليل. |
| ما الموارد المطلوبة لكل عملية؟ | تحديد Resources. | يؤثر على الطاقة والتكلفة. |
| هل الموارد عمال، آلات، أدوات، مواقع، أو مورد خارجي؟ | تصنيف نوع المورد. | يحدد طريقة الجدولة والتكلفة. |
| هل تستخدمون Work Centers أو Resource Groups؟ | فهم تجميع الموارد. | يساعد في تخطيط الطاقة. |
| هل لكل مورد Calendar أو ساعات عمل؟ | تحديد الطاقة المتاحة. | خطأ التقويم يؤدي إلى جدولة غير واقعية. |
| هل يوجد Setup Time؟ | تحديد وقت الإعداد الثابت. | يؤثر على التكلفة وحجم الدفعة. |
| هل يوجد Run Time؟ | تحديد وقت التشغيل لكل وحدة. | يؤثر على تكلفة الوحدة والجدولة. |
| هل يتم تسجيل الوقت الفعلي؟ | معرفة الفرق بين المخطط والواقع. | ضروري لتحليل الانحرافات. |
| هل تدخل تكلفة الوقت أو الآلة في تكلفة المنتج؟ | تحديد Cost Categories. | يؤثر على Standard/Actual Cost. |
| هل توجد Cost Categories؟ | تحديد تكلفة الإعداد والتشغيل. | بدونها قد لا تظهر تكلفة الموارد ماليًا. |

## 7. أسئلة أوامر الإنتاج Production Orders

| السؤال | الهدف | أثر الإجابة على دورة العمل |
|---|---|---|
| من ينشئ أمر الإنتاج؟ | تحديد المسؤولية. | يؤثر على الصلاحيات وسير العمل. |
| هل مصدر الأمر يدوي؟ | معرفة دور المستخدم. | يتطلب شاشة وإجراءات إدخال. |
| هل مصدر الأمر Master Planning؟ | معرفة وجود تخطيط آلي. | يتطلب إعداد MRP وCoverage. |
| هل مصدر الأمر Sales Order؟ | ربط الإنتاج بطلب العميل. | مناسب لـ Make to Order. |
| هل مصدر الأمر أمر إنتاج أعلى؟ | معرفة وجود أوامر فرعية. | يؤثر على Pegged supply. |
| من يعتمد أمر الإنتاج؟ | تحديد الموافقات. | يمنع بدء أوامر غير جاهزة. |
| هل توجد حالات واضحة للأمر؟ | ضبط دورة الحياة. | يربط Created/Estimated/Scheduled/Released/Started/RAF/Ended. |
| هل يتم تقدير الأمر قبل البدء؟ | حساب Estimated Cost والاحتياجات. | يؤثر على المخزون والتكلفة. |
| هل يتم جدولته؟ | حجز الموارد. | يؤثر على الطاقة والالتزام بالمواعيد. |
| هل يتم إطلاقه Release قبل التنفيذ؟ | جاهزية الصالة والمستودع. | يؤثر على Picking وJob/Route cards. |
| هل يسمح بتعديل BOM أو الكمية بعد إنشاء الأمر؟ | فهم المرونة المطلوبة. | يؤثر على التدقيق والانحرافات. |
| هل يسمح بتعديل الأمر بعد البدء؟ | ضبط التحكم. | قد يسبب فروقات إن لم يكن محكومًا. |
| هل يتم إنتاج الكمية كاملة أم جزئيًا؟ | تحديد RAF جزئي. | يؤثر على WIP والمخزون. |
| هل يمكن إلغاء الأمر؟ | تحديد حالات الاستثناء. | يحتاج قواعد للمخزون والتكلفة. |
| هل توجد أوامر إنتاج فرعية؟ | تحديد بنية الإنتاج. | يؤثر على التخطيط والتكلفة. |

## 8. أسئلة التخطيط والجدولة

| السؤال | الهدف | أثر الإجابة على التخطيط |
|---|---|---|
| هل تستخدمون تخطيط احتياجات المواد MRP / Master Planning؟ | معرفة مصدر الطلب الإنتاجي. | يؤثر على Planned production orders. |
| هل الإنتاج مبني على Forecast؟ | تحديد Make to Stock. | يؤثر على التخطيط والمخزون. |
| هل الإنتاج مبني على طلبات بيع؟ | تحديد Make to Order. | يربط الإنتاج بالمبيعات. |
| هل توجد حد أدنى وحد أعلى للمخزون؟ | تحديد سياسات التغطية. | يؤثر على أوامر التخطيط. |
| هل يتم التخطيط حسب توفر المواد؟ | ربط الإنتاج بالمخزون. | يمنع أوامر غير قابلة للتنفيذ. |
| هل يتم التخطيط حسب توفر الموارد؟ | ربط الإنتاج بالطاقة. | يحدد Finite capacity. |
| هل تستخدمون Finite Capacity أم Infinite Capacity؟ | تحديد واقعية الجدولة. | Finite يحترم القيود، Infinite يتجاهلها. |
| هل يتم التخطيط للأيام أو الساعات؟ | تحديد مستوى التفصيل. | يؤثر على Operations/Job scheduling. |
| هل توجد تقاويم عمل للموارد؟ | تحديد ساعات العمل المتاحة. | خطأ التقويم يسبب فشل الجدولة. |
| هل توجد أوقات توقف وصيانة تؤثر على الجدولة؟ | معرفة العوائق الواقعية. | يحتاج تحقق إضافي من المصدر الأصلي. |
| هل تحتاجون Forward Scheduling أو Backward Scheduling؟ | تحديد اتجاه الجدولة. | يؤثر على تاريخ البداية أو الالتزام بتاريخ التسليم. |

## 9. أسئلة صرف المواد Material Issue

| السؤال | الهدف | أثر الإجابة على المخزون والتكلفة |
|---|---|---|
| متى تصرف المواد للإنتاج؟ | تحديد توقيت الاستهلاك. | يؤثر على المخزون وWIP. |
| هل الصرف قبل البدء أم عند البدء أم عند الانتهاء؟ | تحديد Flushing principle. | يؤثر على Preflushing/Backflushing. |
| هل الصرف يدوي أم تلقائي؟ | تحديد مسؤولية التسجيل. | اليدوي أدق رقابيًا، التلقائي أسرع لكنه يخفي الفروقات حتى الإغلاق. |
| هل تستخدمون Picking List؟ | معرفة اليومية الرسمية للصرف. | يؤثر على المستودعات والقيود. |
| هل الصرف مبني على BOM؟ | ربط الصرف بالوصفة. | أي اختلاف يظهر كفروقات. |
| هل يسمح بصرف كمية أكبر من BOM؟ | تحديد سماحية الاستهلاك. | قد يسبب Production quantity variance. |
| هل يسمح بصرف كمية أقل؟ | تحديد معالجة النقص. | قد يخفض التكلفة ظاهريًا ويخلق فرقًا تشغيليًا. |
| هل يسمح بصرف مادة بديلة؟ | تحديد سياسة البدائل. | قد يسبب Substitution variance. |
| من يعتمد الصرف؟ | تحديد الرقابة. | يقلل صرف مواد غير مبرر. |
| هل توجد مرتجعات مواد من الإنتاج؟ | تحديد عكس الصرف. | تخفض WIP وتعيد المواد للمخزون. |
| هل يوجد مستودع خاص بالإنتاج؟ | تحديد Warehouse/Location. | يؤثر على حركة المواد والتتبع. |
| هل يتم تسجيل Batch/Serial عند الصرف؟ | ضمان التتبع. | مهم للجودة وتتبع الدفعات. |
| هل يوجد فرق بين المواد المخططة والمصروفة فعليًا؟ | تحليل الانحرافات. | يؤثر على تكلفة الأمر والإدارة. |

## 14. أسئلة WIP والإغلاق المالي

| هل المحاسبة تتابع الإنتاج تحت التشغيل؟ | معرفة مستوى الرقابة. | يؤثر على تقارير الفترة المالية. |

| هل تريدون تحليل الانحرافات بعد الإغلاق؟ | استثمار نتائج الإغلاق. | يربط المالية بتحسين الإنتاج. |
| هل توجد أوامر إنتاج مفتوحة لفترات طويلة؟ | كشف تضخم WIP. | يشوه الميزانية والتقارير. |
| كيف تتعاملون مع أوامر إنتاج غير مكتملة نهاية الشهر؟ | تحديد سياسة Period Closing. | يؤثر على WIP والمالية. |

## 21. علامات الخطر أثناء التحليل

| علامة الخطر | ماذا تعني؟ | الإجراء المقترح |
|---|---|---|
| العميل لا يملك BOM واضحة | لا توجد قاعدة موثوقة للصرف والتكلفة. | تأجيل تطبيق الإنتاج حتى توثيق BOM. |
| لا يوجد مسؤول يعتمد BOM | لا يوجد مالك بيانات. | تحديد مسؤول واعتماد رسمي. |
| العميل لا يعرف تكلفة المنتج | نموذج التكلفة غير ناضج. | عقد جلسة تكلفة مع المالية والإنتاج. |
| لا يوجد إغلاق إنتاج | WIP قد يكون متضخمًا. | وضع سياسة إغلاق أسبوعية أو شهرية. |
| لا توجد سياسة هالك | التكلفة والمواد غير دقيقة. | تعريف Constant/Variable Scrap وسياسة Error quantity. |
| لا يوجد ربط بين الإنتاج والمالية | التقارير لن تطابق الأستاذ العام. | ضبط Posting وWIP وVariance accounts. |
| التقارير المطلوبة غير محددة | لا يوجد تصور رقابي. | تحديد تقارير لكل دور. |
| المستخدمون لا يعرفون الفرق بين الدعم والتطوير | خطر توسع النطاق. | فصل Gap عن Change Request. |
| العميل يريد تكلفة دقيقة لكنه لا يسجل بيانات دقيقة | تناقض تشغيلي. | إلزام تسجيل الصرف والوقت وRAF والإغلاق. |
| Route غير موثقة | الجدولة والتكلفة التشغيلية غير موثوقة. | توثيق العمليات والأوقات والموارد. |
| Standard Cost غير محدّثة | ستظهر Variances كبيرة وغير مفيدة. | تحديد دورة مراجعة تكلفة. |
| تطبيق نموذج متقدم على عميل غير جاهز | تعقيد يفوق نضج العميل. | تبسيط المرحلة الأولى وتأجيل التعقيد. |

## 23. ملخص تنفيذي

أسئلة التحليل هي أداة لاكتشاف الواقع التشغيلي، وليست مجرد نموذج مقابلة. نجاح تطبيق الإنتاج في ERP يعتمد على وضوح خمسة محاور رئيسية:

1. **BOM دقيقة ومعتمدة** تحدد المواد والهالك والبدائل.
2. **Route واضح** يحدد العمليات والموارد والأوقات.
3. **سياسة تكلفة محددة** تبيّن هل سيتم استخدام Standard Cost أو Actual Cost أو Planned Cost.
4. **قرار WIP والإغلاق** لضبط العلاقة بين الإنتاج والمالية.
5. **تقارير رقابية** تكشف الاستهلاك والانحرافات والأوامر المفتوحة.

لا يجب تطبيق Module الإنتاج قبل وضوح المخزون والمالية. أي تطبيق يبدأ قبل توثيق BOM وRoute وسياسة التكلفة سيولد تقارير غير موثوقة، وسيحوّل النظام إلى مصدر خلاف بين الإنتاج والمستودعات والمالية.

المطلوب من المستشار بعد جلسات التحليل هو تحويل إجابات العميل إلى أحد ثلاثة أشياء:

- إعدادات نظام واضحة.
- تقارير مطلوبة.
- متطلبات تطوير أو فجوات موثقة.

القاعدة العملية: **العميل الذي يريد تكلفة دقيقة يجب أن يلتزم بتسجيل بيانات دقيقة**: صرف المواد، وقت التشغيل، الاستلام، الهالك، والإغلاق.
